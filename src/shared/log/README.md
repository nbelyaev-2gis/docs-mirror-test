# Как залогировать ошибку

Обязательные поля смотри на примере:

```tsx
logError({
  // случайный уникальный id, чтобы искать потом этот лог в коде
  id: '3xmp1',

  // fatal, error, warn или info — обсуди с командой, если не уверен
  level: 'error'

  // сообщение, по которому должно быть понятно, что именно пошло не так
  message: error?.message || 'Неизвестная ошибка',

  payload: {
    // тип ошибки, по которому потом можно будет фильтровать логи
    // (если много разных ошибок с одним типом, добавь subtype для уточнения)
    type: 'ApiNetworkError',
  }
});
```

## Поля в payload

### type — тип ошибки

Кратко описывает, где произошло, например:

- `ApiNetworkError` — ошибка сети при запросе в API
- `ComponentDidCatch` — компонент при рендере поймал ошибку
- `ClientStartup` — приложение запустилось

Не шаблонизируй `type`, если для разных значений хочешь сделать разные ошибки — для этого есть `subtype`.

### subtype — подтип ошибки, если ошибки с type бывают разными

В `subtype` можно положить название API и прочие вариации ошибки.

Например, `type=ApiNetworkError, subtype=gefest` — отвалился запрос к апи гефеста.

### data — произвольная строка для дебага

Если по типу ошибки не очевидно, как её чинить, можно прокинуть дополнительную информацию через поле `data`. Это поможет быстрее воспроизвести проблему или сразу по логу понять, на чьей она стороне.

Например:

- Ошибка при парсинге какого-то значения? Кладём в `data` сырое значение.
- Ошибка при вызове метода с конкретными параметрами? Кладём в `data` параметры вызова.

### value — произвольное число для диаграмм/графиков

Если нам хочется визуализировать какую-то численную величину, используем универсальное поле `value`.

### stack — stack-trace для исключений

Прокидывай сюда `e?.stack`, если логируешь исключение.

## Как выбрать уровень ошибки <a href="#type">

Исходи из того, какую реакцию на такую ошибку ты ожидаешь.

- `fatal`. Бросаем всё и фиксим.
- `error`. Минимизируем количество.
- `warn`. Реагируем только на резко повысившийся фон.
- `info` Не реагируем (не используем для мониторинга).
