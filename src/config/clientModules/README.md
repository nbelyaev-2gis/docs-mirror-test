# Система редиректов

## Важно

Перед внесением изменений убедитесь, что вы понимаете, как работают редиректы и их приоритеты.

**ВНИМАНИЕ:** Использование редиректов замедляет переходы по ссылкам. Добавляйте редиректы только при крайней необходимости.

**ОБЯЗАТЕЛЬНО:** При добавлении нового редиректа всегда указывайте в комментарии дату, когда его можно будет удалить. Например:

```typescript
// TODO: Удалить после марта 2026
['/old-path', '/new-path'],
```

### Добавление редиректов

Для добавления новых редиректов отредактируйте файл `src/config/clientModules/clientRedirects.ts`.

#### Типы редиректов:

##### Точные редиректы (exactRedirects)

Для простых точных совпадений URL:

```typescript
const exactRedirects = new Map<string, string>([
  ['/старый-путь', '/новый-путь'],
  ['/pages', '/mapgl/overview'],
]);
```

##### Суффиксные редиректы (suffixRedirects)

Для редиректов, которые работают с любым языковым префиксом:

```typescript
const suffixRedirects = new Map<string, string>([
  ['/старый-путь', '/новый-путь'],
  ['/pro/layers', '/pro/dashboards/layers/create'],
]);
```

##### Регулярные выражения (regexRedirects)

Для сложных правил редиректов:

```typescript
const regexRedirects = [
  {
    pattern: /^\/старый-путь\/(.*)$/,
    replacement: '/новый-путь/$1',
  },
];
```

## Примеры использования

### Пример 1: Простой редирект

```typescript
// В exactRedirects
['/old-page', '/new-page'];
```

### Пример 2: Редирект с сохранением языка

```typescript
// В suffixRedirects
['/pro/old-section', '/pro/new-section'];
// Будет работать для /ru/pro/old-section → /ru/pro/new-section
```

### Пример 3: Редирект с регулярными выражениями

```typescript
// В regexRedirects
{
  pattern: /^(.*)\/api\/old\/(.*)$/,
  replacement: '$1/api/new/$2',
}
```

## Работа с регулярными выражениями

Для отладки регулярок желательно использовать онлайн инструменты, например [regex101](https://regex101.com/).
Гайд по регулярным выражениям на русском: [MDN RegExp](https://developer.mozilla.org/ru/docs/Web/JavaScript/Guide/Regular_Expressions).

## Тестирование редиректов

1. Добавьте новые редиректы в соответствующий раздел
2. Перезапустите сервер разработки
3. Проверьте работу редиректов в браузере
4. Убедитесь, что редиректы работают для всех языков

## Важные замечания

- Редиректы обрабатываются в порядке приоритета: точные → суффиксные → регулярные выражения
- Хеши URL автоматически сохраняются при редиректах
- Система поддерживает работу с историей браузера
- Редиректы применяются как при загрузке страницы, так и при клике по ссылкам
  ь, что URL изменяется корректно без перезагрузки страницы
