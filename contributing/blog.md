## Где лежат посты

- Все посты хранятся в папке `blog`.
- Один пост = один файл `.md` или `.mdx`.
- Для новых материалов используйте формат имени файла: `YYYY-MM-DD-post-slug.mdx`.

Примеры:

- `blog/2026-04-01-2gis-api-platform-updates.mdx`
- `blog/2026-03-31-my-first-post.md`

## Как создать новый пост

1. Создайте новый файл в папке `blog` по шаблону имени `YYYY-MM-DD-post-slug.mdx`.
2. Добавьте front matter (метаданные в начале файла).
3. Напишите лид и добавьте `<!--truncate-->` в месте, где должна обрываться превью-часть поста в списке.
4. Добавьте основной контент поста.
5. Запустите локально сайт и проверьте, что пост открывается и корректно рендерится.

## Шаблон поста

```mdx
---
slug: post-slug
title: "Заголовок поста"
authors:
	- name: Имя Фамилия
		title: Роль
tags: [новости, api]
date: 2026-04-02
---

# Заголовок поста

Короткий лид (1-2 абзаца), который объясняет, о чём материал.

<!--truncate-->

## Раздел

Основной контент поста.
```

## Поля front matter

- `slug`: часть URL поста.
- `title`: заголовок поста.
- `authors`: автор(ы) поста.
- `tags`: теги поста.
- `date`: дата публикации в формате `YYYY-MM-DD`.

## Авторы

Есть два способа указать автора:

- Inline-описание в `authors` прямо в посте.
- Переиспользуемый автор через `blog/authors.yml`.

Пример через `authors.yml`:

```mdx
---
title: 'Пример'
slug: example
authors: [username]
tags: [пример]
date: 2026-04-02
---
```

Где `username` — ключ из `blog/authors.yml`.

## MDX и медиа

- Для сложного контента (React-компоненты, демо, табы) используйте `.mdx`.
- Картинки храните в `static/img` и подключайте по пути вида `/img/...`.
- Видео и другие медиа размещайте в `static` и подключайте относительным веб-путём.

## Локальная проверка

Запуск локального сервера:

```bash
npm start
```

Что нужно проверить локально перед публикацией:

- пост появляется в списке блога;
- корректно работает обрезка превью по `<!--truncate-->`;
- URL соответствует `slug`;
- нет ошибок рендера MDX-компонентов.

## Локализация поста (en)

Если пост должен быть доступен на английском, добавьте перевод в папку `i18n/en/docusaurus-plugin-content-blog`.

Порядок действий:

1. После создания исходного поста в `blog` создайте файл-перевод с тем же именем в `i18n/en/docusaurus-plugin-content-blog`.
2. Скопируйте структуру документа и переведите контент на английский.
3. Сохраните совместимые метаданные (`slug`, `date`, структура разделов), чтобы русская и английская версии соответствовали друг другу.
4. Если используете `authors`, убедитесь, что авторы корректно отображаются и для en-версии.

Пример:

- RU: `blog/2026-04-01-2gis-api-platform-updates.mdx`
- EN: `i18n/en/docusaurus-plugin-content-blog/2026-04-01-2gis-api-platform-updates.mdx`

Что нужно проверить локально перед публикацией:

- пост открывается в русской локали;
- пост открывается в английской локали;
- у обеих версий совпадает структура и нет ошибок рендера.
