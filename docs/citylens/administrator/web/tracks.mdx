# Треки (Tracks)

Вкладка **Tracks** содержит треки без группировки. Трек — это временной интервал и собранные в течение него данные.

![Вкладка Tracks](/img/citylens/web-tracks.png)

## Фильтры {#filters}

Настройте фильтры для удобного просмотра данных:

* **Date**: тип даты, по которой фильтровать результаты:

   * **Uploads**: дата загрузки видео в CityLens Web.
   * **Recordings**: дата записи видео водителем.

* **Type**: источник видео:

   * **Mobile app**: мобильное приложение <Brand /> Ситискан.
   * **Dashcam**: видеорегистратор.
   * **All**: оба источника.

* **From**/**To**: ограничить результаты по временному интервалу (включая выбранные даты).

* **Polygon**: территория.

* **Email**: email водителя.

Нажмите **Submit**, чтобы применить фильтры.

## Данные {#data}

Данные появляются в CityLens Web только после того, как водитель [загружает записанные треки](/citylens/driver/record#upload-tracks).

Таблица данных содержит следующие параметры:

* **Type**: тип источника видео. Возможные значения: `mob` (мобильное приложение с указанием версии) и `dash` (видеорегистратор).

* **Upload/recording date**: cостоит из двух строк:

   * дата и временная метка (UTC) загрузки видео;
   * дата и временная метка (UTC) записи видео.

* **Track ID**: cостоит из двух строк:

   * уникальный ID трека;
   * email автора (водителя).

* **Length/Time**: информация о треке. Состоит из двух строк:

   * Длина трека в километрах. Нажмите на это значение, чтобы отобразить маршрут на карте.
   * Длительность трека в минутах.

* **Frames count**: количество кадров, извлечённых из трека.

* **Upload**: статус загрузки видео с мобильного устройства. Нажмите, чтобы увидеть детали.

* **Detections localization**: статус распознавания объектов на треке.

* **Processing PRO**: статус отправки трека в <Brand /> Про.

* **Processing**: статус трека. Нажмите, чтобы увидеть информацию обо всех кадрах на треке.

* **Signs/Errors/Corrections**: содержит три значения, разделённые `/`:

   * количество распознанных объектов;
   * количество обнаруженных ошибок;
   * количество предложенных исправлений.

* **KML**: ссылка для загрузки трека в виде файла формата `.kml`.

## Операции с данными {#operations}

### Скачивание данных {#downloading}

1. Если необходимо, настройте фильтры или используйте чекбоксы, чтобы выбрать отдельные элементы.
2. Прокрутите страницу вниз и нажмите **CSV**.

### Удаление данных из <Brand /> Про {#removing}

По умолчанию в <Brand /> Про автоматически загружаются все данные. Чтобы скрыть в <Brand /> Про определённые элементы:

1. Если необходимо, настройте фильтры или используйте чекбоксы, чтобы выбрать отдельные элементы.
2. Прокрутите страницу вниз и нажмите **Hide from PRO**.

### Повторная отправка данных в <Brand /> Про {#reuploading}

Если данные были удалены из <Brand /> Про или были некорректно обработаны, вы можете отправить их заново:

1. Если необходимо, настройте фильтры или используйте чекбоксы, чтобы выбрать отдельные элементы.
2. Прокрутите страницу вниз и нажмите **Send to PRO**.
