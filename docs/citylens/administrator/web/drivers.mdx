# Водители (Drivers)

Вкладка **Drivers** предоставляет статистику по записанным и отправленным данным для каждого водителя по дням.

![Вкладка Drivers](/img/citylens/web-drivers.png)

## Фильтры {#filters}

Настройте фильтры для удобного просмотра данных:

* **Date**: тип даты, по которой фильтровать результаты:

  * **Uploads**: дата загрузки видео в CityLens Web.
  * **Recordings**: дата записи видео водителем.

* **Type**: источник видео:

  * **Mobile app**: мобильное приложение <Brand /> Ситискан.
  * **Dashcam**: видеорегистратор.
  * **All**: оба источника.

* **From**/**To**: ограничить результаты по временному интервалу (включая выбранные даты).

* **Polygon**: территория.

* **Email**: email водителя.

Нажмите **Submit**, чтобы применить фильтры.

## Данные {#data}

Данные появляются в CityLens Web только после того, как водитель [загружает записанные треки](/citylens/driver/record#upload-tracks).

Таблица данных содержит следующие параметры:

* **Date**: дата загрузки или записи видео в зависимости от установленного [фильтра](#filters).

   Под датой отображается тег, соответствующий типу источника видео: `mobile` (мобильное приложение) или `dash` (видеорегистратор).

* **Email**: email пользователя, который загрузил видео.

* **Length, km**: состоит из двух строк:

   * Пройденное расстояние.

     Нажмите на это значение, чтобы посмотреть на вкладке **Tracks** все видео, загруженные водителем в этот день. Статус обработки или повторная загрузка одного и того же видео не влияет на это значение.

   * Количество распознанных объектов на 1 км до записи (ожидаемое) и после (фактическое). Это значение помогает оценить плотность объектов на пройденных маршрутах.

* **Time**: общая продолжительность видео.

* **Frames count**: количество кадров, извлечённых из видео.

* **Upload**: статистика загрузки видео. Состоит из двух значений, разделённых `|`:

   * количество видео, которые ещё загружаются или загружены не до конца;
   * количество полностью загруженных видео.

* **Processing**: статистика обработки видео. Состоит из трёх значений, разделённых `|`:

   * количество успешно обработанных видео;
   * количество видео, которые не были обработаны из-за проблем с данными (плохие условия записи или отсутствие объектов);
   * количество видео, которые не были обработаны из-за технических проблем.

   Нажмите на каждое значение, чтобы увидеть статистику по отдельному видео.

* **Errors/Corrections**: обнаруженные ошибки и предложенные исправления. Состоит из двух строк:

   * Количество ошибок в данных, обнаруженных в процессе обработки видео. Ошибкой считается распознавание нового объекта и удаление ранее распознанного объекта.
   * Количество созданных исправлений.

* **Processed**: результат обработки ошибок и исправлений. Состоит из двух строк:

   * количество обработанных ошибок | количество актуальных (полезных) ошибок;
   * количество обработанных исправлений | количество полезных исправлений.

* **Efficiency**: отношение полезных ошибок и уточнений ко всем ошибкам и уточнениям.

* **KML**: ссылка для загрузки всех треков в виде файла формата `.kml`.

## Операции с данными {#operations}

### Скачивание данных {#downloading}

1. Если необходимо, настройте фильтры или используйте чекбоксы, чтобы выбрать отдельные элементы.
2. Прокрутите страницу вниз и нажмите **Download KML** или **Download CSV**.

### Удаление данных из <Brand /> Про {#removing}

По умолчанию в <Brand /> Про автоматически загружаются все данные. Чтобы скрыть в <Brand /> Про определённые элементы:

1. Если необходимо, настройте фильтры или используйте чекбоксы, чтобы выбрать отдельные элементы.
2. Прокрутите страницу вниз и нажмите **Hide from PRO**.

### Повторная отправка данных в <Brand /> Про {#reuploading}

Если данные были удалены из <Brand /> Про или были некорректно обработаны, вы можете отправить их заново:

1. Если необходимо, настройте фильтры или используйте чекбоксы, чтобы выбрать отдельные элементы.
2. Прокрутите страницу вниз и нажмите **Send to PRO**.
