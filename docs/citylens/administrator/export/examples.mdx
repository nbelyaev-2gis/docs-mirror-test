# Примеры

Ниже приведены основные сценарии работы и примеры запросов к Export API. В примерах запросов используется [cURL](https://curl.se/). Подробную информацию о параметрах запросов см. в [Справочнике API](/citylens/administrator/export/reference).

Для начала работы с Export API [получите API-ключ](/citylens/administrator/export/overview#get-access-key).

## Список кадров {#frames-list}

Вы можете получить список исходных кадров с данными о них: ID, координатами, временем съёмки, скоростью движения транспортного средства и другими.

Чтобы получить список кадров, отправьте GET-запрос на [/export-api/v1/frames](/citylens/administrator/export/reference) со следующими параметрами:

* `from_dt` (обязательный параметр) — начальная дата и время записи кадра в формате `YYYY-MM-DDTHH:MM:SS`.
* `to_dt` (обязательный параметр) — конечная дата и время записи кадра в формате `YYYY-MM-DDTHH:MM:SS`.
* `limit` — максимальное количество кадров в ответе.

**Пример запроса:**

```curl
curl --request GET \
  --url 'API_URL/export-api/v1/frames?from_dt=2025-09-11T11:03:00&to_dt=2025-09-11T11:05:00&limit=50' \
  --header 'X-API-Key: API_KEY'
```

## Список детекторов {#detector-list}

Детекторы — это категории объектов городской среды, которые можно распознать на кадрах (например, дорожные знаки, светофоры, вывески на зданиях и другие). Список доступных детекторов определяется при интеграции сервиса <Brand /> Ситискан.

Чтобы получить список возможных детекторов, отправьте GET-запрос на [/export-api/v1/detectors](/citylens/administrator/export/reference).

**Пример запроса:**

curl --request GET \
  --url 'API_URL/export-api/v1/detectors' \
  --header 'X-API-Key: API_KEY'

## Список распознанных объектов {#detected-objects-list}

Чтобы получить список объектов, распознанных на кадрах, отправьте GET-запрос на [/export-api/v1/detections/objects](/citylens/administrator/export/reference) со следующими параметрами:

* `from_dt` (обязательный параметр) — начальная дата и время записи кадра в формате `YYYY-MM-DDTHH:MM:SS`.
* `to_dt` (обязательный параметр) — конечная дата и время записи кадра в формате `YYYY-MM-DDTHH:MM:SS`.
* `detector_name` — название [детектора](#detector-list) (категории распознаваемых объектов) для фильтрации. Например, `signs` — дорожные знаки.
* `locale` — язык названий распознанных объектов в ответе.
* `limit` — максимальное количество объектов в ответе.

**Пример запроса:**

```curl
curl --request GET \
  --url 'API_URL/export-api/v1/detections/objects?from_dt=2025-09-01T00:00:00&to_dt=2025-12-30T00:00:00&detector_name=signs&locale=ru_RU&limit=100' \
  --header 'X-API-Key: API_KEY'
```
