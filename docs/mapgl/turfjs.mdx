# Работа с геометриями

Для манипуляций с GeoJSON рекомендуется использовать библиотеку [turf.js](https://turfjs.org/). Ниже описывается использование данной библиотеки в связке с MapGL JS API.

## Граничный прямоугольник (bbox) {#bbox}

Метод `turf.bbox` рассчитывает прямоугольную область, в которой находятся заданные объекты или группа объектов.

```js
const line = turf.lineString([
    [32.013931, 55.491648],
    [35.491901, 56.392901],
    [37.144102, 55.491648],
    [40.371536, 56.392901],
    [42.128462, 55.491648],
]);
const bbox = turf.bbox(line); // [minX, minY, maxX, maxY]
```

Документация на метод: [@turf/bbox](https://turfjs.org/docs/api/bbox)

**Примеры использования**

* Спозиционировать карту так, чтобы все объекты поместились на экран при помощи метода [map.fitBounds](/mapgl/reference/class#map/fit-bounds).

  ```js
  map.fitBounds({ southWest: bbox.slice(0, 2), northEast: bbox.slice(2, 4) });
  ```

* Запросить дополнительные данные к имеющимся. Например, имея полигоны муниципальных районов, запросить близлежащие объекты застройки.

Граничный прямоугольник является наиболее распространенным и быстрым способом определить границы нахождения объектов. Обратной стороной является то, что границы таким образом определяются достаточно грубо. Если нужна большая точность, воспользуйтесь [выпуклой оболочкой](#convex-hull).

## Выпуклая оболочка (convex hull) {#convex-hull}

Метод `turf.convex` принимает на вход объект или группу объектов и создаёт на их основе [выпуклую оболочку](https://ru.wikipedia.org/wiki/Выпуклая_оболочка).

```js
const line = turf.lineString([
    [32.013931, 55.491648],
    [35.491901, 56.392901],
    [37.144102, 55.491648],
    [40.371536, 56.392901],
    [42.128462, 55.491648],
]);
const convexHull = turf.convex(line);
```

Документация на метод: [@turf/convex](https://turfjs.org/docs/api/convex)

**Пример использования**

Выпуклая оболочка — более точный по сравнению с [граничным прямоугольником](#bbox) метод определения границ объекта. Обратной стороной является большая вычислительная сложность данного метода.

```js
    [40.371536, 56.392901],
    [42.128462, 55.491648],
```

## Получение центра (centroid) {#centroid}

Методы `turf.center`, `turf.centroid` и `turf.centerOfMass` позволяют получить тот или иной центр объекта или группы объектов.

```js
const polygon = turf.polygon([
    [
      [32.013931, 55.491648],
      [35.491901, 56.392901],
      [37.144102, 55.491648],
      [40.371536, 56.392901],
      [42.128462, 55.491648],
      [32.013931, 55.491648],
    ],
]);
const center = turf.center(polygon);
const centroid = turf.centroid(polygon);
const centerOfMass = turf.centerOfMass(polygon);
```

Документация на методы:

* [@turf/center](https://turfjs.org/docs/api/center),
* [@turf/centroid](https://turfjs.org/docs/api/centroid),
* [@turf/centerOfMass](https://turfjs.org/docs/api/centerOfMass).

**Примеры использования**

Есть 3 метода получения центра:

* `center` получает центр граничного прямоугольника объекта или группы.
* `centroid` вычисляет [геометрический центр](https://ru.wikipedia.org/wiki/Барицентр) объекта или группы.
* `centerOfMass` получает [центр масс](https://ru.wikipedia.org/wiki/Центр_масс) фигуры.

На правильных и симметричных полигонах между этими методами практически нет разницы. В случае невыпуклых полигонов или других полигонов неправильной формы лучше всего работает `centerOfMass`.

Эти методы удобны для расстановки подписей объектов.

## Ближайшая точка (nearestPoint) {#nearest-point}

Метод `turf.nearestPoint` получает ближайшую точку из множества точек.

```js
const line = turf.lineString([
    [32.013931, 55.491648],
    [35.491901, 56.392901],
    [37.144102, 55.491648],
    [40.371536, 56.392901],
    [42.128462, 55.491648],
]);
const target = turf.point([35.491901, 55.491648]);

const points = turf.explode(line);
const nearest = turf.nearestPoint(target, points);
```

Документация на метод: [@turf/nearestPoint](https://turfjs.org/docs/api/nearestPoint)

Особенностью метода является работа только с множеством точек. Чтобы преобразовать произвольный объект в множество точек, воспользуйтесь методом [@turf/explode](https://turfjs.org/docs/api/explode).

**Пример использования**

Найти ближайший к заданному объект.

## Точка на линии (nearestPointOnLine) {#nearest-point-on-line}

Метод `turf.nearestPointOnLine` получает ближайшую к заданной точку на линии или на группе линий.

```js
const line = turf.lineString([
    [32.013931, 55.491648],
    [35.491901, 56.392901],
    [37.144102, 55.491648],
    [40.371536, 56.392901],
    [42.128462, 55.491648],
]);
const target = turf.point([35.491901, 55.491648], { name: 'Target' });

const nearestOnLine = turf.nearestPointOnLine(line, target);
```

Документация на метод: [@turf/nearestPointOnLine](https://turfjs.org/docs/api/nearestPointOnLine)

**Пример использования**

Найти ближайшую точку на маршруте или дороге. Как и в случае с методом [turf.buffer](#buffer), это работает только в самых простых случаях, когда не требуется учитывать препятствия на местности. В остальных случаях воспользуйтесь [Routing API](/api/navigation/routing/start).

## Объединение (union) {#union}

Метод `turf.union` объединяет несколько фигур в одну.

```js
const points = [
    [38.771739, 55.491648],
    [37.144102, 56.392901],
];
const circleA = turf.buffer(turf.point([38.771739, 55.491648]), 150);
const circleB = turf.buffer(turf.point([37.144102, 56.392901]), 150);
const united = turf.union(circleA, circleB);
```

Документация на метод: [@turf/union](https://turfjs.org/docs/api/union)

**Примеры использования**

* Отобразить на карте область, которую составляют несколько объектов, как один объект.
* Собрать излишне фрагментированные объекты, что иногда случается при обработке геоданных.

## Пересечение (intersect) {#intersect}

Метод `turf.intersect` находит фигуру, образованную пересечением заданных фигур.

```js
const points = [
    [32.013931, 55.491648],
    [35.491901, 56.392901],
];
const circleA = turf.buffer(turf.point([38.771739, 55.491648]), 150);
```

## Упрощение (simplify) {#simplify}

Метод `turf.simplify` упрощает геометрию при помощи [алгоритма Рамера-Дугласа-Пекера](https://ru.wikipedia.org/wiki/Алгоритм_Рамера_—_Дугласа_—_Пекера).

```js
const gpsTrack = turf.lineString([
    [37.588144, 55.733842],
    [37.593771, 55.740372],
    [37.601547, 55.742088],
    [37.608223, 55.750391],
]);
const simplifiedTrack = turf.simplify(gpsTrack, { tolerance: 0.2 });
// { ... geometry: { ..., coordinates: [[37.588144, 55.733842], [37.601547, 55.742088], [37.608223, 55.750391]] } }
```

Степень упрощения геометрии определяется параметром `tolerance`, который обозначает максимальную ошибку, которую может допустить алгоритм при упрощении. Величина `tolerance` имеет ту же размерность, что и исходные данные. То есть, если геокоординаты указываются в градусах, то и `tolerance` будет в градусах.

Документация на метод: [@turf/simplify](https://turfjs.org/docs/api/simplify)

**Примеры использования**

* Получить менее сложные геометрии, которые будут отрисовываться быстрее.
* Отфильтровать данные, когда известно, что их точность не может превышать определённое значение.
* Отобразить точные данные на большом масштабе: упрощённые геометрии имеют более эстетичный внешний вид и плавные линии без визуального шума.
