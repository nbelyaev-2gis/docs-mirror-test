# Иммерсивные дороги

Иммерсивная дорожная инфраструктура отображает дороги как реалистичные трёхмерные объекты, а не как схематичные линии. Это повышает детализацию карты и создаёт эффект присутствия, особенно при использовании совместно с другими иммерсивными возможностями: [3D-рельефом](/mapgl/map-style/immersive/terrain), [освещением](/mapgl/map-style/immersive/lighting) и [3D-моделями](/mapgl/map-style/immersive/models).

## Компоненты инфраструктуры {#components}

К объектам иммерсивной дорожной инфраструктуры относятся:

## Включение иммерсивных дорог {#enable-immersive-roads}

:::note[Требования к устройствам]

Иммерсивные данные требуют больше ресурсов устройств (памяти, CPU и трафика) и могут снижать их производительность.

:::

Чтобы отобразить на карте реалистичную дорожную инфраструктуру, используйте подготовленный стиль и включите иммерсивный режим через API.

Чтобы включить режим, установите для глобальной переменной `immersiveRoadsOn` значение `true` с помощью метода [patchStyleState()](/mapgl/reference/class#map/patch-style-state). После этого карта начнёт запрашивать тайлы с иммерсивными данными. Подробнее о глобальных переменных см. в разделе [Глобальные переменные стиля карты](/mapgl/map-style/global-style-variables).

```javascript
map.patchStyleState({ immersiveRoadsOn: true });
```

Чтобы отключить режим, установите для глобальной переменной `immersiveRoadsOn` значение `false`. После этого тайлы с иммерсивными данными не будут запрашиваться.

```javascript
map.patchStyleState({ immersiveRoadsOn: false });
```

## Настройка стилевых слоёв {#configuring-layers}

Отображаемые объекты дорожной инфраструктуры можно настраивать в [Редакторе стилей](https://styles.2gis.com/). Чтобы включить режим иммерсивности в Редакторе стилей, в разделе **Настройки** включите **Иммерсивные дороги**:

<img src="/img/mapgl/road-infrastructure/immersive-roads.png" alt="Включение режима в Редакторе стилей" width="650" />

Подробнее о слоях для MapGL JS API см. в [спецификации стиля](/mapgl/styles-specification) и в [документации Редактора стилей](/maps/styles/layers/configured/mapgl). О создании стиля на основе базового см. в разделе [Создание стилей и настройка стилевых слоёв](/maps/styles/create).

Среди общих настроек важен режим отображения неиммерсивных слоёв в зависимости от наличия у объекта атрибута `db_has_immersive_counterpart`. Этот атрибут указывает на наличие иммерсивного двойника — альтернативного иммерсивного представления того же объекта дорожной инфраструктуры вместо обычного объекта плоского слоя.

Например, слой **Bike road** в базовом стиле использует следующее значение параметра **Настройки отображения слоя** на вкладке **Отображение**:

```json
[
    "match",
    ["get", "db_has_immersive_counterpart"],
    [1],
    false,
    true
]
```

Это означает:

* если у объекта есть иммерсивный двойник, неиммерсивный слой не отрисовывается;
* если иммерсивного двойника нет, объект отрисовывается плоско, с игнорированием уровней.

### Модели опор {#pillars}

В иммерсивном режиме опоры мостов и дорожных полотен представлены трёхмерными моделями в формате [glTF/GLB](https://www.khronos.org/gltf/).

<img src="/img/mapgl/road-infrastructure/bridge_pillar_layer.png" alt="Слой модели опоры" width="800" />

Параметр **Размер** (Scale) опор не является константой и задаётся выражением:

```json
[
    "literal",
    [
        1,
        1,
        ["+",
            ["*",
                ["get", "db_nominal_height"],
                ["*", ["get", "db_level"], 5]
            ],
            -0.3
        ]
    ]
]
```

Такое выражение согласует модель опоры с геометрией развязки: по вертикальной оси используется номинальная высота, умножается на 5 (высота пролёта для России) и на текущий уровень, затем вычитается 0.3, чтобы опора визуально не пересекалась с развязкой.

Более подробную информацию об использовании 3D-моделей на карте см. в разделе [3D-модели](/mapgl/map-style/immersive/models).

### Точечная разметка {#point-marking}

Точечная разметка отображается с помощью слоя [Плоское изображение](/mapgl/styles-specification#style-for-metricpoint). Этот тип слоя подходит для стрелок направления, пешеходных переходов и других точечных элементов дорожной разметки.

Для настройки точечной разметки в Редакторе стилей можно задать:

* иконку,
* цвет иконки и её прозрачность,
* угол поворота иконки,
* размер иконки.

<img src="/img/mapgl/road-infrastructure/metricpoint_example.png" alt="Точечная разметка" width="800" />

Угол поворота задаётся `get`-выражением, поэтому для каждого объекта он может быть своим.

### Линейная разметка {#linear-marking}

Линейная разметка отрисовывается слоем [Линия](/mapgl/styles-specification#style-for-line). Этот тип слоя используется для разделительных полос, краевых линий и других линейных элементов дорожной разметки.

Для настройки линейной разметки в Редакторе стилей можно задать:

* цвет линии и прозрачность,
* ширину линии,
* тип паттерна.

<img src="/img/mapgl/road-infrastructure/line_example.png" alt="Линейная разметка" width="800" />

### Развязки и туннели {#tunnels}

Развязки и туннели отрисовываются на карте с помощью слоя **Транспортная развязка**. Этот слой можно редактировать при работе с готовым шаблоном в Редакторе стилей.

У развязок и туннелей общие параметры, кроме параметров:

* **Толщина** — используется только для развязки;
* **Высота туннеля** — используется только для туннеля.
