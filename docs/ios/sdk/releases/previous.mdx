# Релизы

:::warning

Прекращена поддержка версий SDK, описанных ниже. Используйте [версию 12.0.0 и выше](/ios/sdk/releases/latest). Cм. инструкции по [миграции с предыдущих версий](/ios/sdk/releases/migration).

:::

## 10.8.0 (2024-11-02)

### 🛠 Исправления

* Исправлено потенциальное зависание в [MapObjectManager](/ios/sdk/reference/10.8/class#map-object-manager) при частом добавлении и удалении объектов через метод [MapObjectManager.removeAndAddObjects](/ios/sdk/reference/10.8/class#map-object-manager/remove-and-add-objects).
* Добавлена дополнительная проверка на корректность обработки жеста наклона карты, чтобы это не приводило к падению приложения.
* Исправлена ошибка, при которой приложение падало при логировании сетевых запросов.
* Исправлена ошибка, при которой [MapViewOverlay](/ios/sdk/reference/10.8/struct#map-view-overlay) работал с прерыванием при взаимодействии с картой.
* Исправлена работа SwiftUI-элементов для карты, которые пересоздавались при каждом обновлении интерфейса.
* Исправлена ошибка, при которой для некоторых регионов не работало отображение дорожных событий на карте.

## 10.7.0 (2024-08-13)

### 🌟 Новые возможности

* [DashboardButtonSettings](/ios/sdk/reference/10.7/struct#dashboard-button-settings) в [NavigationViewOptions](/ios/sdk/reference/10.7/struct#navigation-view-options) для настройки кнопки добавления дорожных событий в дашборде навигатора.

## 10.6.0 (2024-08-07)

### 🌟 Новые возможности

* [PackageInfo.finalSizeOnDisk](/ios/sdk/reference/10.6/struct#package-info/var-final-size-on-disk) — размер места на диске, которое территория занимает после установки, в байтах.
* [PackageInfo.currentSizeOnDisk](/ios/sdk/reference/10.6/struct#package-info/var-current-size-on-disk) — размер места на диске, которое занимают установленные файлы территории на данный момент, в байтах.
* В [NavigationViewTheme](/ios/sdk/reference/10.6/struct#navigation-view-theme) добавлены параметры для настройки UI-элементов [INavigationView](/ios/sdk/reference/10.6/INavigationView). Вы можете настроить цвет иконок основного и дополнительного манёвров, шрифты во всех UI-элементах, кастомизировать UI-элементы компаса и уровня пробок в баллах.

### 🛠 Исправления

* Исправлена работа карты с включенной настройкой `Metal API Validation` в Xcode.
* Исправлено обновление позиции в дереве кластеризации через [MapObjectManager.withClustering](/ios/sdk/reference/10.6/class#map-object-manager) при изменении позиции маркера.
* Обновлён дизайн копирайта снимка карты, полученного через [IMapSnapshotter](/ios/sdk/reference/10.6/IMapSnapshotter).
* Исправлена ошибка, при которой [MapViewOverlay](/ios/sdk/reference/10.6/struct#map-view-overlay) не обновлялся при изменении параметров.
* В [INavigationView](/ios/sdk/reference/10.6/INavigationView) UI-элемент спидометра теперь отображается при навигации по велосипедным маршрутам.
* В [INavigationView](/ios/sdk/reference/10.6/INavigationView) UI-элемент слежения за позицией теперь не скрывается при пешей навигации.

## 10.5.0 (2024-06-05)

### 🌟 Новые возможности

* В настройках ключа можно задавать каскадный `appId`. Благодаря этой настройке ключ можно привязать не к конкретному приложению, а к группе приложений с одинаковой шаблонной частью в идентификаторе.
* Обновлён дизайн копирайта.

### 🛠 Исправления

* Исправлена работа SDK для CarPlay.
* При отображении маркеров с помощью [Marker](/ios/sdk/reference/10.5/class#marker) текстовые подписи не перекрывают друг друга.
* Исправлено отображение маркера геопозиции, когда перестают поступать данные от датчика направления.

## 10.4.0 (2024-05-20)

### 🌟 Новые возможности

* Добавлены SwiftUI-элементы для карты: [CurrentLocationView](/ios/sdk/reference/10.4/struct#current-location-view), [ZoomView](/ios/sdk/reference/10.4/struct#zoom-view), [TrafficView](/ios/sdk/reference/10.4/struct#traffic-view), [IndoorView](/ios/sdk/reference/10.4/struct#indoor-view), [CompassView](/ios/sdk/reference/10.4/struct#compass-view). Все элементы могут быть созданы через фабрику [IMapControlViewFactory](/ios/sdk/reference/10.4/IMapControlViewFactory).
* [ZoomFollowSettings.speedRangeToStyleZoomFreeRoamSequence](/ios/sdk/reference/10.4/class#zoom-follow-settings) стал сеттером.

### 🛠 Исправления

* Исправлена ошибка, при которой изменение этажа через [IndoorBuilding.activeLevelIndex](/ios/sdk/reference/10.4/class#indoor-building) не влияло на отображение [IndoorControl](/ios/sdk/reference/10.4/class#indoor-control).
* Обновлён файл `PrivacyInfo.xcprivacy`.

### ❗ Изменения, влияющие на обратную совместимость

* В объект карты [Map](/ios/sdk/reference/10.4/class#map) теперь можно добавить только один источник текущего местоположения [MyLocationMapObjectSource](/ios/sdk/reference/10.4/class#my-location-map-object-source).
* SDK поддерживает только iOS 13 и выше.

## 10.3.0 (2024-05-06)

### 🌟 Новые возможности

* Теперь информация о пробках в регионах обновляется независимо от обновления конфигурационного файла SDK.
* Теперь при zoom-уровне выше 2 карту нельзя сдвинуть для отображения пустых полос сверху и снизу.
* Добавлен арабский голосовой пакет для навигатора.

### 🛠 Исправления

* Исправлена работа кластеризации через [MapObjectManager.withClustering](/ios/sdk/reference/10.3/class#map-object-manager). Теперь иерархия кластеров перестраивается, если позиция одного из маркеров изменилась через [Marker.position](/ios/sdk/reference/10.3/class#marker).
* Исправлена некорректная обработка тапа в [Marker](/ios/sdk/reference/10.3/class#marker) с Lottie-анимацией.
* [Map.dataLoadingStateChannel](/ios/sdk/reference/10.3/class#map) больше не зависает в состоянии `loading` при режиме слежения за геопозицией.
* Добавлен файл `PrivacyInfo.xcprivacy`.

### ❗ Изменения, влияющие на обратную совместимость

* Заканчивается поддержка iOS 12 версии. Со следующей версии SDK работает только для iOS 13 и выше.

## 10.2.0 (2024-03-07)

### 🌟 Новые возможности

* Опции [CircleOptions.dashedStrokeOptions](/ios/sdk/reference/10.2/struct#circle-options) и [Circle.dashedStrokeOptions](/ios/sdk/reference/10.2/class#circle) для создания пунктирной окружности или добавления пунктира для уже созданной окружности.
* Метод [RoadEventSource.setHighlighted](/ios/sdk/reference/10.2/class#road-event-source) для выделения дорожных событий с идентификаторами из [RoadEventMapObject.id](/ios/sdk/reference/10.2/class#road-event-map-object).
* Функция [calculateClosestPoint](/ios/sdk/reference/10.2/method#calculate-closest-point-route-geometry-point--) для получения ближайшей точки на маршруте относительно переданной точки.

### 🛠 Исправления

* Исправлена ошибка в работе `layedId`, который задается в [MapObjectManager](/ios/sdk/reference/10.2/class#map-object-manager): теперь он влияет на отображение кластеризованных маркеров.
* Исправлен отрисовка карты: раньше карта зависала при отображении модального окна над `MapView`.
* Исправлена работа карты при сворачивании приложения.

### 🗺 Изменения в стилях карты

* Для корректной работы пунктирных окружностей нужно заново выгрузить стили.

## 10.1.0 (2024-01-16)

### 🌟 Новые возможности

* Добавлена структура [DirectoryObject.buildingInfo](/ios/sdk/reference/10.1/class#directory-object) со справочной информацией о здании: `buildingName`, `purposeName`, `purposeCode` и `buildingLevels`. [DirectoryObject.buildingLevels](/ios/sdk/reference/10.1/class#directory-object) помечено `deprecated` и будет удалено в будущих мажорных релизах.
* Добавлен [DirectoryObject.address.fiasCode](/ios/sdk/reference/10.1/class#directory-object) — ФИАС-код объекта справочника.
* В [DirectoryObject.address.components](/ios/sdk/reference/10.1/class#directory-object) в структурах [AddressComponent.streetAddress](/ios/sdk/reference/10.1/enumeration#address-component) и [AddressComponent.number](/ios/sdk/reference/10.1/enumeration#address-component) добавлено поле `fiasCode` — ФИАС-код улицы.

## 10.0.0 (2023-12-29)

:::note[Обновление с версии 7.x]

Перед обновлением см. [инструкцию по миграции](/ios/sdk/releases/migration#from-7-to-10).

:::

### 🌟 Новые возможности

* Иммерсивные деревья на карте отображаются по умолчанию.
* Поддержана новая модель для отображения маркера текущего местоположения. Если необходимо переопределить иконку текущего местоположения, нужно передать [MyLocationMapObjectMarkerType.svgIcon](/ios/sdk/reference/10.0/enumeration#my-location-map-object-marker-type) в конструктор [MyLocationMapObjectSource](/ios/sdk/reference/10.0/class#my-location-map-object-source), а в стилях отредактировать соответствующий слой.
* Добавлены [MarkerOptions.iconAnimationMode](/ios/sdk/reference/10.0/struct#marker-options) и [Marker.iconAnimationMode](/ios/sdk/reference/10.0/class#marker) для возможности задания режима анимации в маркерах. Режим может быть обычным или зацикленным.
* Добавлены [SearchResult.actionWidgets](/ios/sdk/reference/10.0/class#search-result) и [SearchResult.mainWidgets](/ios/sdk/reference/10.0/class#search-result) для динамической фильтрации уже полученной поисковой выдачи. [Widget](/ios/sdk/reference/10.0/class#widget) — базовый класс, представляющий виджет для фильтрации поисковой выдачи.
* В [LocaleManager](/ios/sdk/reference/10.0/class#locale-manager) добавлены методы `locales` и `systemLocales` и соответствующие каналы для получения всех локалей приложения и системных локалей.
* Добавлен метод [BaseCamera.changePosition](/ios/sdk/reference/10.0/class#base-camera) для изменения только части параметров позиции камеры. Метод принимает новую структуру [CameraPositionChange](/ios/sdk/reference/10.0/struct#camera-position-change).
* Добавлен метод [Camera.move](/ios/sdk/reference/10.0/class#camera) для запуска анимированного перемещения карты с возможностью сброса слежения за наклоном.
* Добавлен параметр `styleZoomToTiltRelation` для функций [calcPosition](/ios/sdk/reference/10.0/method#calc-position-camera-geometry-style-zoom-to-tilt-relation-screen-area-tilt-bearing-size--) и [zoomOutToFit](/ios/sdk/reference/10.0/method#zoom-out-to-fit-camera-geometry-style-zoom-to-tilt-relation-size--) — функция зависимости угла наклона камеры от стилевого zoom-уровня.
* Добавлена новая функция [createDefaultStyleZoomToTiltRelation](/ios/sdk/reference/10.0/method#create-default-style-zoom-to-tilt-relation--) для получения зависимости угла наклона камеры от стилевого zoom-уровня для стандартного контроллера слежения за наклоном.
* Добавлена функция [createDefaultMaxTiltRestriction](/ios/sdk/reference/10.0/method#create-default-max-tilt-restriction--) для получения дефолтной зависимости ограничения угла наклона от масштаба. По умолчанию эта зависимость устанавливается при создании карты — [BaseCamera.maxTiltRestriction](/ios/sdk/reference/10.0/class#base-camera).
* Добавлен enum [BearingSource](/ios/sdk/reference/10.0/enumeration#bearing-source) — источник направления.
* Добавлено поле [PackedMapState.cameraBehaviour](/ios/sdk/reference/10.0/class#packed-map-state) для сериализации режима слежения камеры.
* В [PackageInfo](/ios/sdk/reference/10.0/struct#package-info) добавлено поле `error` — ошибка, произошедшая при попытке загрузки и/или установки пакета.
* Добавлен метод [IndoorManager.setIndoorState](/ios/sdk/reference/10.0/class#indoor-manager) для управления отображением этажных планов на карте.
* Добавлено поле [NavigationManager.alternativeRouteSelector](/ios/sdk/reference/10.0/class#navigation-manager) для выбора альтернативного маршрута.
* Добавлено поле [Route.bicycleLanes](/ios/sdk/reference/10.0/class#route) — признак наличия выделенной велополосы.
* Добавлено значение [RoutePointKind.bicycleLane](/ios/sdk/reference/10.0/enumeration#route-point-kind) — начало и конец велополосы.
* Добавлен флаг [PedestrianRouteSearchOptions.avoidUnderpassesAndOverpasses](/ios/sdk/reference/10.0/struct#pedestrian-route-search-options) — избегать подземных и надземных переходов.
* Добавлен метод [RouteMapObjectSource.replaceObject](/ios/sdk/reference/10.0/class#route-map-object-source) для замены одного объекта маршрута в источнике на другой.
* Добавлено поле [NavigationFollowController.mapLocationController](/ios/sdk/reference/10.0/class#navigation-follow-controller). [MapLocationController](/ios/sdk/reference/10.0/class#map-location-controller) — контроллер для управления позиционированием карты в навигаторе.
* Добавлено поле [NavigationFollowController.cameraBehaviour](/ios/sdk/reference/10.0/class#navigation-follow-controller) для возможности менять режим слежения карты за маркером геопозиции.
* Добавлено поле [ZoomFollowSettings.speedRangeToStyleZoomFreeRoamSequence](/ios/sdk/reference/10.0/class#zoom-follow-settings) — последовательность интервалов скоростей и соответствующих им масштабов при ведении в режиме FreeRoam.

### ❗ Изменения, влияющие на обратную совместимость

* Изменена работа с [FollowController](/ios/sdk/reference/10.0/class#follow-controller):

  * Удалены методы `createRawMyLocationController()` и `createSmoothMyLocationController()`. Добавлен конструктор [MyLocationController](/ios/sdk/reference/10.0/class#my-location-controller) для создания контроллера перемещения маркера геопозиции.
  * Удалены значения `FollowBearing.satellite` и `FollowBearing.magnetic`, т. к. теперь не нужно указывать источник направления. Добавлено значение [FollowBearing.on](/ios/sdk/reference/10.0/enumeration#follow-bearing) для включения слежения — источник выбирается автоматически.
  * Удалено поле [FollowPosition.tilt](/ios/sdk/reference/10.0/struct#follow-position). Добавлен новый конструктор [CameraBehaviour](/ios/sdk/reference/10.0/struct#camera-behaviour) с параметром `tilt` для слежения только за наклоном.
  * Удалены значения `FollowValue.satelliteBearing`, `FollowValue.magneticBearing` и `FollowValue.accuracy`. Вместо них добавлено значение [FollowValue.bearing](/ios/sdk/reference/10.0/enumeration#follow-value) с автоматическим выбором источника.
  * Удалены методы `satelliteBearing()`, `magneticBearing()` и `accuracy()` из интерфейса [CustomFollowController](/ios/sdk/reference/10.0/interface#custom-follow-controller). Вместо них добавлен метод [CustomFollowController.bearing()](/ios/sdk/reference/10.0/interface#custom-follow-controller).
  * Удалены методы `setClock()`, `satelliteBearing()`, `magneticBearing()` и `accuracy()` из [FollowController](/ios/sdk/reference/10.0/class#follow-controller). Вместо них добавлены методы `onAddedToMap()`, `onRemovedFromMap()` и `bearing()`.
  * Удалён канал `FollowController.nextTimestampChannel`. Вместо него добавлен [FollowController.changedChannel](/ios/sdk/reference/10.0/class#follow-controller).
  * Изменена сигнатура метода [FollowController.updateValues()](/ios/sdk/reference/10.0/class#follow-controller).
  * Удалён контроллер `PlatformLocationFollowController`. Вместо него добавлены контроллеры [BearingFollowController](/ios/sdk/reference/10.0/class#bearing-follow-controller) и [CoordinatesFollowController](/ios/sdk/reference/10.0/class#coordinates-follow-controller) для слежения за направлением карты и за координатами позиции карты, соответственно.
  * Изменён конструктор контроллера [StyleZoomFollowController](/ios/sdk/reference/10.0/class#style-zoom-follow-controller). Удалён метод `StyleZoomFollowController.setAnimationDuration()`.
  * Изменён конструктор контроллера [TiltFollowController](/ios/sdk/reference/10.0/class#tilt-follow-controller). Удалены методы `TiltFollowController.setTilt()` и `TiltFollowController.setAnimationDuration()`.
  * Изменены параметры метода [FollowController.setThresholds()](/ios/sdk/reference/10.0/class#follow-controller) на типизированные.
  * Удалён класс `FollowControllerClock`. Тип параметра в методе [FollowController.setClock](/ios/sdk/reference/10.0/class#follow-controller) заменён на timestamp.
  * Удалён `TransitionType`. Конструктор [MyLocationControlModel](/ios/sdk/reference/10.0/class#my-location-control-model) теперь не принимает `transitionType`. Удалён параметр `transitionType` из конструктора [CameraBehaviour](/ios/sdk/reference/10.0/struct#camera-behaviour).
  * Удалён `MyLocationDirectionBehaviour`. Конструктор источника геопозиции [MyLocationMapObjectSource](/ios/sdk/reference/10.0/class#my-location-map-object-source) теперь не принимает `directionBehaviour`. Удалён метод `MyLocationMapObjectSource.setDirectionBehaviour()`.

* Удалено поле `BaseCamera.styleZoomToTiltRelation`.

* Переименован класс `MagneticHeadingProvider` в `HeadingProvider`.

* Конструкторы динамических объектов [Circle](/ios/sdk/reference/10.0/class#circle), [Marker](/ios/sdk/reference/10.0/class#marker), [Polygon](/ios/sdk/reference/10.0/class#polygon) и [Polyline](/ios/sdk/reference/10.0/class#polyline) помечены как `throws`.

* Удалена опция [MapOptions.mapRendererType](/ios/sdk/reference/10.0/struct#map-options) и enum `MapRendererType`. Теперь всегда включён `Metal` для рендеринга карты.

* Удалены `Circle.position` и `Circle.radius`. Вместо них добавлены методы [Circle.setPosition()](/ios/sdk/reference/10.0/class#circle) и [Circle.setRadius()](/ios/sdk/reference/10.0/class#circle), которые могут генерировать исключение.

* Структура [Locale](/ios/sdk/reference/10.0/struct#locale) теперь создаётся на основе отдельных значений языка `language` и региона `region`.

* Из Map-сборки удалены [PackageManager](/ios/sdk/reference/10.0/class#package-manager) и [Package](/ios/sdk/reference/10.0/class#package), что уменьшило размер продукта.

* Метод [Voice.playWelcome()](/ios/sdk/reference/10.0/class#voice) теперь возвращает [Future](/ios/sdk/reference/10.0/class#future), который становится готовым, когда проигрывание сэмпла завершается.

* Удалено поле `RoadSubtype.stairway`. Вместо него добавлены поля [RoadSubtype.stairwayUp](/ios/sdk/reference/10.0/enumeration#road-subtype) и [RoadSubtype.stairwayDown](/ios/sdk/reference/10.0/enumeration#road-subtype) для использования лестниц с явным направлением.

* Удалено поле `NavigationFollowController.mapTiltController`.

* При запуске навигатора в режиме ведения или симуляции с маршрутом на общественном транспорте теперь генерируется исключение.

* Отключено кеширование тайлов `FreeRoam` при навигации.

* [MyLocationMapObjectSource](/ios/sdk/reference/10.0/class#my-location-map-object-source) для навигатора теперь устанавливается внутри [INavigationView](/ios/sdk/reference/10.0/INavigationView).

### ❗ Известные ошибки

* При частом обновлении динамических объектов (около 100 FPS) зависает карта.

### 🛠 Исправления

* Оптимизирована перерисовка кадров карты при любых изменениях.

* Исправлено отображение пробок на карте в зависимости от zoom-уровня.

* Исправлено неплавное обновление маркера геопозиции.

* Оптимизировано и исправлено отображение карты на последних версиях iOS.

* Исправлена работа метода [SimpleClusterObject.setIcon](/ios/sdk/reference/10.0/class#simple-cluster-object) при первом вызове.

* Исправлена работа кластеризации через [MapObjectManager.withClustering](/ios/sdk/reference/10.0/class#map-object-manager), если передать дробные значения `minZoom` и `maxZoom`.

* Обновлен логотип <Brand /> в копирайте на снапшотах карты.
