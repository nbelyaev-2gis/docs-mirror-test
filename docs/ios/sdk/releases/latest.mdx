# Релизы

## 13.6.0 (2026-06-22)

### 🌟 Новые возможности

* Новый класс [MapController](/ios/sdk/reference/stable/class#map-controller):

  * [MapController](/ios/sdk/reference/stable/class#map-controller) – класс для управления картой, объединяющий [Map](/ios/sdk/reference/stable/class#map) и [MapGestureRecognizer](/ios/sdk/reference/stable/class#map-gesture-recognizer).
  * [MapControllerOptions](/ios/sdk/reference/stable/struct#map-controller-options) – структура с параметрами создания [MapController](/ios/sdk/reference/stable/class#map-controller).
  * [MapController.create](/ios/sdk/reference/stable/class#map-controller/create) – статический метод для асинхронного создания [MapController](/ios/sdk/reference/stable/class#map-controller) на основе [Context](/ios/sdk/reference/stable/class#context) и [MapControllerOptions](/ios/sdk/reference/stable/struct#map-controller-options).
  * [MapController.map](/ios/sdk/reference/stable/class#map-controller/map) – свойство для получения объекта [Map](/ios/sdk/reference/stable/class#map) из [MapController](/ios/sdk/reference/stable/class#map-controller).
  * [MapController.gestureRecognizer](/ios/sdk/reference/stable/class#map-controller/gesture-recognizer) – свойство для получения объекта [MapGestureRecognizer](/ios/sdk/reference/stable/class#map-gesture-recognizer) из [MapController](/ios/sdk/reference/stable/class#map-controller).

* [CameraZoomRestrictions.minZoom](/ios/sdk/reference/stable/struct#camera-zoom-restrictions/min-zoom) – значение по умолчанию изменено с `0` на `2`.

### 🛠 Исправления

* Механизм восстановления автонаклона доработан так, чтобы он работал независимо от других перелётов.
* Исправлен [WorkStatus](/ios/sdk/reference/stable/struct#work-status) для расписаний, работающих почти круглосуточно.
* Исправлена ошибка, вызывавшая сбой приложения при увеличении масштаба карты на Samsung Tab A8 с Android 14.
* Исправлена ошибка, из-за которой не работали повторные добавления и удаления дорожных событий через [RoadEventManager](/ios/sdk/reference/stable/class#road-event-manager).

## 13.5.0 (2026-05-19)

### 🌟 Новые возможности

* [SearchQueryBuilder()](/ios/sdk/reference/stable/class#search-query-builder/init) – конструктор по умолчанию, который позволяет устанавливать различные параметры поиска: текст, идентификаторы рубрик, зданий или организаций.
* [SearchQueryBuilder.setQueryText()](/ios/sdk/reference/stable/class#search-query-builder/set-query-text) — метод для задания текста поискового запроса.
* [SearchQueryBuilder.setRubricIds()](/ios/sdk/reference/stable/class#search-query-builder/set-rubric-ids) – метод для задания идентификаторов рубрик в поисковом запросе.
* [SearchQueryBuilder.setOrgId()](/ios/sdk/reference/stable/class#search-query-builder/set-org-id) – метод для задания фильтра по идентификатору организации.
* [SearchQueryBuilder.setBuildingId()](/ios/sdk/reference/stable/class#search-query-builder/set-building-id) – метод для задания фильтра по идентификатору здания.
* [SearchQueryBuilder.setRestrictionGeometry()](/ios/sdk/reference/stable/class#search-query-builder/set-restriction-geometry) – метод для задания строгого ограничения области поиска в виде геометрии.
* [SearchQueryBuilder.setTerritoryOfInterest()](/ios/sdk/reference/stable/class#search-query-builder/set-territory-of-interest) – метод для задания нестрогого ограничения области поиска: если объект не попадает в область, он не отбрасывается, но его ранг в результатах поиска понижается.
* [SuggestQueryBuilder.setRestrictionGeometry()](/ios/sdk/reference/stable/class#suggest-query-builder/set-restriction-geometry) – метод для задания строгого ограничения области для поисковых подсказок в виде геометрии.
* [SuggestQueryBuilder.setTerritoryOfInterest()](/ios/sdk/reference/stable/class#suggest-query-builder/set-territory-of-interest) – метод для задания нестрогого ограничения области для поисковых подсказок: если объект не попадает в область, он не отбрасывается, но его ранг в результатах поиска понижается.
* [PackedSearchQuery.geometryRestriction](/ios/sdk/reference/stable/class#packed-search-query/var-geometry-restriction) – свойство для получения геометрии, ограничивающей область поиска.
* [PackedSearchQuery.territoryOfInterest](/ios/sdk/reference/stable/class#packed-search-query/var-territory-of-interest) – свойство для нестрогого ограничения области поиска объектов: если объект не попадает в область, он не отбрасывается, но его ранг в результатах поиска понижается.
* [PolygonGeometry.elevation](/ios/sdk/reference/stable/class#polygon-geometry/var-elevation) – свойство для задания высоты полигона над поверхностью.
* [PolylineGeometry.elevation](/ios/sdk/reference/stable/class#polyline-geometry/var-elevation) – свойство для задания высоты полилинии над поверхностью.
* [Polyline.elevation](/ios/sdk/reference/stable/class#polyline/var-elevation) – свойство для задания высоты полилинии над поверхностью.
* [PolylineOptions.elevation](/ios/sdk/reference/stable/struct#polyline-options/var-elevation) – свойство для задания высоты полилинии над поверхностью в параметрах полилинии.
* [Polygon.elevation](/ios/sdk/reference/stable/class#polygon/var-elevation) – свойство для задания высоты полигона над поверхностью.
* [PolygonOptions.elevation](/ios/sdk/reference/stable/struct#polygon-options/var-elevation) – свойство для задания высоты полигона над поверхностью в параметрах полигона.
* [DashedPolygonOptions](/ios/sdk/reference/stable/struct#dashed-polygon-options) – класс для параметров пунктирного контура полигона.
* [Polygon.dashedPolygonOptions](/ios/sdk/reference/stable/class#polygon/var-dashed-polygon-options) – свойство для параметров пунктирного контура полигона.
* [PolygonOptions.dashedPolygonOptions](/ios/sdk/reference/stable/struct#polygon-options/var-dashed-polygon-options) – свойство для настройки пунктирного контура полигона в параметрах полигона.
* В SwiftUI-элемент [NavigationView](/ios/sdk/reference/stable/struct#navigation-view) добавлена возможность задавать выбранное на карте дорожное событие и отображать его через [RoadEventInfoView](/ios/sdk/reference/stable/struct#road-event-info-view).
* [RouteEnclosingRectsOptions](/ios/sdk/reference/stable/struct#route-enclosing-rects-options) – класс для параметров получения прямоугольников, описанных вокруг упрощённой геометрии маршрута.
* [getRouteEnclosingRects()](/ios/sdk/reference/stable/function#get-route-enclosing-rects) – метод для получения набора прямоугольников вокруг упрощённой геометрии маршрута.
* [SharedRouteData](/ios/sdk/reference/stable/struct#shared-route-data) – класс с информацией о заранее построенном маршруте и параметрах его поиска.
* [TrafficRouter.fetchSharedRoute()](/ios/sdk/reference/stable/class#traffic-router/fetch-shared-route) – метод для получения заранее построенного маршрута и параметров его поиска по идентификатору. Подробнее см. в примере [Готовый маршрут](/ios/sdk/examples/routing#using-ready-route).

## 13.4.0 (2026-04-22)

### 🌟 Новые возможности

* [IModernMapViewsFactory](/ios/sdk/reference/stable/interface#imodern-map-views-factory) — фабрика SwiftUI-элементов карты с обновлённым дизайном.
* [IModernMapUIControlsFactory](/ios/sdk/reference/stable/interface#imodern-map-uicontrols-factory) — фабрика UIKit-элементов карты с обновлённым дизайном.

* Свойство [IndoorControlModel.levelNames](/ios/sdk/reference/stable/class#indoor-control-model/var-level-names) теперь остаётся пустым, если на карте не отображается здание с этажными планами, даже если у здания один этаж.
* [LocationQuality](/ios/sdk/reference/stable/enumeration#location-quality) — качество определения текущего местоположения.
* [MyLocationControlModel.locationQuality](/ios/sdk/reference/stable/class#my-location-control-model/var-location-quality) — свойство для получения качества определения текущего местоположения.
* Увеличено максимальное значение угла наклона камеры [CameraPosition.tilt](/ios/sdk/reference/stable/class#camera-position/var-tilt) до 70.0.

### 🛠 Исправления

* Добавлена поддержка моделей iPhone 17e и iPad Air M4 с учётом их PPI при вычислении DPI.
* Исправлено отображение UI-элемента полной карточки объекта справочника [DirectoryObjectView](/ios/sdk/reference/stable/struct#directory-object-view) в альбомной ориентации.
* Исправлена ошибка, при которой свойство [LocationService.lastLocation](/ios/sdk/reference/stable/class#location-service/var-last-location) оставалось пустым, если создать только объект [LocationService](/ios/sdk/reference/stable/class#location-service).
* Скорректированы значения свойств [WorkStatus.openStatusHint](/ios/sdk/reference/stable/struct#work-status/var-open-status-hint) и [WorkStatus.scheduleHint](/ios/sdk/reference/stable/struct#work-status/var-schedule-hint) для заведений, работающих ночью.
* Исправлено исчезновение тайлов пробок, полученных через источник [TrafficSource](/ios/sdk/reference/stable/class#traffic-source), при изменении масштаба карты.
* Исправлены сбои в работе [IndoorManager](/ios/sdk/reference/stable/class#indoor-manager).
* Улучшено скачивание голосовых пакетов и пакетов территорий через [Package.install()](/ios/sdk/reference/stable/class#package/install).

## 13.3.1 (2026-03-20)

### 🌟 Новые возможности

* Теперь вы можете создать и скачать файл ключа в Менеджере Платформы: см. [Получение ключей доступа](/ios/sdk/start#get-api-key).
* В манифесте ключа, передаваемого в [Container](/ios/sdk/reference/stable/class#container), теперь необязательно наличие заполненного `restrictions/app_id`, если не планируется работа с офлайн-данными (`offlineRouting`, `offlineMap`, `offlineDirectory`). В противном случае наличие `restrictions/app_id` обязательно.
* При передаче пустого идентификатора этажа в параметр [IndoorControlModel.activeLevelIndex](/ios/sdk/reference/stable/class#indoor-control-model/var-active-level-index) этажный план здания будет скрываться.
* [Closed](/ios/sdk/reference/stable/struct#closed) — структура, представляющая статус `Закрыто` у объекта справочника с информацией о причине закрытия и времени до открытия.
* [ClosedReason](/ios/sdk/reference/stable/enumeration#closed-reason) — причины установки статуса `Закрыто` у объекта справочника.
* [ClosingHours](/ios/sdk/reference/stable/struct#closing-hours) — структура с информацией о нерабочих интервалах и причине закрытия объекта справочника.
* [ClosingSoon](/ios/sdk/reference/stable/struct#closing-soon) — структура, представляющая статус `Скоро закроется` у объекта справочника с информацией о причине и времени закрытия.
* [OpenStatus](/ios/sdk/reference/stable/enumeration#open-status) — класс, описывающий детализированный статус работы объекта справочника в данный момент (`Открыто`, `Скоро откроется`, `Скоро закроется`, `Закрыто`).
* [Opened](/ios/sdk/reference/stable/struct#opened) — структура, представляющая статус `Открыто` у объекта справочника с информацией о времени до закрытия.
* [OpeningSoon](/ios/sdk/reference/stable/struct#opening-soon) — структура, представляющая статус `Скоро откроется` у объекта справочника с информацией о времени до открытия.
* [OpeningHours.weekClosingHours](/ios/sdk/reference/stable/struct#opening-hours/var-week-closing-hours) — свойство для получения списка нерабочих интервалов объекта справочника по каждому дню недели.
* [OpeningHours.isOpen24x7](/ios/sdk/reference/stable/struct#opening-hours/var-is-open24x7) — свойство для признака круглосуточной работы объекта справочника.
* [WorkStatus.openStatus](/ios/sdk/reference/stable/struct#work-status/var-open-status) — свойство с детальным статусом `Открыто` у объекта справочника.
* [WorkStatus.openStatusHint](/ios/sdk/reference/stable/struct#work-status/var-open-status-hint) — свойство текстовой подсказки о статусе `Открыто` у объекта справочника.
* [WorkStatus.scheduleHint](/ios/sdk/reference/stable/struct#work-status/var-schedule-hint) — свойство текстовой подсказки о расписании работы объекта справочника.
* [WorkStatus.breakHint](/ios/sdk/reference/stable/struct#work-status/var-break-hint) — свойство текстовой подсказки о текущем или ближайшем перерыве в работе объекта справочника.
* [Reviews.countWithStars](/ios/sdk/reference/stable/struct#reviews/var-count-with-stars) — свойство для общего количества оценок объекта справочника, включая отзывы без текста.

### ❗ Изменения, влияющие на обратную совместимость

* Обновлен формат офлайн-пакетов для поддержки новых справочных данных.

### 🛠 Исправления

* Запросы тайлов пробок через источник [TrafficSource](/ios/sdk/reference/stable/class#traffic-source) теперь компонуются для оптимизации количества запросов.
* Оптимизирована отрисовка тайлов карты, чтобы не нагружать все доступные ядра процессора.
* Увеличен уровень масштабирования для растровых тайлов, добавляемых через источник [RasterTileSource](/ios/sdk/reference/stable/class#raster-tile-source), до 16.
* Исправлена ошибка при добавлении анимированной модели [ModelMapObject](/ios/sdk/reference/stable/class#model-map-object), из-за которой не перезапускались анимации и не поворачивались модели к камере.
* Исправлена работа канала готовности карты [Map.dataLoadingStateChannel](/ios/sdk/reference/stable/class#map/var-data-loading-state-channel) в иммерсивном режиме (когда для [Map.graphicsPreset](/ios/sdk/reference/stable/class#map/var-graphics-preset) установлено значение [GraphicsPreset.immersive](/ios/sdk/reference/stable/enumeration#graphics-preset/immersive)).
* Исправлено поведение значения [TextStyle.textPlacement](/ios/sdk/reference/stable/struct#text-style/var-text-placement) в свойстве [Marker.textStyle](/ios/sdk/reference/stable/class#marker/var-text-style): теперь при изменении значения корректно меняется положение текста относительно маркера.
* В мини-карточке объектов [SearchResultItemView](/ios/sdk/reference/stable/struct#search-result-item-view) теперь используется корректное значение для количества отзывов ([Reviews.countWithStars](/ios/sdk/reference/stable/struct#reviews/var-count-with-stars)) вместо общего количества отзывов.
* Исправлены рывки и рассинхронизация отображения маркера геопозиции и линии маршрута при навигации по длительным маршрутам между регионами.
* Исправлено отображение километров в баблах маршрутов в арабской локали.
* Обновлены значения по умолчанию настроек [ZoomFollowSettings](/ios/sdk/reference/stable/class#zoom-follow-settings) в навигаторе для корректного отображения видимого пространства на карте.
* Исправлены UX-ошибки в [RouteEditorView](/ios/sdk/reference/stable/struct#route-editor-view): например улучшена обработка жестов и скроллинга маршрутов в ландшафтном режиме.
* Теперь для получения маршрутов на общественном транспорте через [PublicTransportRouteSearchOptions](/ios/sdk/reference/stable/struct#public-transport-route-search-options) по умолчанию используется [Routing API](/api/navigation/routing/overview).

* [RoadEventFilter](/ios/sdk/reference/stable/interface#road-event-filter) и [RoadEventSource.roadEventFilter](/ios/sdk/reference/stable/class#road-event-source/var-road-event-filter) — интерфейс фильтра отображаемых дорожных событий.
