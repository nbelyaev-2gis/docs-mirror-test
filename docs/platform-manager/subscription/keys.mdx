# Создание ключей доступа

Создайте бесплатный демо-ключ или основной ключ внутри подписки для работы с API <Brand />.

## Создание демо-ключа {#demo}

:::note[Доступность для мобильной разработки]

Для работы с мобильным SDK недоступны демо-ключи. Оформите [подписку](/platform-manager/subscription/purchase) и создайте ключ внутри неё.

:::

Бесплатный демо-ключ создаётся только один раз и действует один месяц со дня создания. На каждый сервис действует ограничение в 1000 запросов.

1. Зарегистрируйтесь в личном кабинете Менеджер Платформы.

   <CtaButton href="https://platform.2gis.ru/ru/?utm_source=docs&utm_medium=referral&utm_campaign=account" target="_blank">
      В кабинет
   </CtaButton>

2. На вкладке **Главная** нажмите **Создать демо-ключ**.
3. Введите название ключа и нажмите **Создать**.

После создания ключа для начала работы с API вы можете использовать примеры в документации, например [примеры запросов к Places API](/api/search/places/examples).

Подробнее об управлении ключами см. в инструкции [Управление ключами доступа](/platform-manager/subscription/managing-keys). Также вы можете посмотреть [статистику расходования лимитов и распределения запросов](/platform-manager/subscription/statistics).

## Создание API-ключа {#production}

Если вы впервые оформили подписку, первый API-ключ создастся автоматически.

Чтобы создать ключ:

1. [Оформите подписку](/platform-manager/subscription/purchase).
2. Перейдите на вкладку **Ключи API** и нажмите **Создать API-ключ**.
3. Введите название ключа и нажмите **Создать**.
4. Если вы планируете использовать ключ для работы с мобильным SDK:

    1. Выберите опцию **Использовать для SDK**.
    2. (Опционально) Укажите ID приложения (App ID), для которого будет использоваться ключ. Указание App ID обязательно только для версий SDK ниже 13.3.0 или если вы планируете работу с офлайн-данными. Требования к App ID:

        <Tabs>
        <TabItem value="ios" label="Для iOS" default>

        Можно использовать буквы латинского алфавита (A–Z, a–z), цифры (0–9), точки (.) для разделения сегментов и дефисы (-).

        Подробнее см. в [официальной документации](https://developer.apple.com/documentation/bundleresources/information-property-list/cfbundleidentifier).

        </TabItem>
        <TabItem value="android" label="Для Android и Flutter">

        * Можно использовать буквы латинского алфавита (A–Z, a–z), цифры (0–9), точки (.) для разделения сегментов и нижнее подчёркивание (_).
        * App ID должен содержать минимум два сегмента, разделённых точками (например, `com.example.app`). Каждый сегмент должен начинаться с буквы.

        Подробнее см. в [официальной документации](https://developer.android.com/build/configure-app-module#kts).

        </TabItem>
        </Tabs>

        :::warning

        Если в ключе указан App ID, вы можете использовать этот ключ только для работы с мобильным SDK. Чтобы работать с API <Brand /> напрямую (через HTTP-запросы), создайте отдельный ключ.

        :::

    3. Нажмите **Сохранить и скачать ключ**.

5. Перейдите к документации интересующего вас продукта, чтобы начать работу с API.

Вы можете использовать созданный API-ключ для работы со всеми сервисами, которые включены в подписку.
