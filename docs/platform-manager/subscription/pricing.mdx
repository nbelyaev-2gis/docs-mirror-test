# Тарифы и правила тарификации

На этой странице представлена информация о стоимости [сервисов](/platform-manager/subscription/services) в подписке, деталях тарификации и лимитах.

Тарификация основана на количестве успешно обработанных запросов или полученных единиц тарификации (маршрутов, тайлов и т. д.). Доступ к сервисам приобретается пакетами с фиксированным количеством единиц тарификации и лимитом запросов.

## Единицы тарификации {#billable-units}

В таблицах ниже указано, что считается единицей тарификации для каждого сервиса. Успешным запросом к API считается запрос, который возвращает HTTP-код ответа 200 или 204.

### Карты {#billable-units-maps}

| <div style={{"width":"150px"}}>Сервис</div> | Единица&nbsp;тарификации | Детали |
| ------ | ------------------------ | ---------------------------- |
| Map&nbsp;Tiles&nbsp;API | Успешный запрос | Запрос возвращает один векторный тайл карты размером 256×256 пикселей. Точное количество тайлов для отображения карты зависит от размера области покрытия. См. алгоритм расчёта ниже. |
| Raster&nbsp;Tiles&nbsp;API | Успешный запрос | Запрос возвращает один растровый тайл карты размером 256×256 пикселей. Точное количество тайлов для отображения карты зависит от размера области покрытия. См. алгоритм расчёта ниже. |
| Static&nbsp;API | Успешный запрос | Запрос возвращает одно статическое изображение карты. |

Расчёт количества векторных или растровых тайлов для отображения карты:

1. Определите размер карты для отображения в пикселях (ширина и высота).
2. Разделите ширину и высоту карты на 256 (размер одного тайла) и округлите полученные значения до ближайшего целого числа в большую сторону.
3. Умножьте количество тайлов по горизонтали на количество тайлов по вертикали.

Полученное число — минимальное количество тайлов, необходимое для отображения видимой области карты. Рекомендуется добавить по одному тайлу с каждой стороны, чтобы обеспечить плавную прокрутку и масштабирование.

Например, чтобы отобразить область карты размером 800×600 пикселей, нужно минимум 12 тайлов (4 по горизонтали и 3 по вертикали), а оптимально — 30 тайлов (6 по горизонтали и 5 по вертикали).

## Лимиты {#limits}

В рамках подписки действуют лимиты на использование [единиц тарификации](#billable-units) двух видов:

* В течение платёжного месяца. Этот лимит определяется при [оформлении подписки](/platform-manager/subscription/purchase#purchasing) и расходуется всеми ключами в подписке. См. доступные пакеты единиц тарификации в разделе [Стоимость](#pricing).
    
    При превышении этого лимита доступ к сервису будет временно ограничен для всех ключей в подписке. Вы можете дополнительно уменьшить этот лимит для отдельных ключей: см. раздел [Настройка лимитов для сервисов](/platform-manager/subscription/managing-keys#configuring-limits). Чтобы увеличить этот лимит, вы можете докупить дополнительные единицы тарификации: см. раздел [Покупка дополнительных лимитов и сервисов](/platform-manager/subscription/purchase#upgrading). 

* В минуту. Этот лимит фиксирован для каждого сервиса и не может быть увеличен. Точные значения указаны в таблицах ниже.
    
    При превышении этого лимита любым из ключей в подписке доступ к сервису будет временно ограничен для всех ключей. Вы можете дополнительно уменьшить этот лимит для отдельных ключей: см. раздел [Настройка лимитов для сервисов](/platform-manager/subscription/managing-keys#configuring-limits).
