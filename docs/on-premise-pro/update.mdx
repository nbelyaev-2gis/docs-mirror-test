# Обновление <Brand /> Про

## Обновление Pro UI {#update-pro-ui}

1. Получите новые артефакты для установки.

   1. Укажите в конфигурационном файле 2GIS CLI:

      * Новую версию компонента `pro` (см. [Релизы <Brand /> Про](/on-premise-pro/releases)).
      * Версии компонентов `core` и `api-platform`, которые требуются для установки новой версии <Brand /> Про (см. подраздел **Зависимости** в описании релизов <Brand /> Про).

   2. [Выполните команду загрузки](/on-premise-api-platform/preparation#fetch-artifacts).

      Дополнительные настройки см. в [справке по командам и аргументам 2GIS CLI](/on-premise-pro/overview/2gis-cli/help).

2. Скопируйте путь к новому файлу манифеста из консольного вывода команды и подставьте его в параметр `dgctlStorage.manifest` в конфигурационном файле `values-pro-ui.yaml`. Подробнее см. в [руководстве по установке](/on-premise-pro/install/installation#install-pro-ui).

   :::note[Выбор манифеста]

   Если при очередном обновлении сервиса Pro UI вы не хотите импортировать новые данные, **используйте тот же манифест, что и в предыдущем импорте**. В остальных случаях рекомендуется использовать новый манифест.

   :::

3. Ознакомьтесь сo [списком критических изменений](/on-premise-pro/releases). Если необходимо, обновите другие настройки в конфигурационном файле `values-pro-ui.yaml`.

4. Выполните команду для обновления сервиса:

   ```bash
   helm upgrade --version=VERSION --atomic --values ./values-pro-ui.yaml pro-ui 2gis-on-premise/pro-ui
   ```

   В параметр `--version` подставьте новую версию компонента `pro`, которую вы указывали в конфигурационном файле 2GIS CLI.

   :::warning

   Версии сервисов внутри компонента должны быть одинаковыми и совпадать с версией компонента.

   :::

5. [Проверьте работоспособность сервиса](/on-premise-pro/install/installation#test).

## Обновление Pro API {#update-pro-api}

1. (Рекомендуется) Выполните резервное копирование наборов данных, которые загружали пользователи.

2. Получите новые артефакты для установки.

   1. Укажите в конфигурационном файле 2GIS CLI:

      * Новую версию компонента `pro` (см. [Релизы <Brand /> Про](/on-premise-pro/releases)).
      * Версии компонентов `core` и `api-platform`, которые требуются для установки новой версии <Brand /> Про (см. подраздел **Зависимости** в описании релизов <Brand /> Про).

   2. [Выполните команду загрузки](/on-premise-api-platform/preparation#fetch-artifacts). Дополнительно вы можете указать следующие флаги:

      * Чтобы обновить только образ сервиса, добавьте флаг `--only-apps` после команды `pull`.
      * Чтобы обновить только данные, добавьте флаг `--only-data` после команды `pull`.
      * Чтобы обновить и сервис, и данные, выполните команду без добавления флагов выше.

      Дополнительные настройки см. в [справке по командам и аргументам 2GIS CLI](/on-premise-pro/overview/2gis-cli/help).

3. Скопируйте путь к новому файлу манифеста из консольного вывода команды и подставьте его в параметр `dgctlStorage.manifest` в конфигурационном файле `values-pro-api.yaml`. Подробнее см. в [руководстве по установке](/on-premise-pro/install/installation#install-pro-api).

   :::note[Выбор манифеста]

   Если при очередном обновлении сервиса Pro API вы не хотите импортировать новые данные, **используйте тот же манифест, что и в предыдущем импорте**. В остальных случаях рекомендуется использовать новый манифест.

   :::

4. Ознакомьтесь сo [списком критических изменений](/on-premise-pro/releases). Если необходимо, обновите другие настройки в конфигурационном файле `values-pro-api.yaml`.

5. Выполните команду для обновления сервиса:

   ```bash
   helm upgrade --version=VERSION --atomic --values ./values-pro-api.yaml pro-api 2gis-on-premise/pro-api
   ```

   В параметр `--version` подставьте версию компонента `pro`:

   * Чтобы обновить только сервис или сервис и данные, используйте новую версию, которую вы указывали в конфигурационном файле 2GIS CLI.
   * Чтобы обновить только данные, подставьте текущую установленную версию.

6. [Проверьте работоспособность сервиса](/on-premise-pro/install/installation#test).
