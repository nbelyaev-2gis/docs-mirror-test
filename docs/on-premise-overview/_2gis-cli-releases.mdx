## 3 {#v3}

Изменился процесс работы с артефактами установки:

* Все сервисы, доступные для установки в On-Premise, объединены в компоненты: см. <Link to="./config">обновлённый конфигурационный файл 2GIS CLI</Link>.

* Обновлён список параметров для работы с утилитой:

  * Добавлены новые параметры `--component` и `--components` для разных режимов работы утилиты.
  * Изменено поведение параметра `--version`.

  См. <Link to="./help">Справку по командам и аргументам</Link>.

* Обновлён процесс очистки данных: см. раздел <Link to="../lifecycle#data-cleanup">Жизненный цикл артефактов установки</Link>.

* Обновлены команды для установки сервисов через Helm-чарты.

:::note[Требования к версии 2GIS CLI]

С сентября 2025 г. новые версии всех сервисов внутри On-Premise могут быть установлены только с помощью утилиты 2GIS CLI версии 3.х и при выполнении шагов ниже. Чтобы установить версии On-Premise 1.43.0 и ниже, вы можете использовать утилиту версии 2.

:::

Чтобы перейти на новый процесс установки и обновления сервисов, выполните следующие шаги:

1. Установите последнюю версию утилиты 2GIS CLI.
2. В конфигурационном файле утилиты укажите необходимые компоненты для загрузки артефактов: см. <Link to="./config">обновлённое описание структуры файла</Link>.
3. Следуйте обновлённым инструкциям по <Link to="../../preparation#fetch-artifacts">загрузке артефактов установки</Link> и установке отдельных сервисов.

## 2.21.0 {#v2.21.0}

* Обновлена работа режимов `license`, `save` и `restore`: теперь при использовании этих режимов в On-Premise версии 1.16.0 и ниже необходимо добавлять аргумент `--with-license-v1` для корректного выполнения команд. См. <Link to="./help">Справку по командам</Link>.

## 2.19.1 {#v2.19.1}

* При выполнении `dgctl pull` в лог записывается размер выгрузки (образов сервисов, данных и т.д.). См. <Link to="./overview#pull-mode">описание режима `pull`</Link>.

## 2.17.0 {#v2.17.0}

* Добавлен обязательный параметр `--user` для вызова `dgctl`. См. <Link to="./help">Справку по командам</Link>.

## 2.14.0 {#v2.14.0}

* Добавлен новый параметр `secure: bool` в <Link to="./config">конфигурационный файл `dgctl-config.yaml`</Link>, позволяющий работать с S3-совместимым хранилищем по HTTPS.

## 2.9.0 {#v2.9.0}

* При выполнении `dgctl restore`, восстанавливается два манифест файла: `latest.json` и манифест с временной меткой `{timestamp}.json`, как при выполнении `dgctl pull`.
* Оптимизирована работа утилиты, при загрузке артефактов большого размера.

## 2.8.5 {#v2.8.5}

* Представлена команда `dgctl license` с необязательным флагом `--from-file`.
