В программном комплексе <Brand /> используется несколько типов ключей и токенов.

## Лицензия на программный комплекс <Brand /> {#license}

Лицензия на программный комплекс <Brand /> используется <Link to="./2gis-cli/overview">утилитой 2GIS CLI</Link> для получения артефактов установки, а также [сервисом лицензий](/on-premise-api-platform/architecture/core/license) для контроля доступа к остальным сервисам.

Лицензия выдаётся при покупке программного комплекса <Brand /> и привязана к определённому набору сервисов.

## API-ключи {#api-keys}

![API-ключи](/img/on-premise/keys-types-2.png)

Администратор [сервиса API-ключей](/on-premise-api-platform/architecture/core/keys) создаёт API-ключи с помощью интерфейса администратора. Конечные приложения затем используют эти ключи для доступа к развёрнутым сервисам программного комплекса <Brand />.


## Сервисные токены {#service-tokens}

Сервисы программного комплекса <Brand /> используют сервисные токены для связи с [сервисом API-ключей](/on-premise-api-platform/architecture/core/keys) для валидации API-ключей и других операций.

Сервисные токены формируются после установки сервиса API-ключей и уникальны для каждого инстанса сервиса. Администратор сервиса API-ключей не может самостоятельно генерировать сервисные токены, но может получить их для настройки сервисов, работающих с API-ключами.
