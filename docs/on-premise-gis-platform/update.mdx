# Обновление GIS-платформы

1. Получите новые артефакты для установки.

   1. Укажите в конфигурационном файле 2GIS CLI:

      * Новую версию компонента `gis-platform` (см. [Релизы GIS-платформы](/on-premise-gis-platform/releases)).
      * Версии компонентов `core` и `api-platform`, которые требуются для установки новой версии GIS-платформы (см. подраздел **Зависимости** в описании релизов GIS-платформы).

   2. [Выполните команду загрузки](/on-premise-api-platform/preparation#fetch-artifacts).

      Дополнительные настройки см. в [справке по командам и аргументам 2GIS CLI](/on-premise-api-platform/overview/2gis-cli/help).

2. Ознакомьтесь сo [списком критических изменений](/on-premise-gis-platform/releases). Если необходимо, обновите настройки в конфигурационном файле `values-gis-platform.yaml`. Подробнее см. в [руководстве по установке](/on-premise-gis-platform/installation#install-gis-platform).

3. Выполните команду для обновления сервиса:

   ```bash
   helm upgrade --version=VERSION --atomic --wait-for-jobs --values ./values-gis-platform.yaml gis-platform 2gis-on-premise/gis-platform
   ```

   В параметр `--version` подставьте новую версию компонента `gis-platform`, которую вы указывали в конфигурационном файле 2GIS CLI.

   :::warning

   Версии сервисов внутри компонента должны быть одинаковыми и совпадать с версией компонента.

   :::
