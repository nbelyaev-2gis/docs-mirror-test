# Работа с Менеджером Платформы

## Отличия версий On-Cloud и On-Premise {#on-cloud-vs-on-premise}

Менеджер Платформы доступен как On-Cloud (подробнее см. в документации [личного кабинета](/platform-manager/overview)), так и On-Premise. Доступные функции различаются в зависимости от инсталляции:

|                                            | On-Cloud                                                                       | On-Premise                                                                       |
| ------------------------------------------ | ------------------------------------------------------------------------------ | -------------------------------------------------------------------------------- |
| Мониторинг состояния сервисов              | ✗                                                                              | ✓                                                                                |
| Загрузка пользовательских<br />стилей карты | ✗<br />(Можно указать стиль в [настройках карты](/mapgl/map-style/create))   | ✓                                                                                 |
| Тестирование сервисов<br />в песочнице      | ✓                                                                              | ✓                                                                               |
| Создание ключей доступа                    | ✓                                                                              | ✗<br />(Только в [веб-интерфейсе управления API-ключами](/on-premise-api-platform/core/keys)) |
| Управление ключами доступа                 | ✓                                                                              | ✗<br />(Только в [веб-интерфейсе управления API-ключами](/on-premise-api-platform/core/keys)) |
| Просмотр статистики<br />по ключам доступа  | ✓                                                                              | ✓                                                                               |
| Управление пользователями    | ✓<br />(См. инструкцию [Управление пользователями](/platform-manager/users#managing-users)) | ✗<br />(Только в [веб-интерфейсе управления API-ключами](/on-premise-api-platform/core/keys))  |
| Управление компаниями | ✓<br />(См. инструкцию [Управление компаниями](/platform-manager/users#managing-companies)) | ✗ |

## Начало работы {#getting-started}

Чтобы начать работу с Менеджером Платформы, выполните следующие шаги:

1. Убедитесь, что сервис Менеджер Платформы установлен и работает корректно. Подробнее см. в инструкции по [установке Менеджера Платформы](/on-premise-api-platform/api-platform/install/platform).
2. Перейдите по ссылке вида `https://platform.example.com`, которую вы получили в результате установки сервиса.

<br />

Интерфейс Менеджера Платформы содержит три вкладки:

* **Мониторинг**

  <img src="/img/on-premise/admin-platform-manager-status.png" alt="Мониторинг" width="600"  />

  Здесь вы можете:

  * Просматривать список сервисов, установленных внутри комплекса On-Premise.
  * Проверять статусы их работы.

* **Стили карты**

  <img src="/img/on-premise/admin-platform-manager-styles.png" alt="Стили карты" width="800"  />

  Здесь вы можете:

  * [Загружать пользовательские стили карты](#add-map-style) для использования в закрытом контуре.
  * [Обновлять стили карты](#update-map-style) и переключаться между ними. Вы можете как использовать новые стили, так и загружать обновлённые версии одного стиля.
  * Просматривать список загруженных стилей и информацию о них: ID стиля, версии и тип карты, для которой создан стиль.
  * [Удалять загруженные стили](#delete-map-style).

* **Песочницы**

  <img src="/img/on-premise/admin-platform-manager-playgrounds-532.png" alt="Песочницы" width="800"  />

  Здесь вы можете протестировать работу ключевых API (для работы с песочницами устанавливать сервисы не требуется):

  * Map Tiles API: трёхмерную интерактивную карту на движке WebGL с использованием MapGL JS.
  * Geocoder API: API поиска для определения координат объекта по адресу (прямой геокодинг) и наоборот (обратный геокодинг).
  * Directions API: API навигации для построения автомобильных, велосипедных и пешеходных маршрутов.

## Управление стилями карты {#map-styles}

### Загрузка и применение стиля карты {#add-map-style}

Вы можете создать уникальный стиль карты и использовать его в своём контуре.

1. Убедитесь, что в составе сервисов карт [установлен Styles API](/on-premise-api-platform/api-platform/install/maps#install-styles-api).

2. Откройте [Редактор стилей](https://styles.2gis.com/) и создайте стиль карты.

   См. инструкции по созданию и настройке стилей в [документации Редактора стилей](/maps/styles/create).

3. Выгрузите созданный стиль из Редактора стилей:

   1. В Редакторе стилей в блоке **Мои стили** откройте карточку нужного стиля.

   2. В верхнем меню нажмите **Поделиться**, а затем в открывшемся окне нажмите **Экспорт стиля**.

       {/*  prettier-ignore  */}

      <img src="/img/styles/share-style-links.png" alt="Экспорт стиля" width="600"  />

   3. Отметьте необходимый стиль в списке и нажмите **Экспортировать**.

      В новой панели начнётся подготовка архива для скачивания.

   4. Когда стиль будет в статусе **Готово**, выберите его в списке и нажмите **Скачать стиль**.

   5. Сохраните созданный архив в формате `.tar.gz` локально.

4. Загрузите стиль в контур On-Premise:

   1. Откройте Менеджер Платформы в браузере.
   2. Перейдите на вкладку **Стили карты** и нажмите **Загрузить стиль**.
   3. Добавьте скачанный архив в формате `.tar.gz` и дождитесь его успешной загрузки (архив должен перейти в статус **Загружено**).

5. Чтобы проверить, как выглядит загруженный стиль в песочнице, на вкладке **Стили карты** нажмите на название нужного стиля.

6. Скопируйте сгенерированный ID стиля в поле **ID стиля карты** и [примените его к карте с помощью MapGL JS API](/mapgl/examples/styles).

Вы можете загружать разные версии одного стиля и переключаться между ними в Менеджере Платформы: подробнее см. в разделе [Обновление стиля карты](#update-map-style).

### Обновление стиля карты {#update-map-style}

Вы можете применить к картам новый стиль или обновить созданный ранее.

1. Откройте [Редактор стилей](https://styles.2gis.com/) и создайте новый стиль или отредактируйте существующий.

   См. инструкции по созданию и настройке стилей в [документации Редактора стилей](/maps/styles/create).

2. Выполните шаги по [выгрузке стиля из Редактора стилей и загрузке в Менеджер Платформы](#add-map-style).

3. Если вы загружаете обновлённую версию существующего стиля, его ID остаётся прежним. Менеджер Платформы распознает обновление как новую версию стиля. В дальнейшем вы сможете переключаться между версиями одного стиля.

4. Если вы загружаете новый стиль, скопируйте сгенерированный ID стиля в поле **ID стиля карты** и [примените его к карте с помощью MapGL JS API](/mapgl/examples/styles).

### Удаление стиля карты {#delete-map-style}

:::warning

Перед удалением стиля убедитесь, что он не применён ни к одной из действующих карт. В противном случае после удаления стиля карта может отображаться некорректно.

:::

1. Откройте Менеджер Платформы в браузере.
2. Перейдите на вкладку **Стили карты** и нажмите ![Корзина](/img/on-premise/trash-icon.svg) рядом с нужным стилем.

## Просмотр статистики {#viewing-statistics}

На вкладке **Статистика** вы можете посмотреть подробные данные о количестве запросов. Вы можете отфильтровать данные по временному периоду, сервису и API-ключу. Пример отображения статистики:

<img src="/img/platform-manager/keys-statistics.png" alt="Статистика" width="700" />

На графике отображается количество запросов к сервисам за каждый день выбранного периода.

Данные на всех графиках обновляются с задержкой в несколько минут.

### Статистика распределения запросов

Чтобы посмотреть статистику распределения запросов:

1. Перейдите на вкладку **Статистика**.

2. Выберите период, за который нужно получить статистику (сегодня, последняя неделя, месяц, год), или укажите произвольный период в поле **Период**. Дата и время указываются в формате UTC.

3. В поле **Ключ** выберите ключи доступа, по которым нужно получить статистику. Чтобы выбрать все ключи, выберите **Выбрать все**.

   Чтобы отфильтровать ключи по статусу и режиму использования, нажмите значок ![Фильтр](/img/platform-manager/filter-icon.svg), выберите статус ключа (`Активен`, `Неактивен`) и режим (`Demo` для демо-ключа и `Prod` для основного ключа).

4. В поле **Сервис** выберите сервисы, по которым нужно получить статистику. Чтобы выбрать все сервисы, выберите **Выбрать все**.

### Скачивание файла со статистикой {#downloading-a-file-with-the-statistics}

Вы можете скачать статистику в виде файла с таблицей в формате `.xlsx`. Таблица содержит следующие поля:

* **Ключ**: ID ключа.
* **Дата**: дата запроса.
* **Сервис**: сервис, к которому был отправлен запрос.
* **Режим**: режим использования ключа (демо-ключ `demo` или основной ключ `prod`).
* **Метод**: endpoint, к которому был отправлен запрос.
* **Результат**: результат запроса (`Успех`, `Пустой ответ`, `Ошибка`).
* **Код результата**: код результата запроса.
* **Количество запросов**: количество запросов в течение дня, указанного в дате.
* **Организации**: количество организаций, полученных через Places API, Suggest API и Markers API.
* **Геообъекты**: количество геообъектов, полученных через [API поиска](/platform-manager/subscription/services#search).
* **Маршруты**: количество маршрутов, построенных через [API навигации](/platform-manager/subscription/services#navigation), кроме Isochrone API.

Чтобы скачать файл со статистикой распределения запросов:

1. Перейдите на вкладку **Статистика**.
2. Выберите период, ключи и сервисы.
3. Нажмите **Скачать в формате XLSX**.
