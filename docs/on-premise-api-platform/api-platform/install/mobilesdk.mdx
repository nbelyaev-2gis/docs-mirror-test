# Установка мобильного SDK

:::note[Доступность версий]

Для использования в закрытом контуре доступен мобильный SDK для iOS, Android и Flutter до версии 12.10 включительно. 

:::

## 1. Перед установкой {#before-installing}

1. Ознакомьтесь с основной информацией:

   * [Архитектура мобильного SDK в комплексе On-Premise](/on-premise-api-platform/architecture/api-platform/mobilesdk)
   * [Документация Android Mobile SDK](/android/sdk/overview)
   * [Документация iOS Mobile SDK](/ios/sdk/overview)

2. Обратитесь в службу поддержки On-Premise, чтобы получить необходимые данные:

   * Файл ключа для использования SDK. При обращении в поддержку уточните:

     * Ключ должен быть настроен на использование в режиме **On-Premise**.
     * App ID ключа должен соответствовать `applicationId`/`bundleId` приложения, в котором подключается мобильный SDK.

     Также вы можете запросить включение wildcard в ключе для использования каскадного App ID. Например, если в ключе включен wildcard и настроен App ID == `com.geo.app`, то с этим ключом мобильный SDK можно использовать в приложениях с `applicationId`/`bundleId` типа `com.geo.app.testing`, `com.geo.app.staging`, `com.geo.app.dev` и другими.

   * Конфигурационный файл `vendor-config.jsonx` с настроенными сервисами On-Premise.

     В этом файле помимо URL сервисов указан идентификатор ключа из [Сервиса API-ключей](/on-premise-api-platform/core/install/keys) в поле `dgis/native-sdk/keys/user_web_api_key_for_on_premise`. Этот идентификатор пользователь сервиса может задавать самостоятельно на основе ключей из [Сервиса API-ключей](/on-premise-api-platform/core/install/keys).

## 2. Установите мобильный SDK {#install-mobile-sdk}

1. Установите мобильный SDK, следуя инструкциям:

   * для [Android-платформы](/android/sdk/overview),
   * для [iOS-платформы](/ios/sdk/overview).

2. При инициализации SDK в своём приложении подключите `vendor-config.jsonx`, следуя инструкциям:

   * для [Android-платформы](/android/sdk/start#vendor-config),
   * для [iOS-платформы](/ios/sdk/start#vendor-config).
