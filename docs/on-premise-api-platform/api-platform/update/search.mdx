# Обновление API для работы с поиском

## Обновление сервиса Search API {#update-search-api}

1. Получите новые артефакты для установки.

   1. Укажите в конфигурационном файле 2GIS CLI:

      * Новую версию компонента `api-platform` (см. [Релизы API-платформы](/on-premise-api-platform/api-platform/releases)).
      * Версию компонента `core`, которая требуется для установки новой версии API-платформы (см. подраздел **Зависимости** в описании релизов API-платформы).

   2. [Выполните команду загрузки](/on-premise-api-platform/preparation#fetch-artifacts). Дополнительно вы можете указать следующие флаги:

      * Чтобы обновить только образ сервиса, добавьте флаг `--only-apps` после команды `pull`.
      * Чтобы обновить только данные, добавьте флаг `--only-data` после команды `pull`.
      * Чтобы обновить и сервис, и данные, выполните команду без добавления флагов выше.

      Дополнительные настройки см. в [справке по командам и аргументам 2GIS CLI](/on-premise-api-platform/overview/2gis-cli/help).

2. Скопируйте путь к новому файлу манифеста из консольного вывода команды и подставьте его в параметр `dgctlStorage.manifest` в конфигурационном файле `values-search.yaml` или `values-search-api-v8.yaml`. Подробнее см. в руководстве по установке [Search API](/on-premise-api-platform/api-platform/install/search#install-search-api) и [Search API v8](/on-premise-api-platform/api-platform/install/search-v8#install-services).

   :::note[Выбор манифеста]

   Если при очередном обновлении сервисов поиска вы не хотите импортировать новые данные, **используйте тот же манифест, что и в предыдущем импорте**. В остальных случаях рекомендуется использовать новый манифест.

   :::

3. Ознакомьтесь сo [списком критических изменений](/on-premise-api-platform/api-platform/releases). Если необходимо, обновите другие настройки в конфигурационном файле `values-search.yaml`.

4. Выполните команду:

   * для обновления Search API:

     ```bash
     helm upgrade --version=VERSION --atomic --values ./values-search.yaml search-api 2gis-on-premise/search-api
     ```

   * для обновления Search API v8:

     ```bash
     helm upgrade --version=VERSION --atomic --values ./values-search-api-v8.yaml search-api-v8 2gis-on-premise/search-api-v8
     ```

   В параметр `--version` подставьте версию компонента `api-platform`:

   * Чтобы обновить только сервис или сервис и данные, используйте новую версию, которую вы указывали в конфигурационном файле 2GIS CLI.
   * Чтобы обновить только данные, подставьте текущую установленную версию.

5. [Проверьте работоспособность сервиса](/on-premise-api-platform/api-platform/install/search#test).

4. Выполните команду для обновления сервиса:

   ```bash
   helm upgrade --version=VERSION --atomic --values ./values-catalog.yaml catalog-api 2gis-on-premise/catalog-api
   ```

   * Чтобы обновить только данные, подставьте текущую установленную версию.

5. [Проверьте работоспособность сервиса](/on-premise-api-platform/api-platform/install/search#test).

### Удаление старых наборов данных

Если включено автоматическое создание новых схем при импорте данных об объектах (см. настройки группы `importer` [конфигурационного файла](/on-premise-api-platform/api-platform/install/search#install-catalog-apis)), то предыдущий набор данных становится ненужным после успешного переключения на новый набор. Для экономии места на диске рекомендуется удалять старые наборы данных. Это можно делать автоматически следующим образом:

1. В настройке `importer.cleaner.enabled` укажите значение `true`.

2. В настройке `importer.cleaner.versionLimit` укажите количество старых наборов, которые нужно хранить. Например, `1` означает, что в базе будет сохранено не более 3 схем (текущая, предыдущая и временная, созданная в момент импорта), а все более старые будут удалены. Обратите внимание, что этот лимит не может быть менее `1`, так как это сделает невозможным возврат к предыдущей версии при необходимости.
