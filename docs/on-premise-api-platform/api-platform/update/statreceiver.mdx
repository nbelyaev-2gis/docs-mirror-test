# Обновление сервиса сбора статистики

Если у вас установлен компонент Stat API как часть сервиса сбора статистики, обновите его вместе с компонентом Stat Receiver.

1. Получите новые артефакты для установки.

   1. Укажите в конфигурационном файле 2GIS CLI:

      * Новую версию компонента `api-platform` (см. [Релизы API-платформы](/on-premise-api-platform/api-platform/releases)).
      * Версию компонента `core`, которая требуется для установки новой версии API-платформы (см. подраздел **Зависимости** в описании релизов API-платформы).

   2. [Выполните команду загрузки](/on-premise-api-platform/preparation#fetch-artifacts).

      Дополнительные настройки см. в [справке по командам и аргументам 2GIS CLI](/on-premise-api-platform/overview/2gis-cli/help).

2. Ознакомьтесь сo [списком критических изменений](/on-premise-api-platform/api-platform/releases). Если необходимо, обновите настройки в конфигурационных файлах `values-stat-receiver.yaml` и `values-stat-api.yaml`. Подробнее см. в [руководстве по установке](/on-premise-api-platform/api-platform/install/statreceiver#install-services).

3. Обновите сервисы по отдельности с помощью команд:

   * Stat Receiver:

      ```bash
      helm upgrade --version=VERSION --atomic --values ./values-stat-receiver.yaml stat-receiver 2gis-on-premise/stat-receiver
      ```

   * Stat API:

      ```bash
      helm upgrade --version=VERSION --atomic --values ./values-stat-api.yaml stat-api 2gis-on-premise/stat-api
      ```

   В параметр `--version` подставьте новую версию компонента `api-platform`, которую вы указывали в конфигурационном файле 2GIS CLI.

   :::warning

   Версии сервисов внутри компонента должны быть одинаковыми и совпадать с версией компонента.

   :::
