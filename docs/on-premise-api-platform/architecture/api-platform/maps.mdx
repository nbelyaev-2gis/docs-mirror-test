# Сервисы карт

Сервисы карт позволяют отображать карты <Brand /> на сайтах, а также в веб- и мобильных приложениях.

В этой статье описывается, как установить и настроить сервисы карт в вашем окружении. Чтобы узнать, как использовать API, предоставляемые сервисами карт, обратитесь к [документации MapGL JS API](/mapgl/overview/features).

## Архитектура {#architecture}

<NoInvert>
![Архитектура сервисов карт](/img/on-premise/maps-architecture-522.svg)
</NoInvert>

### Nginx Web Server

Предоставляет статические файлы (шрифты, иконки) для MapGL JS API – фактически, для сайтов и веб-приложений. Чтобы использовать API, необходимо загрузить с веб-сервера основной файл библиотеки (`api.js`) и светлый стиль карты, который используется по умолчанию (находится в директории `style`).

### Прокси для API пробок

Предоставляет данные о пробках в реальном времени с серверов <Brand />. Конечные пользователи и приложения затем могут с помощью MapGL JS API или Mobile SDK отобразить пробки в виде цветных линий на отдельном слое карты.

### Tiles API

Предоставляет API для получения и отображения *тайлов карты*. Тайл – квадратное изображение, соответствующее участку карты ([пример тайлированной карты](https://en.wikipedia.org/wiki/Tiled_web_map#/media/File:Tiled_web_map_Stevage.png)). С помощью тайлов становится возможным рендерить только видимую часть карты, подгружая необходимые тайлы при перемещении по карте или при изменении масштаба карты. Таким образом, потребление памяти при работе с картой снижается.

Детали архитектуры программного комплекса <Brand /> приведены в документе [Архитектура](/on-premise-api-platform/architecture/solution-architecture).

Тайлы, предоставляемые Tiles API:

| Вид тайлов | Потребители                             |
| ---------- | --------------------------------------- |
| Векторные  | MapGL JS API, Mobile SDK                |
| Мобильные  | Mobile SDK                              |
| Растровые  | MapGL JS API, Mobile SDK, GIS-платформа |

### Tilegen API

Предоставляет API для создания векторных тайлов из пользовательских данных.

### Styles API

Предоставляет API для удобного подключения пользовательских стилей карты. Вы можете создать собственный стиль в [Редакторе стилей](https://styles.2gis.com/), загрузить его в контур On-Premise через интерфейс [Менеджера Платформы](/on-premise-api-platform/architecture/api-platform/platform) и подключить к своей карте в настройках MapGL JS API. Подробнее см. в инструкции [Загрузка и применение стиля карты](/on-premise-api-platform/api-platform/user-guide/platform-manager#adding-map-style).

Если Styles API не установлен, к карте будет применён стиль по умолчанию.

### Сервис лицензий

Предоставляет лицензию для доступа к сервису карт. Подробнее см. [архитектуру сервиса лицензий](/on-premise-api-platform/architecture/core/license).

### Сервис сбора статистики

Собирает статистику использования API-ключей для доступа к сервисам карт. Подробнее см. [архитектуру сервиса сбора статистики](/on-premise-api-platform/architecture/api-platform/statreceiver).

### Сервис API-ключей

Предоставляет API-ключ для доступа к сервису карт. Подробнее см. [архитектуру сервиса API-ключей](/on-premise-api-platform/architecture/core/keys).

## Зависимости {#dependencies}

Зависимости, обязательные для всех конфигураций:

* Сервис лицензий.
* Сервис API-ключей.
* Хранилище данных Apache Cassandra для тайлов. Конфигурации с MapGL JS API и с Mobile SDK используют отдельные наборы тайлов.

:::tip[Можно использовать аналог из реестра Минцифры]

Доступные аналоги Apache Cassandra, включённые в реестр Минцифры, см. в разделе [Системные требования](/on-premise-api-platform/requirements).

:::

Опциональные зависимости, включение которых зависит от выбранной конфигурации:

* [Прокси для API пробок](/on-premise-api-platform/architecture/api-platform/trafficproxy). Конфигурации с MapGL JS API и с Mobile SDK используют отдельные экземпляры.
* Веб-сервер nginx, необходимый для загрузки стилей и иконок.
* Сервис Styles API, необходимый для загрузки пользовательских стилей, созданных в [Редакторе стилей](https://styles.2gis.com/). Для работы Styles API необходимо настроить PostgreSQL и S3-совместимое хранилище.
* Сервис Tilegen API, необходимый для создания векторных тайлов из пользовательских данных. Для работы Tilegen API необходимо настроить Redis.

Подробнее о том, как проверить требования для каждого сервиса, см. в документе [Системные требования](/on-premise-api-platform/requirements).
