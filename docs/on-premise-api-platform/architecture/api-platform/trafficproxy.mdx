# Прокси для API пробок

Сервис прокси позволяет получать данные о пробках в реальном времени для сервисов программного комплекса <Brand />, которым нужны эти данные.

## Архитектура {#architecture}

![Архитектура прокси для API пробок](/img/on-premise/traffic-proxy-architecture.png)

Этот сервис включает в себя только сервис nginx (обратный прокси).

Обратный прокси nginx:

1. Получает данные о пробках в реальном времени с публичных серверов обновлений для пробок <Brand />.

   Прокси для API пробок может быть установлен в разных конфигурациях, которые зависят от используемых серверов обновлений:

   * Если прокси использует серверы обновлений `https://datagateway.api.2gis.com`, то данные о пробках предоставляются для использования сервисами навигации.
   * Если прокси использует серверы обновлений `https://jam.api.2gis.com`, то данные о пробках предоставляются в векторном и растровом форматах для использования сервисами карт.

2. Раздаёт эти данные по HTTP.

   :::note[Особенности настройки прокси для API пробок]

   Различные сервисы требуют установки определённой конфигурации прокси для API пробок, [использующей конкретные серверы обновлений для пробок](/on-premise-api-platform/api-platform/install/trafficproxy) (настройка `proxy.host`).
   См. детали в документации соответствующих сервисов.

   :::

   Обычно прокси раздает данные о пробках конечным пользователям и приложениям, которые пользуются сервисами. Однако из этого правила есть несколько исключений:

   * Прокси, который раздает данные о пробках в растровом формате, также может использоваться [бэкендом SPCore](/on-premise-gis-platform/architecture/gis-platform-architecture).
   * Прокси, который раздает данные о пробках в формате для навигации, недоступен для конечных пользователей и приложений, и предоставляет данные исключительно сервису [Navi-Back](/on-premise-api-platform/architecture/api-platform/navigation).

   Пример для сервиса карт в конфигурации с MapGL JS API и Tiles API:

   ![Пример организации доступа в интернет для сервиса карт](/img/on-premise/internet-access-maps.png)

## Зависимости {#dependencies}

Обратному прокси nginx должен быть предоставлен доступ в интернет, чтобы этот сервис мог обращаться к публичным серверам обновлений для пробок <Brand />.

Подробнее о том, как проверить требования для каждого сервиса, см. в документе [Системные требования](/on-premise-api-platform/requirements).

## Использование сервиса {#using-traffic-proxy}

Настройте сервис на использование адреса, указанного в параметре `ingress.hosts.host`, в качестве Traffic API endpoint. См. детали в документации соответствующих сервисов.
