# Менеджер Платформы

Менеджер Платформы — инструмент администратора установленного комплекса On-Premise, который позволяет:

* проверить статус работы сервисов;
* загрузить пользовательские стили карты (если установлен [Styles API](/on-premise-api-platform/architecture/api-platform/maps#styles-api));
* протестировать в локальной сети ключевые API: [карт](/on-premise-api-platform/architecture/api-platform/maps) (Map Tiles API), [поиска](/on-premise-api-platform/architecture/api-platform/search) (Geocoder API) и [навигации](/on-premise-api-platform/architecture/api-platform/navigation) (Directions API);
* посмотреть статистику запросов к сервисам (если установлен [сервис сбора статистики](/on-premise-api-platform/architecture/api-platform/statreceiver)).

Менеджер Платформы:

* Отправляет запрос к сервисам карт, поиска и навигации с помощью API-ключей и использует полученные данные для отображения примеров интерактивной карты, определения координат объекта и построения маршрутов на вкладке **Песочницы**.
* Обращается к сервису Styles API для импорта архива с пользовательским стилем карты.
* Обращается к сервису сбора статистики для получения статистических данных о запросах к сервисам и их отображения на вкладке **Статистика**.
* Обращается к сервису API-ключей для получения информации о компании, к которой относится пользователь. На основе этой информации Менеджер Платформы предоставляет пользователю доступ к ключам его компании на вкладке **Статистика**.

## Зависимости {#dependencies}

Сервисы On-Premise:

* [Сервис API-ключей](/on-premise-api-platform/architecture/core/keys).
* [Сервисы карт](/on-premise-api-platform/architecture/api-platform/maps): Styles API (опционально) и MapGL JS API.
* [Сервис поиска](/on-premise-api-platform/architecture/api-platform/search): Geocoder API.
* [Сервис навигации](/on-premise-api-platform/architecture/api-platform/navigation): Directions API.
* [Сервис сбора статистики](/on-premise-api-platform/architecture/api-platform/statreceiver): Stat API (опционально).

Подробнее о том, как проверить требования для каждого сервиса, см. в документе [Системные требования](/on-premise-api-platform/requirements).
