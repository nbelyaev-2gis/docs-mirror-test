# Обзор решения

API-платформа для сервера — набор картографических и геопространственных сервисов <Brand /> для работы в вашей инфраструктуре, а также сервисов для управления API-ключами и лицензией.

В API-платформу для сервера входят все сервисы, доступные в версии On-Cloud, за исключением Radar API. Функциональность сервисов может отличаться от версии On-Cloud.

К компонентам API-платформы для сервера относятся [базовые сервисы](#core-services) и [API-платформа](#api-platform). Их установка также необходима для работы продуктов [<Brand /> Про](/on-premise-pro/overview/summary) и [<Brand /> Ситискан](/on-premise-citylens/overview/summary).

## API-платформа {#api-platform}

API-платформа — набор RESTful API для работы с данными и функциями, доступными в <Brand />. Она позволяет интегрировать карты, справочник и навигацию <Brand /> в ваши приложения и сервисы.

### Сервис сбора статистики {#statistics}

Сервис сбора статистики принимает, обрабатывает и сохраняет данные об использовании API-ключей сервисами карт, поиска и навигации. Это необязательный сервис для работы API-платформы, но если вы хотите получать статистику по использованию API-ключей, сервис нужно установить до установки API ниже.

См. подробнее:

* [Архитектура сервиса сбора статистики](/on-premise-api-platform/architecture/api-platform/statreceiver)

* [Обновление сервиса сбора статистики](/on-premise-api-platform/api-platform/update/statreceiver)

### API поиска {#search}

После установки API-платформы конечные пользователи смогут использовать следующие API для работы с поиском:

| Реализуемое API                                      | Описание                                                                                                                                                                                                                                                  |
| ---------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Places API](/api/search/places/overview)         | Ищет по различным критериям организации, здания и места.                                                                                                                                                                                                  |
| [Geocoder API](/api/search/geocoder/overview)     | Определяет координаты объекта по его адресу (прямой геокодинг) и наоборот (обратный геокодинг). Также предоставляет сведения об объекте.                                                                                                                  |
| [Suggest API](/api/search/suggest/overview)       | Предлагает подсказки при вводе, подходящие под заданные критерии поиска.                                                                                                                                                                                  |
| [Categories API](/api/search/categories/overview) | Предоставляет информацию о категориях. Категория — это группа, объединяющая компании по сферам деятельности. Также предоставляет информацию о родительских и дочерних сферах деятельности для выбранной организации. Используется совместно с Places API. |
| [Regions API](/api/search/regions/overview)       | Позволяет выбрать территорию (города и ближайших населенных пунктов). Полученный идентификатор территории затем можно использовать в других API, чтобы ограничить область, в границах которой будет выполняться поиск организаций, зданий и мест.         |

### API навигации {#navigation}

После установки API-платформы (в зависимости от конфигурации компонентов) конечные пользователи смогут использовать следующие API навигации:

| Реализуемое API | Описание |
| --------------- | -------- |
| [Routing API](/api/navigation/routing/overview) | Строит маршруты на автомобиле, такси, велосипеде, самокате, мотоцикле, для экстренных служб, пешеходные и на общественном транспорте. |
| [Directions API](/api/navigation/directions/overview) | Строит маршруты на автомобиле, такси, велосипеде, самокате, мотоцикле, грузовике, а также пешеходные. |
| [Truck Directions API](/api/navigation/truck-directions/overview) | Строит маршруты для грузового транспорта с учётом параметров автомобиля и действующих дорожных ограничений. |
| [Pairs Directions API](/api/navigation/pairs/overview) | Строит несколько маршрутов за один запрос. |
| [Distance Matrix API](/api/navigation/distance-matrix/overview) | Рассчитывает расстояние и время в пути для множества точек. Это позволяет найти точки с необходимым временем достижения. |
| [TSP API](/api/navigation/tsp/overview) | Решает задачу коммивояжёра (TSP/VRP): строит кратчайший по времени или расстоянию маршрут обхода указанных точек. |
| [Isochrone API](/api/navigation/isochrone/overview) | Определяет область, достижимую за указанное время из заданной точки для разных типов маршрутов: автомобильного, велосипедного или пешеходного. Можно определять область доставки в зависимости от заданного времени доставки и доступных курьерам способов передвижения. |
| [Map Matching API](/api/navigation/map-matching/overview) | Строит автомобильный маршрут по набору точек, привязанных к автомобильным дорогам. Такой набор точек записывается транспортным средством. |

См. подробнее:

* [Архитектура API для работы с навигацией](/on-premise-api-platform/architecture/api-platform/navigation)
* [Установка API для работы с навигацией](/on-premise-api-platform/api-platform/install/navigation)

### Мобильный SDK

С помощью мобильного SDK в мобильном приложении вы можете:

* отображать карту <Brand /> и взаимодействовать с ней;
* добавлять пользовательские объекты на карту (маркеры, геометрические фигуры и т. д.);
* получать информацию об объектах на карте и искать объекты в справочнике;
* строить маршруты для различных типов транспорта и запускать навигацию.

Вы можете подключить мобильный SDK для [iOS](/ios/sdk/overview), [Android](/android/sdk/overview) и [Flutter](/flutter/sdk/overview) к установленной API-платформе.

:::note[Доступность версий]

Для использования в закрытом контуре доступен мобильный SDK для iOS, Android и Flutter до версии 12.10 включительно.

:::
