# Обновление сервиса API-ключей

1. (Рекомендуется) Выполните резервное копирование данных, которые использует сервис.

2. Получите новые артефакты для установки.

   Для этого укажите в конфигурационном файле 2GIS CLI новую версию компонента `core` (см. [Релизы базовых сервисов](/on-premise-api-platform/core/releases)) и [выполните команду загрузки](/on-premise-api-platform/preparation#fetch-artifacts).

   См. [справку по командам и аргументам 2GIS CLI](/on-premise-api-platform/overview/2gis-cli/help).

3. Скопируйте путь к новому файлу манифеста из консольного вывода команды и подставьте его в параметр `dgctlStorage.manifest` в конфигурационном файле `values-keys.yaml`. Подробнее см. в [руководстве по установке](/on-premise-api-platform/core/install/keys#install-keys).

4. Ознакомьтесь сo [списком критических изменений](/on-premise-api-platform/core/releases). Если необходимо, обновите другие настройки в конфигурационном файле `values-keys.yaml`.

5. Выполните команду для обновления сервиса:

   ```bash
   helm upgrade --version=VERSION --atomic --wait-for-jobs --values ./values-keys.yaml keys 2gis-on-premise/keys
   ```

   В параметр `--version` подставьте новую версию компонента `core`, которую вы указывали в конфигурационном файле 2GIS CLI.

   :::warning

   Версии сервисов внутри компонента должны быть одинаковыми и совпадать с версией компонента.

   :::

6. [Проверьте работоспособность сервиса](/on-premise-api-platform/core/install/keys#test).
