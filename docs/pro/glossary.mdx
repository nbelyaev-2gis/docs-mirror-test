# Глоссарий

## А

### Агрегация данных {#data-aggregation}

Преобразование нескольких значений в одно результирующее значение, которое отображается в [виджетах](#widget). Результирующее значение рассчитывается на основе значений в одной группе или диапазоне с помощью математической функции ([типа агрегации](#aggregation-type)).

**Пример:** значения этажности зданий в районах города агрегируются в среднее значение по городу.

В <Brand /> Про термин также обозначает функциональность для создания слоя, в котором каждому площадному объекту присваивается рассчитанное значение. Подробнее см. в инструкции [Агрегация данных](/pro/geospatial/aggregation).

### Агрегирующий атрибут {#aggregation-attribute}

Атрибут, по значениям которого выполняется расчёт при [агрегации данных](#data-aggregation). В качестве агрегирующего атрибута используется количество объектов или [числовой атрибут](#numeric-attribute).

**Пример:** при расчёте средней этажности зданий в разных районах города агрегирующий атрибут — количество этажей.

Для виджета [Сравнение](/pro/dashboards/widgets/comparison) используется термин **атрибут сравнения**, который выполняет ту же функцию.

### Атрибут группировки {#grouping-attribute}

Атрибут, по значениям которого данные разбиваются на группы при [агрегации данных](#data-aggregation). В качестве атрибута группировки используется [текстовый](#text-attribute), [логический](#logical-attribute), [числовой](#numeric-attribute) атрибут или атрибут с типом `Дата`.

**Пример:** при расчёте средней этажности зданий в разных районах города атрибут группировки — район.

## В

### Виджет {#widget}

Визуальный элемент [сцены](#scene) для представления данных [слоя](#layer) в виде графика, диаграммы, таблицы и т. д.

Виды виджетов:

* **Пользовательские** — создаются пользователем: линейный график, столбчатая диаграмма и т. д.
* **Системные** — доступны по умолчанию: легенда, временная шкала и т. д.

Полный список см. в инструкции [Обзор виджетов](/pro/dashboards/widgets/overview).

### Визуализация данных {#data-visualization}

Способ отображения данных [слоя](#layer) на карте: точки, полигоны, соты и т. д.

Подробнее см. в разделе [Визуализация данных](/pro/data/visualization/overview).

### Выборка данных {#data-sample}

Подмножество данных из [набора](#dataset), к которому применены [фильтры](#filter). Если фильтры не заданы, выборка включает все данные из набора. Из выборок создаются [слои](#layer).

**Пример:** к системному набору данных **Здания** применены фильтры **Назначение** со значением `Жилые` и **Кол-во этажей** со значением `>5`. В выборке останутся только жилые дома выше пяти этажей.

## Г

### Геофильтры {#geofilters}

Фильтры, ограничивающие данные по территории: район, радиус от точки, произвольный полигон и т. д. Применяются при создании [выборки данных](#data-sample) и при фильтрации на [сцене](#scene).

Подробнее см. в разделе [Геофильтры](/pro/data/overview#geofilters).

### Группировка {#grouping}

Разбиение данных на группы по значениям [атрибута группировки](#grouping-attribute) для отображения в [виджетах](#widget). Для каждой группы рассчитывается агрегированное значение с помощью выбранного [типа агрегации](#aggregation-type).

**Пример:** группировка организаций по категориям для сравнения их количества.

## Д

### Дашборд {#dashboard}

Рабочее пространство для аналитики, состоящее из одной или нескольких [сцен](#scene) со [слоями](#layer) и [виджетами](#widget). Дашборды позволяют разделять сценарии работы с данными и управлять доступом для разных пользователей.

**Пример:** дашборд «Анализ конкурентов» с двумя сценами: обзор рынка и детализация по районам.

## З

### Зона доступности {#proximity-zone}

Область на карте (изохрона), до границы которой можно добраться за указанное время определённым способом: пешком, на автомобиле, на общественном транспорте.

Подробнее см. в инструкции [Зоны доступности](/pro/geospatial/proximity).

## И

### Изменение выборки {#modifying-sample}

Изменение созданной выборки данных в [слое](#layer) путём добавления, удаления или редактирования [фильтров](#filter). Исходный [набор данных](#dataset) при этом не затрагивается.

Подробнее см. в инструкции [Изменение выборки данных](/pro/data/manage/modify).

## Л

### Легенда {#legend}

Системный [виджет](#widget), отображающий условные обозначения для элементов [визуализации данных](#data-visualization) на карте: цвета, диапазоны значений и привязку к [слоям](#layer). Создаётся автоматически при создании [сцены](#scene) и обновляется при изменении данных в слоях.

Подробнее см. в инструкции [Легенда](/pro/dashboards/widgets/legend).

Термин **легенда виджета** обозначает легенду, которая отображается в пользовательских виджетах.

### Логический атрибут {#logical-attribute}

Атрибут, принимающий одно из двух значений: истина или ложь.

**Примеры:** `Круглосуточно` (работает ли магазин круглосуточно), `Доступная среда` (доступно ли здание для маломобильных граждан).

## Н

### Набор данных {#dataset}

Исходные данные в <Brand /> Про до применения фильтров и создания [слоёв](#layer).

Типы наборов данных:

* **Системные** — предоставляются в сервисе по умолчанию или по запросу.
* **Загруженные как файл** — загружаются пользователем однократно.
* **Загруженные через API** — загружаются пользователем с возможностью дозагрузки данных.

Подробнее см. в разделе [Данные](/pro/data/overview).

## С

### Сквозной атрибут {#global-attribute}

Атрибут, который используется в разных слоях. Сквозные атрибуты позволяют применять фильтрацию через [виджеты](#widget) и [срезы](#slice) к нескольким слоям одновременно.

**Пример**: наборы данных **Карта спроса** и **Фирмы** содержат сквозной атрибут `Категория`.

### Слой {#layer}

Сохранённая [выборка данных](#data-sample) с настроенной [визуализацией](#data-visualization) на карте. Слои создаются в [сценах](#scene) дашбордов и являются основной единицей работы с данными.

**Пример:** слой «Жилые дома» — выборка зданий с визуализацией в виде полигонов, окрашенных по этажности.

### Срез {#slice}

Фильтр на [сцене](#scene), ограничивающий данные по значениям атрибутов. Настраивается через системный виджет **Фильтр-срез**. Может применяться к одному [слою](#layer) или к нескольким слоям одновременно через сквозные атрибуты.

В виджете **Фильтр-срез** доступны только значения из исходной [выборки](#data-sample).

Подробнее см. в разделе [Фильтрация через срезы](/pro/data/manage/filtering#via-slices).

### Стиль карты {#map-style}

Набор параметров, определяющих внешний вид базовой карты: цветовая схема, видимость подписей, отображение зданий и дорог.

Подробнее см. в инструкции [Управление стилем карты](/pro/styles/management).

### Сцена {#scene}

Страница [дашборда](#dashboard), содержащая набор [слоёв](#layer) и [виджетов](#widget). Сцены позволяют разделять сценарии аналитики внутри одного дашборда.

**Пример:** одна сцена дашборда используется для анализа населения, другая — для оценки локаций для открытия бизнеса.

### Сценарии аналитики {#analytics-scenarios}

Готовые сцены с набором слоёв на основе системных данных с настроенной визуализацией и виджетами.

Подробнее см. в инструкции [Сценарии аналитики](/pro/presets/overview).

## Т

### Текстовый атрибут {#text-attribute}

Атрибут, содержащий произвольные текстовые значения.

**Примеры:** `Язык обслуживания` (язык, на котором говорят сотрудники), `Тип сделки` (аренда или продажа).

### Тип агрегации {#aggregation-type}

Математическая функция, применяемая при [агрегации данных](#data-aggregation): сумма, среднее, медиана, минимум, максимум, процентили.

**Пример:** при агрегации данных с типом **Среднее** для атрибута `Количество этажей` рассчитается средняя этажность зданий в группе.

### Тултип {#tooltip}

Всплывающая подсказка, которая появляется при наведении на элемент [визуализации данных](#data-visualization) на карте. Содержит список атрибутов, выбранных при настройке визуализации слоя, и их значения.

Подробнее см. в разделе [Настройка тултипа](/pro/data/visualization/overview#configuring-tooltip).

**Пример:** при наведении на организацию тултип покажет название организации, количество филиалов и количество отзывов.

## Ф

### Фильтр {#filter}

Условие для отбора данных из [набора](#dataset). Фильтры применяются при создании [выборки](#data-sample) и при [фильтрации на сцене](#scene-filtering). По типу значений бывают [текстовыми](#text-attribute), [числовыми](#numeric-attribute), [логическими](#logical-attribute) и географическими ([геофильтрами](#geofilters)).

**Пример:** при применении фильтра **Материал** со значением `Кирпич` в выборке останутся только кирпичные здания.

### Фильтрация данных на сцене {#scene-filtering}

Дополнительная фильтрация [выборки данных](#data-sample) при работе со [сценой](#scene): через [геофильтры](#geofilters), [виджеты](#widget), [срезы](#slice) или виджет [Временная шкала](/pro/dashboards/widgets/timeline).

При фильтрации изменяется только отображение данных на текущей [сцене](#scene). Исходная выборка, [слои](#layer) и виджеты в других сценах остаются без изменений.

Подробнее см. в инструкции [Фильтрация данных на сцене](/pro/data/manage/filtering).

## Ч

### Числовой атрибут {#numeric-attribute}

Атрибут, содержащий числовые значения (целые или дробные).

**Примеры:** `Количество этажей` (этажность здания), `Количество филиалов` (число филиалов организации).

## Э

### Экспорт {#export}

Выгрузка дашбордов в формате `.pdf` или пользовательских наборов данных в формате `.csv` из <Brand /> Про для использования в других сервисах или для хранения.

Подробнее см. в инструкциях [Экспорт дашборда](/pro/dashboards/dashboards/manage#exporting-dashboard) и [Операции с наборами данных](/pro/data/manage/datasets).
