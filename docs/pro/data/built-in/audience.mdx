# Активность аудитории

В наборе данных **Активность аудитории** представлена информация об активности пользователей мобильного приложения <Brand />.

![Активность аудитории](/img/pro/data-audience-overview.png)

## Сценарии использования {#usage-scenarios}

Набор данных **Активность аудитории** помогает решать задачи:

* **Выбор времени для рекламы**. Помогает определить пиковые часы и дни для размещения наружной или цифровой рекламы на билбордах, фасадах и экранах на основе данных об активности пользователей.
* **Сравнение районов по активности**. Позволяет бизнесу (кафе, магазинам и сервисам) выявить районы с высокой концентрацией аудитории и постоянным потоком пользователей.
* **Планирование графика работы**. Помогает выявить пиковые часы активности в выбранных локациях и определить расписание сотрудников, часы доставки, время открытия и график работы точек.

## Как были получены данные? {#data-collection}

Данные обезличены, считается только общая статистика по всем пользователям. Алгоритм вычисления активности аудитории состоит из следующих шагов:

1. Сбор записей о входах в мобильное приложение за месяц.

2. Группировка записей о входах:

   * По местоположению устройства во время входа. Местоположение преобразуется в ячейку H3 (гексагон) с уровнем детализации `9`. Подробнее о способе визуализации H3 см. в инструкции [H3](/pro/data/visualization/h3).
   * По дню недели и времени суток входа. В зависимости от дня, когда был совершён вход, он учитывается в категории `Будние` или `Выходные`. В зависимости от времени, вход учитывается в одной из категорий: `Ночь` (с 00:00 до 05:59), `Утро` (с 06:00 до 11:59), `День` (с 12:00 до 17:59) или `Вечер` (с 18:00 до 23:59).
   * По операционной системе устройства. Если вход был совершён с iOS, он учитывается в категории `Apple`; если с Android — в категории `Android`.

3. Расчёт количества уникальных пользователей в каждой группе. Пример расчёта: в определённом районе (H3) утром (категория `Утро`) в будний день (категория `Будние`) с устройств на iOS (категория `Apple`) заходило 150 уникальных пользователей.

## Обновление данных и историчность {#data-update-and-history}

* **Частота обновления**: данные обновляются один раз в месяц.
* **Историчность**: исторические данные доступны за последние 12 месяцев.

## Доступные территории {#available-territories}

Данные доступны для следующих стран:

* Азербайджан,
* Армения,
* Беларусь,
* Грузия,
* Казахстан,
* Кыргызстан,
* Россия,
* Узбекистан.

:::note[Ограниченная доступность на территории]

Данные могут быть доступны не на всей территории страны, а только в населённых пунктах, в которых присутствует компания <Brand />. Чтобы уточнить доступность данных, используйте [геофильтры](/pro/data/overview#geofilters) или обратитесь в [техническую поддержку](mailto:support.pro@2gis.ru).

:::

## Доступность данных {#data-availability}

* **Облако** (сервис [<Brand /> Про](https://pro.2gis.ru/)):

  * **Демо-тариф**: данные недоступны. Чтобы начать работать с данными, [смените тариф](/pro/prices) и запросите доступ к набору данных.
  * **Другие тарифы**: данные доступны по запросу.

* **On-Premise** (установка сервиса в [закрытом контуре](/on-premise-pro/architecture/pro-architecture)): данные недоступны.

## Начало работы {#getting-started}

![Активность аудитории](/img/pro/data-audience.png)

1. Перейдите на вкладку **Главная** или **Данные**.
2. Выберите набор данных **Активность аудитории**.
3. В окне **Настройки геофильтров** выберите до 50 территорий, для которых будет создана выборка данных. Вы можете использовать стандартные и собственные территории, а также строить полигоны, зоны доступности и окружности на карте. Подробнее см. в разделе [Геофильтры](/pro/data/overview#geofilters). После выбора территорий нажмите **Готово**.
4. На вкладке **Фильтры** настройте остальные [фильтры](#filters) при необходимости.

<br />

Данные будут отображены на карте в виде точек-центроидов: центров геометрий объектов. В результатах фильтрации доступны:

* Общее количество уникальных пользователей мобильного приложения. Подробнее об алгоритме расчёта уникальных пользователей см. в разделе [Как были получены данные](#data-collection).
* Диаграмма с распределением операционных систем на устройствах пользователей. Отображается, если выбраны обе операционные системы.
* Распределение активности пользователей (суммарного количества входов в приложение) в зависимости от времени суток. Отображается, если выбрано больше одного значения.

Чтобы сохранить выборку, нажмите **Создать слой** и настройте [визуализацию данных](#data-visualization).

## Фильтры {#filters}

Для набора данных **Активность аудитории** доступны следующие фильтры:

* **Год – месяц**: год и месяц для отображения данных. Вы можете выбрать несколько значений.
* **Тип дня недели**: `Будние` и `Выходные`.
* **Время суток**: `Утро` (с 06:00 до 11:59 в часовом поясе выбранной территории), `День` (с 12:00 до 17:59), `Вечер` (с 18:00 до 23:59) и `Ночь` (с 00:00 до 05:59).
* **Операционная система**: тип операционной системы на устройстве пользователя, на котором было открыто приложение: `Android` и `Apple`.
* **Активность аудитории**: количество входов в мобильное приложение.

Чтобы сбросить фильтр по параметру, нажмите значок ![Крестик](/img/pro/cross-icon.svg) рядом с именем фильтра. Чтобы сбросить все фильтры по параметрам, нажмите **Очистить фильтры** внизу вкладки.

Чтобы скрыть фильтр по территории, в окне **Настройки геофильтров** нажмите значок ![Глаз](/img/pro/hide-eye-icon.svg) рядом с именем геофильтра. Чтобы удалить геофильтр, нажмите значок ![Крестик](/img/pro/cross-icon.svg) рядом с его именем. Чтобы удалить все геофильтры, нажмите **Очистить фильтры**.

## Покупка данных {#data-purchasing}

Вы можете купить данные отдельно в виде выгрузки в следующих форматах:

* Табличные форматы:
   * `.csv`;
   * `.txt`;
   * `.json`;
* Форматы для хранения географических данных:
   * `.shp` (Shape);
   * `.gpkg` (GeoPackage);
   * `.geojson` (GeoJSON).

Чтобы узнать стоимость, доступные территории и приобрести данные, заполните форму на [dev.2gis.ru](https://dev.2gis.ru/data).
