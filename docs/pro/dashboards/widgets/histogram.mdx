# Гистограмма

Гистограмма показывает в виде вертикальных столбцов, как часто значение встречается в указанных диапазонах, а также отображает вид распределения значений. Чем выше столбец гистограммы, тем больше значений атрибута данных входит в соответствующий диапазон.

Используйте гистограмму, если необходимо показать распределение данных по фиксированным диапазонам, которые задаются с помощью размера шага.

Пример задачи: показать распределение зданий по количеству этажей:

<img src="/img/pro/widgets-histogram.png" alt="Гистограмма с распределением количества этажей" width="450" />

## Основные особенности {#main-features}

* На оси X отображаются диапазоны значений данных. В качестве атрибута группировки вы можете выбрать:

  * Числовые атрибуты: например, количество отзывов, площадь здания и другие.
  * Текстовые атрибуты (например, категорию фирм, назначение здания и другие) и атрибуты с типом `Дата` для группировки данных по диапазонам времени. Для этих типов атрибутов вместо гистограммы строится столбчатая диаграмма с количеством объектов в группах данных. Вы можете построить полноценную [столбчатую диаграмму](/pro/dashboards/widgets/column-chart), для которой можно выбрать агрегирующий атрибут и функцию агрегации.

* Вы можете настроить фиксированный размер диапазона (шага), например 10, 20, 50, 100 и другие. Например, для оценки распределения среднего чека используйте шаги по 100 или 200 рублей, а для распределения возраста или количества этажей в зданиях используйте шаги по 1 или 5. Если вы хотите вручную настроить диапазоны разных размеров, используйте [линейчатую](/pro/dashboards/widgets/bar-chart) или [столбчатую диаграмму](/pro/dashboards/widgets/column-chart).

* По умолчанию на гистограмме отображаются только первые 100 диапазонов. Чтобы увидеть все данные, при [создании виджета](#creating-a-widget) или его редактировании увеличьте диапазон в поле **Размер шага**.

* Вы можете нажимать на столбцы диаграммы, чтобы фильтровать данные по выбранному атрибуту, не изменяя при этом выборку данных. Подробнее см. в разделе [Фильтрация через виджеты](/pro/data/manage/filtering#via-widgets).

* Если на сцене дашборда применена фильтрация по территориям или атрибутам данных, в виджете отобразятся только значения, соответствующие фильтрам. Подробнее см. в инструкции [Фильтрация данных на сцене](/pro/data/manage/filtering).

## Создание виджета {#creating-a-widget}

<img src="/img/pro/widgets-histogram-create.png" alt="Пример создания гистограммы" width="650" />

1. Перейдите на вкладку **Дашборды**.

2. Откройте дашборд.

3. Откройте сцену с помощью стрелок ![Стрелка влево](/img/pro/left-arrow-icon.svg) и ![Стрелка вправо](/img/pro/right-arrow-icon.svg). Если в дашборде одна сцена, она откроется автоматически.

4. В верхнем меню дашборда нажмите значок ![Виджеты](/img/pro/widgets-icon.svg).

5. Выберите тип виджета **Гистограмма**.

6. Укажите обязательные параметры:

   * **Слой**: слой с данными, на основе которых строится диаграмма в виджете. Если вы работаете с [загруженными данными](/pro/data/files-upload), убедитесь, что они содержат хотя бы один атрибут.
   * **Атрибут группировки**: атрибут, на основе которого группируются данные.
   * **Размер шага**: фиксированный размер диапазонов с данными.

7. Настройте остальные [параметры](#parameters) виджета при необходимости.

8. Нажмите **Создать**.

Новый виджет автоматически добавится в выбранную сцену дашборда.

Доступные форматы:<ul><li>**Без форматирования**: разряды тысяч разделяются пробелами, дробная часть отделяется запятой.<br />Пример: `12345.6789 → 12 345,6789`.</li><li>**1,2345 → 1,235**: округление до тысячных (по правилу округления к ближайшему целому).<br />Пример: `12345.6789 → 12 345,679`.</li><li>**1,2345 → 1,23**: округление до сотых (по правилу округления к ближайшему целому).<br />Пример: `12345.6789 → 12 345,68`.</li><li>**1,2345 → 1,2**: округление до десятых (по правилу округления к ближайшему целому).<br />Пример: `12345.6789 → 12 345,7`.</li><li>**1,2345 → 1**: округление до целых (по правилу округления к ближайшему целому).<br />Пример: `12345.6789 → 12 346`.</li><li>**1 200 → 1 тыс.**: компактная запись числа с указанием разряда. Только для чисел от 1000.<br />Примеры: `12345 → 12 тыс.`, `1234567 → 1 млн`.</li><li>**1 256 → 1 260**: умное округление до десятков. Целая часть округляется до десятков, дробная отбрасывается. Только для чисел от 10.<br />Пример: `12345.6789 → 12 350`.</li><li>**1 256 → 1 300**: умное округление до сотен. Целая часть округляется до сотен, дробная отбрасывается. Только для чисел от 100.<br />Пример: `12345.6789 → 12 300`.</li><li>**1 257 → 1 000**: умное округление до тысяч. Целая часть округляется до тысяч, дробная отбрасывается. Только для чисел от 1000.<br />Пример: `12345.6789 → 12 000`.</li><li>**0,12345 → 12,345 %**: перевод в проценты. Максимальное количество знаков после запятой — 3.<br />Примеры: `0.12345 → 12,345%`, `1 → 100%`.</li><li>**1 200 000 → 1,2E6**: научный формат (экспоненциальная запись). <br />Примеры: `12345 → 1,235E4`, `1 → 1E0`.</li></ul> |
