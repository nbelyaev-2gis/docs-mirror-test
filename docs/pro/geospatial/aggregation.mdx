# Агрегация данных

**Агрегация** — это преобразование нескольких значений данных в одно результирующее значение с помощью функций. Доступны два вида агрегированных значений:

* Абсолютный показатель: сумма, среднее, минимальное значение, максимальное значение и другие. Пример использования: рассчитать общее количество жителей или средний рейтинг конкурентов в пределах полигона или 5-минутной зоны доступности.
* Удельный показатель: отношение двух абсолютных показателей. Пример использования: рассчитать отношение количества работающего населения к проживающему.

При агрегации данных создаётся новый слой со способом визуализации [Полигон](/pro/data/visualization/polygon). При расчёте абсолютного показателя слой содержит значения агрегирующего атрибута, рассчитанные по выбранной функции агрегации для каждого площадного объекта. При расчёте удельного показателя — отношения значений двух агрегирующих атрибутов. Чтобы посмотреть полученные значения в тултипе, наведите курсор на площадной объект (полигон или зону доступности):

<img src="/img/pro/aggregation-tooltip.png" alt="Тултип агрегации данных" width="300" />

## Создание слоя с агрегированными данными {#creating-aggregation}

1. [Создайте слой](/pro/dashboards/layers/create) с данными, которые вы хотите агрегировать.

   Например, если вы хотите агрегировать данные о жителях, создайте слой из набора данных [Население](/pro/data/built-in/population).

2. [Создайте слой](/pro/dashboards/layers/create) с площадными объектами: со способом визуализации [Полигон](/pro/data/visualization/polygon) или [Зоны доступности](/pro/geospatial/proximity).

   Оба слоя (с данными и с площадными объектами) должны находиться на одной сцене дашборда.

3. Перейдите на вкладку **Дашборды** и откройте дашборд.
4. Откройте сцену с помощью стрелок ![Стрелка влево](/img/pro/left-arrow-icon.svg) и ![Стрелка вправо](/img/pro/right-arrow-icon.svg). Если в дашборде одна сцена, она откроется автоматически.
5. В верхнем меню дашборда нажмите значок ![Слои](/img/pro/layers-icon.svg) и выберите слой с площадными объектами.
6. Нажмите **Агрегировать данные**.

   ![Кнопка Агрегировать данные](/img/pro/aggregation.png)

7. Настройте [параметры](#parameters) агрегации.
8. Чтобы добавить другой агрегирующий атрибут, нажмите **Добавить показатель**. Вы можете добавить не более 20 атрибутов в одном слое.
9. Нажмите **Создать**.

Будет создан новый слой, который автоматически добавится в выбранную сцену дашборда. При необходимости вы можете [настроить визуализацию](/pro/data/visualization/polygon) для слоя.

## Параметры {#parameters}

| Параметр | Описание |
| -------- | -------- |
| **Название нового слоя** | Имя нового слоя с агрегированными данными. |
| **Исходный слой** | Слой с площадными объектами. |
| **Вид показателя** | Способ агрегации данных:<ul><li>**Абсолютный показатель**: агрегирует данные по выбранному атрибуту.<br/><img src="/img/pro/aggregation-absolute-520.png" alt="Агрегация данных: абсолютный показатель" width="600" /></li><li>**Удельный показатель**: агрегирует данные по выбранному атрибуту и делит полученное значение на значение другого атрибута.<br/><img src="/img/pro/aggregation-specific-520.png" alt="Агрегация данных: относительный показатель" width="600" /></li></ul> |
| **Слой данных** | Слой с данными, которые содержат агрегирующий атрибут. Для вычисления удельного показателя вы можете выбрать разные слои. |
| **Атрибут** | Атрибут, на основе которого агрегируются данные и рассчитываются значения. |
| **Агрегация** | Функция, на основе которой рассчитываются значения агрегирующего атрибута:<ul><li>**Сумма**: сумма значений.</li><li>**Среднее**: среднее значение.</li><li>**Мин.**: минимальное значение.</li><li>**Макс.**: максимальное значение.</li><li>**Медиана**: медианное значение, которое равно или больше, чем 50% значений в группе.</li><li>**80-ый процентиль**: значение, которое равно или больше, чем 80% значений в группе.</li><li>**95-ый процентиль**: значение, которое равно или больше, чем 95% значений в группе.</li><li>**99-ый процентиль**: значение, которое равно или больше, чем 99% значений в группе.</li></ul> |
| **Название показателя** | Название агрегирующего атрибута, которое отображается в тултипе полигона, а также в карточке при нажатии на полигон на карте. При изменении названия показателя имя атрибута в наборе данных не изменится. |
| **Единица измерения**<br />(для удельных показателей)| Единица измерения агрегирующего атрибута. Отображается в тултипе полигона после значения атрибута. |
| **Форматирование** | Формат значения агрегирующего атрибута, которое отображается в тултипе. Доступные форматы:<ul><li>**Без форматирования**: разряды тысяч разделяются пробелами, дробная часть отделяется запятой.<br />Пример: `12345.6789 → 12 345,6789`.</li><li>**1,2345 → 1,235**: округление до тысячных (по правилу округления к ближайшему целому).<br />Пример: `12345.6789 → 12 345,679`.</li><li>**1,2345 → 1,23**: округление до сотых (по правилу округления к ближайшему целому).<br />Пример: `12345.6789 → 12 345,68`.</li><li>**1,2345 → 1,2**: округление до десятых (по правилу округления к ближайшему целому).<br />Пример: `12345.6789 → 12 345,7`.</li><li>**1,2345 → 1**: округление до целых (по правилу округления к ближайшему целому).<br />Пример: `12345.6789 → 12 346`.</li><li>**1 200 → 1 тыс.**: компактная запись числа с указанием разряда. Только для чисел от 1000.<br />Примеры: `12345 → 12 тыс.`, `1234567 → 1 млн`.</li><li>**1 256 → 1 260**: умное округление до десятков. Целая часть округляется до десятков, дробная отбрасывается. Только для чисел от 10.<br />Пример: `12345.6789 → 12 350`.</li><li>**1 256 → 1 300**: умное округление до сотен. Целая часть округляется до сотен, дробная отбрасывается. Только для чисел от 100.<br />Пример: `12345.6789 → 12 300`.</li><li>**1 257 → 1 000**: умное округление до тысяч. Целая часть округляется до тысяч, дробная отбрасывается. Только для чисел от 1000.<br />Пример: `12345.6789 → 12 000`.</li><li>**0,12345 → 12,345 %**: перевод в проценты. Максимальное количество знаков после запятой — 3.<br />Примеры: `0.12345 → 12,345%`, `1 → 100%`.</li><li>**1 200 000 → 1,2E6**: научный формат (экспоненциальная запись).<br />Примеры: `12345 → 1,235E4`, `1 → 1E0`.</li></ul> |
