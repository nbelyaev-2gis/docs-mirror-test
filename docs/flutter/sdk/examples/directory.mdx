# Справочник

Вы можете искать объекты в справочнике <Brand />, обрабатывать результаты поиска, а также использовать подсказки для уточнения поисковых запросов. В справочнике вы можете найти информацию по объектам следующих типов:

* компании и их филиалы;
* здания;
* парковки;
* остановки общественного транспорта и станции метро;
* дороги (улицы, перекрёстки, проезды);
* населённые пункты разного размера (страны, города, регионы, деревни, микрорайоны и т.д.);
* различные площадные объекты (парки, пляжи и т.д.).

Полный список доступных типов объектов см. в описании класса [ObjectType](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/ObjectType.html).

***

Чтобы начать работу со справочником:

1. Создайте [поисковый движок](#creating-a-search-engine).
2. Сформируйте [поисковый запрос](#create-search-query).
3. Получите, обработайте и отобразите на карте [результаты поиска](#get-search-results).

## Создание поисковика {#creating-a-search-engine}

Для поиска объектов в справочнике создайте объект [SearchManager](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchManager-class.html) и вызовите один из методов, который определяет режим работы справочника:

* [SearchManager.createOnlineManager()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchManager/createOnlineManager.html) — создаёт онлайн-справочник.
* [SearchManager.createOfflineManager()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchManager/createOfflineManager.html) — создаёт офлайн-справочник, работающий только с предзагруженными данными.
* [SearchManager.createSmartManager()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchManager/createSmartManager.html) — создаёт комбинированный справочник, работающий с онлайн-данными при наличии сети и с предзагруженными данными при отсутствии сети.

Пример создания поисковика с онлайн-справочником:

```dart
import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

final SearchManager searchManager = sdk.SearchManager.createOnlineManager(context);
```

:::note[Режимы работы справочника]

Некоторые методы работают только в определённом режиме справочника. Возможные ограничения указаны в описании методов.

:::

## Формирование поискового запроса {#create-search-query}

Чтобы отправить поисковый запрос, создайте объект [SearchQuery](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQuery-class.html) с помощью [SearchQueryBuilder](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder-class.html) и передайте его в метод [search()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchManager/search.html).

Поисковый запрос должен состоять из трёх логических компонентов:

* Запрос для поиска объекта (что нужно искать?). Вы можете сформулировать запрос с помощью одного из методов ниже:

  * Из текстовой строки с помощью метода [fromQueryText()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromQueryText.html). Вы можете сформулировать текстовый запрос для поиска конкретного объекта (`храм Василия Блаженного`) или для получения списка множества объектов по критерию (`магазины музыкальных инструментов`).
  * По идентификаторам рубрик с помощью метода [fromRubricIds()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromRubricIds.html) или в сочетании с текстовой строкой с помощью метода [fromQueryTextAndRubricIds()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromQueryTextAndRubricIds.html). Этот способ полезен для создания выборки объектов определённого типа.
  * По идентификатору:

    * Организации с помощью метода [fromOrgId()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromOrgId.html). Если у компании несколько филиалов, в ответе будут перечислены все точки.
    * Здания с помощью метода [fromBuildingId()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromBuildingId.html).
    * Любого объекта с помощью метода поискового движка [searchById()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchManager/searchById.html).

* Геоограничение поиска (где нужно искать?). Вы можете ограничить зону поиска с помощью одного из методов ниже:

  * Поиск внутри полигона: задайте координаты с помощью метода [setSpatialRestriction()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/setSpatialRestriction.html).

  * Поиск в прямоугольной области интереса: задайте координаты с помощью метода [setAreaOfInterest()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/setAreaOfInterest.html). Этот метод задаёт приоритетную зону поиска, но не ограничивает поиск строго: если внутри области интереса результаты не найдены, поиск продолжится за пределами области.

  * Поиск в радиусе вокруг точки: задайте центр поиска с помощью метода [setGeoPoint()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/setGeoPoint.html) или [fromGeoPoint()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromGeoPoint.html) и, если необходимо, радиус поиска с помощью метода [setRadius()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/setRadius.html).

    Метод [fromGeoPoint()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromGeoPoint.html) работает только в режиме поиска онлайн.

  * Произвольный запрос: если запрос для поиска объекта сформирован с помощью метода [fromQueryText()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromQueryText.html), вы можете в этом же тексте указать и геоограничение (`цветы у Бауманской`).

* (Необязательный компонент) Дополнительные ограничения результатов поиска с помощью одного или нескольких методов ниже:

  * Поиск объектов только определённого типа: перечислите интересующие вас типы объектов с помощью метода [setAllowedResultTypes()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/setAllowedResultTypes.html) (например, только здания).
  * Фильтрация результатов: задайте критерий фильтрации с помощью метода [setDirectoryFilter()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/setDirectoryFilter.html) (например, по времени работы).
  * Сортировка результатов: задайте критерий сортировки с помощью метода [setSortingType()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/setSortingType.html) (например, по рейтингу).
  * Количество результатов на странице: задайте ограничение с помощью метода [setPageSize()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/setPageSize.html). Если вы используете демо-ключ с доступом к Places API, максимальное количество результатов на странице поисковой выдачи — 10.
  * Номер страницы поисковой выдачи: по умолчанию возвращается первая страница. Чтобы получить следующие страницы, используйте метод [fetchNextPage()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Page/fetchNextPage.html). Если вы используете демо-ключ с доступом к Places API, максимально возможно загрузить до 5 страниц.
  * Локаль для поискового запроса: задайте язык и регион с помощью метода [setLocale()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/setLocale.html).

### Примеры

* Найти рестораны итальянской кухни в Пресненском районе Москвы, которые открыты сейчас, с сортировкой по рейтингу:

  ```dart
  import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

  // Создаём фильтр по времени работы
  final filter = sdk.DirectoryFilter(
      workTime: sdk.WorkTimeFilter.isOpenNow(const sdk.IsOpenNow()), // оставить только открытые сейчас объекты
      dynamic_: [],
  );

  final searchQuery = sdk.SearchQueryBuilder
      .fromQueryText("рестораны итальянской кухни в Пресненском районе Москвы")
      .setDirectoryFilter(filter) // фильтрация по времени работы
      .setSortingType(sdk.SortingType.byRating) // сортировка по рейтингу
      .setPageSize(10) // максимум 10 объектов на странице результатов
      .build();
  ```

* Найти все парковки в радиусе 1 км с сортировкой по удалённости:

  ```dart
  import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

  final searchQuery = sdk.SearchQueryBuilder
      .fromQueryText('Парковки')
      .setAllowedResultTypes([sdk.ObjectType.parking]) // только парковки
      .setGeoPoint(const sdk.GeoPoint(latitude: sdk.Latitude(59.936), longitude: sdk.Longitude(30.351))) // центр поиска
      .setRadius(const sdk.Meter(1000)) // радиус поиска
      .setSortingType(sdk.SortingType.byDistance) // сортировка по удалённости
      .build();
  ```

* Найти все населённые пункты (города, деревни, посёлки и т.д.) внутри полигона:

  ```dart
  import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

  // Создаём область поиска в виде полигона
  final polygon = [
    const sdk.GeoPoint(
      latitude: sdk.Latitude(55.751244),
      longitude: sdk.Longitude(37.618423),
    ),
    const sdk.GeoPoint(
      latitude: sdk.Latitude(55.751244),
      longitude: sdk.Longitude(37.618423),
    ),
    const sdk.GeoPoint(
      latitude: sdk.Latitude(55.751244),
      longitude: sdk.Longitude(37.618423),
    ),
    const sdk.GeoPoint(
      latitude: sdk.Latitude(55.751244),
      longitude: sdk.Longitude(37.618423),
    ),
  ];

  final searchQuery = sdk.SearchQueryBuilder.fromQueryText('название города')
      .setAllowedResultTypes([
        sdk.ObjectType.admDivCity,
        sdk.ObjectType.admDivSettlement,
      ]) // только города и мелкие населённые пункты
      .setSpatialRestriction(polygon) // область поиска
      .build();
  ```

* Найти все объекты, которые оказывают услуги печати документов, в области интереса:

  ```dart
  import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

  // Создаём прямоугольную область интереса по координатам двух углов
  const areaOfInterest = sdk.GeoRect(
    southWestPoint: sdk.GeoPoint(
      latitude: sdk.Latitude(59.931),
      longitude: sdk.Longitude(30.344),
    ), // юго-западный угол
    northEastPoint: sdk.GeoPoint(
      latitude: sdk.Latitude(59.936),
      longitude: sdk.Longitude(30.351),
    ), // северо-восточный угол
  );

  final searchQuery =
      sdk.SearchQueryBuilder.fromQueryText('печать документов')
          .setAreaOfInterest(areaOfInterest)
          .build();
  ```

* Найти все компании по адресу «Новосибирск, площадь Карла Маркса, 7»:

  ```dart
  import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

  final searchQuery = sdk.SearchQueryBuilder
      .fromQueryText('Новосибирск, площадь Карла Маркса, 7')
      .setAllowedResultTypes([sdk.ObjectType.branch]) // только компании
      .setPageSize(10) // максимум 10 объектов на странице результатов
      .build();
  ```

* Найти все филиалы компании по известному идентификатору:

  ```dart
  import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

  const id = 4504136498310300;

  final searchQuery = sdk.SearchQueryBuilder
      .fromOrgId(const sdk.OrgId(id))
      .build();
  ```

* Найти конкретный объект по известному идентификатору:

  ```dart
  import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

  const id = 70000001035164789;

  searchManager.searchById('id');
  ```

### Геокодирование {#geocoding}

С помощью SDK вы можете решать задачи геокодирования: определять координаты объекта на карте по его адресу ([прямое геокодирование](#direct-geocoding)) и наоборот, определять адрес объекта на карте по его координатам ([обратное геокодирование](#reverse-geocoding)).

#### Прямое геокодирование {#direct-geocoding}

Чтобы получить координаты объекта по его адресу, сформируйте поисковый запрос, указав следующие данные:

* Адрес в текстовом запросе с помощью метода [fromQueryText()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromQueryText.html).

  Для более точных результатов укажите в тексте город (посёлок, деревню), где выполняется поиск. Название небольшого населённого пункта (например, деревни) рекомендуется указывать вместе с названием области и другими объединениями, к которым он относится (например, сельским или городским поселением).

* (Рекомендуется) Тип объекта, координаты которого нужно получить, с помощью метода [setAllowedResultTypes()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/setAllowedResultTypes.html).

Например, чтобы получить координаты здания по адресу «Москва, ул. Тверская 19а»:

```dart
import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

final searchQuery = sdk.SearchQueryBuilder
    .fromQueryText('Москва, ул. Тверская 19а')
    .setAllowedResultTypes([sdk.ObjectType.building])
    .build();
```

В [результате поиска](#get-search-results) вы получите объект справочника [DirectoryObject](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObject-class.html). Координаты объекта будут представлены в поле `markerPosition` в виде объекта [GeoPointWithElevation](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/GeoPointWithElevation-class.html). Пример:

```dart
markerPosition:
    GeoPointWithElevation(
        latitude: Latitude(value: 55.7659),
        longitude: Longitude(value: 37.602827),
        elevation: Elevation(value: 18.0),
    ),
```

Подробнее о другой информации в результатах поиска см. в разделе [Структура данных объекта](#object-data-structure).

#### Обратное геокодирование {#reverse-geocoding}

Чтобы получить адрес объекта по его координатам, сформируйте поисковый запрос, указав координаты объекта с помощью метода [fromGeoPoint()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromGeoPoint.html).

Например, найти адрес объекта по координатам `55.7659, 37.602827`:

```dart
import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

final searchQuery = sdk.SearchQueryBuilder
    .fromGeoPoint(const sdk.GeoPoint(latitude: sdk.Latitude(55.7659), longitude: sdk.Longitude(37.602827)))
    .build();
```

В [результате поиска](#get-search-results) вы получите объекты справочника [DirectoryObject](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObject-class.html). Адрес каждого объекта будет представлен в поле `address` в виде объекта [Address](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Address-class.html). Пример:

```dart
address:
    Address(
        drillDown: [
        AddressAdmDiv(type: 'country', name: 'Россия'),
        AddressAdmDiv(type: 'region', name: 'Москва'),
        AddressAdmDiv(type: 'city', name: 'Москва'),
        AddressAdmDiv(type: 'district', name: 'Тверской'),
        ],
        components: [
        AddressComponent(
            AddressStreet(
            street: 'Тверская улица',
            number: '19а',
            fiasCode: null,
            ),
        ),
        ],
        buildingName: null,
        buildingId: BuildingId(value: 4504235282747324),
        postCode: '125009',
        buildingCode: null,
        fiasCode: null,
        addressComment: '1-2 этаж',
    ),
```

Подробнее о другой информации в результатах поиска см. в разделе [Структура данных объекта](#object-data-structure).

### Изменение параметров поиска {#modify-search-params}

Вы можете изменять или дополнять параметры уже созданного поискового запроса. Для этого создайте новый объект [SearchQuery](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQuery-class.html), укажите существующий запрос с помощью метода [fromQuery()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromQuery.html) и укажите параметры, которые нужно дополнительно применить. Например, изменить тип сортировки результатов при поиске парковок:

```dart
import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

// Создаём изначальный запрос
final searchQuery = sdk.SearchQueryBuilder
    .fromQueryText('Парковки')
    .setAllowedResultTypes([sdk.ObjectType.parking])
    .setGeoPoint(const sdk.GeoPoint(latitude: sdk.Latitude(59.936), longitude: sdk.Longitude(30.351))) // центр поиска
    .setRadius(const sdk.Meter(1000))
    .setSortingType(sdk.SortingType.byDistance) // сортировка по удалённости
    .build();

// Создаём новый запрос с другим типом сортировки
final searchQueryUpdated = sdk.SearchQueryBuilder
    .fromQuery(searchQuery) // указание на изначальный запрос
    .setSortingType(sdk.SortingType.byRating) // сортировка по рейтингу
    .build();
```

Остальные параметры изначального запроса сохраняются.

## Получение результатов поиска {#get-search-results}

Вызов метода [search()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchManager/search.html) возвращает отложенный результат [SearchResult](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchResult-class.html), содержащий список найденных объектов ([DirectoryObject](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObject-class.html)), разделенный на страницы:

```dart
final SearchResult result = await widget._searchManager.search(searchQuery).value;

final Page? searchPage = result.firstPage;
```

Чтобы получить следующую страницу результатов поиска, вызовите метод страницы [fetchNextPage()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Page/fetchNextPage.html), который вернёт отложенный результат [Page](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Page-class.html):

```dart
import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

final Page? nextPage = await searchPage?.fetchNextPage()?.value;
```

### Отображение результатов на карте {#display-search-results}

Координаты всех найденных объектов возвращаются в поле `itemMarkerInfos` результата поиска [SearchResult](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchResult-class.html) в виде списка элементов [ItemMarkerInfo](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/ItemMarkerInfo-class.html). Список может содержать не более 15000 элементов.

Чтобы отобразить на карте маркеры всех найденных объектов:

1. Подготовьте список объектов [Marker](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/ItemMarkerInfo-class.html). Чтобы задать позицию маркера (параметр `position`), используйте координаты из поля [ItemMarkerInfo.geoPoint](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/ItemMarkerInfo-class.html).
2. Добавьте готовый набор маркеров на карту с помощью метода `addObjects()` менеджера объектов [MapObjectManager](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapObjectManager-class.html). Подробнее см. в инструкции [Добавление нескольких объектов на карту](/flutter/sdk/examples/map#add-multiple-objects).

```dart
import 'package:dgis_mobile_sdk/dgis.dart';

  Future<void> displaySearchResultMarkers(sdk.Map map, sdk.SearchResult searchResult) async {
    // Получаем список маркеров из результатов поиска
    final loader = sdk.ImageLoader(sdkContext);
    final markerInfos = await searchResult.itemMarkerInfos.value;
    if (markerInfos == null) return;

   // Создаём менеджер объектов для добавления маркеров на карту
    final mapObjectManager = sdk.MapObjectManager(map);

    // Подготавливаем список маркеров
    final markers = markerInfos.map((itemMarkerInfo) async {
      final position = itemMarkerInfo.geoPoint;

      return sdk.Marker(
        sdk.MarkerOptions(
          position: position,
          icon: await loader.loadPngFromAsset('assets/icons/marker_icon.png', 100, 100),
        ),
      );
    }).whereType<sdk.Marker>().toList();

    // Добавляем маркеры на карту
    mapObjectManager.addObjects(markers);
  }
```

### Структура данных объекта {#object-data-structure}

Результаты поиска по справочнику представлены в виде списка объектов [DirectoryObject](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObject-class.html) с наборами свойств. В зависимости от типа объекта часть свойств может не содержать значений.

:::note[Доступ к данным]

Получение некоторой информации об объектах доступно при дополнительной настройке ключа за отдельную плату: см. описания полей объектов [DirectoryObject](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObject-class.html), [Address](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Address-class.html) и [ItemMarkerInfo](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/ItemMarkerInfo-class.html). Чтобы обновить настройки вашего ключа доступа, обратитесь в [службу поддержки <Brand />](mailto:api@2gis.ru).

:::

* Основные свойства для классификации объекта:

  * Тип объекта (`types`) из [ObjectType](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/ObjectType.html). Один объект может относиться к нескольким типам (например, ТЦ Сан Сити — это одновременно и филиал организации и здание). В этом случае все типы будут перечислены в списке, где первый элемент — основной тип объекта.
  * Название объекта (`title`) в зависимости от его типа: название организации, достопримечательности, географического объекта. Для жилых домов без названия — адрес.
  * Подтип объекта (`subtitle`) для уточнения. Например, кофейня как подтип организации или жилой дом как подтип здания.
  * Описание объекта (`description`).
  * Категории, к которым относится объект (`rubricIds`).
  * Идентификатор организации в справочнике и информация о ней (`orgInfo`). Для компаний с множеством филиалов — информация о головной организации.
  * (Данные по запросу) Объединение объектов разного типа в одной карточке справочника (`group`). Подробнее см. [ниже](#hybrid-objects).

* Уникальный идентификатор объекта в справочнике (`id`). Для компаний с множеством филиалов — идентификатор конкретного филиала.

* Географические свойства:

  * Координаты для размещения маркера на карте (`markerPosition`).
  * Полный адрес объекта (`address`). Некоторые компоненты адреса доступны только по запросу: см. описание объекта [Address](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Address-class.html).
  * (Данные по запросу) Информация об этаже здания, на котором расположен объект (`levelId` и `buildingLevels`). Актуально для компаний, которые расположены на определённом этаже многоэтажного здания.
  * (Данные по запросу) Информация о входах в объект (`entrances`) с координатами и другими данными. Актуально как для компаний и зданий, так и для других объектов с физически обозначенными входами (например, парков).
  * (Данные по запросу) Дополнительная информация для уточнения адреса (`titleAddition`). Например, номер подъезда или номер квартиры.

* Время работы:

  * Смещение локального времени объекта от UTC в виде временной метки (`timeZoneOffset`). Например, `03:00:00` для часового пояса UTC+3.
  * Время работы (`openingHours`) в виде списка временных промежутков или флага круглосуточной работы. Актуально для компаний.
  * Текущий статус работы (`workStatus`).

* Прочие данные:

  * (Данные по запросу) Информация о торговой лицензии организации (`tradeLicense`).
  * Контакты для связи с организацией (`contactInfos`): номер телефона, e-mail, ссылки на сайт и соцсети и другое.
  * Рейтинг объекта на основе отзывов пользователей (`reviews`).
  * Дополнительные свойства парковок (`parkingInfo`).
  * Дополнительные свойства электрозаправок (`chargingStation`).
  * Информация о здании (`buildingInfo`).

#### Объединение объектов {#hybrid-objects}

Некоторые геообъекты в справочнике могут быть представлены в виде группы объектов разного типа. Например, здание суда — это одновременно и отдельно стоящее здание, и организация внутри этого здания, то есть, два разных [DirectoryObject](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObject-class.html) с характеристиками здания и организации соответственно. Но поскольку эти `DirectoryObject` относятся к одному и тому же геообъекту, в их структуре данных содержатся ссылки друг на друга.

Информация о связанных объектах содержится в поле `group` в виде списка элементов [GroupItem](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/GroupItem-class.html). Для каждого связанного объекта представлен его тип и идентификатор ([DgisObjectId](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DgisObjectId-class.html)), который вы можете в дальнейшем использовать для обращения к этому объекту.

:::note[Доступ к данным]

Доступ к данным в поле `group` предоставляется при дополнительной настройке ключа за отдельную плату. Чтобы обновить настройки вашего ключа доступа, обратитесь в [службу поддержки <Brand />](mailto:api@2gis.ru).

:::

Пример наполнения поля `group` с одним связанным объектом:

```dart
group: [
    GroupItem(
    id: DgisObjectId(objectId: 70030076538159915, entranceId: 0),
    type: ObjectType.attraction,
    ),
],
```

## Поисковые подсказки {#search-suggestions}

Вы можете формировать подсказки для пользователей при текстовом поиске объектов. Для этого создайте объект [SuggestQuery](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SuggestQuery-class.html) с помощью [SuggestQueryBuilder](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SuggestQueryBuilder-class.html) и передайте его в метод [suggest()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchManager/suggest.html):

```dart
import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

final SuggestQuery suggestQuery = sdk.SuggestQueryBuilder.fromQueryText(query).build();

final SuggestQuery = await searchManager.suggest(suggestQuery).value;
```

Вызов вернёт отложенный результат [SuggestResult](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SuggestResult-class.html), содержащий список подсказок ([Suggest](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Suggest-class.html)). Подробнее о формировании подсказок см. в документации [Suggest API](/api/search/suggest/overview).

Когда пользователь выбирает одну из предложенных подсказок, вы можете настроить реакцию на это событие с помощью обработчика [SuggestHandler](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SuggestHandler-class.html) одного из следующих типов:

* [SuggestObjectHandler](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SuggestObjectHandler-class.html) — возвращает объект справочника [DirectoryObject](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObject-class.html).
* [PerformSearchHandler](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/PerformSearchHandler-class.html) — возвращает объект поискового запроса [SearchQuery](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQuery-class.html). Вы можете использовать этот объект для дальнейшего поиска: передайте его в метод [SearchManager.search()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchManager/search.html) или создайте на его основе новый [SearchQuery](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQuery-class.html) с дополнительными параметрами с помощью метода [fromQuery()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromQuery.html) (подробнее см. в разделе [Изменение параметров поиска](#modify-search-params)).
* [IncompleteTextHandler](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/IncompleteTextHandler-class.html) — возвращает автоматически дополненный текст подсказки, который можно использовать для дальнейшего формирования поискового запроса с помощью метода [fromQueryText()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryBuilder/fromQueryText.html).

Пример функции для применения всех типов обработчика:

```dart
void handleSuggestHandler(sdk.SuggestHandler handler) {
    handler.match<void>(
      objectHandler: (sdk.SuggestObjectHandler? value) {
          print('Вернуть объект справочника: {value.item}');
      },
      performSearchHandler: (sdk.PerformSearchHandler? value) {
          print('Вернуть объект поискового запроса: {value.searchQuery}');
      },
      incompleteTextHandler: (sdk.IncompleteTextHandler? value) {
          print('Вернуть дополненный текст подсказки: ${value?.queryText}');
      },
    );
}
```

## История поиска {#search-history}

Вы можете работать с историей поиска с помощью [SearchHistory](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory-class.html).

В истории поиска могут храниться элементы двух типов: объекты справочника ([DirectoryObject](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObject-class.html)) и поисковые запросы ([SearchQueryWithInfo](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryWithInfo-class.html)).

### Добавление элементов в историю поиска {#add-elements-to-history}

Вы можете добавить элементы в историю поиска в виде объектов [SearchHistoryItem](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistoryItem-class.html) по одному или списком. Порядок элементов в списке сохранится при добавлении в историю.

1. Создайте объект [SearchHistoryItem](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistoryItem-class.html) и передайте ему один из следующих объектов в зависимости от типа добавляемого элемента:

   * Для поискового запроса: [SearchQueryWithInfo](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchQueryWithInfo-class.html) с указанием нужного поискового запроса в параметре `searchQuery`. Дополнительно `SearchQueryWithInfo` может содержать заголовок и подзаголовок, которые отобразятся в выдаче истории поиска.

     ```dart
     // Создаём объект SearchQueryWithInfo
     final searchQueryWithInfo = sdk.SearchQueryWithInfo(
         searchQuery, // готовый поисковый запрос
         "Nearby Cafes",
         "Restaurants and cafes nearby",
     );

     final searchHistoryItem = sdk.SearchHistoryItem.searchQuery(searchQueryWithInfo);
     ```

     * Для объекта справочника: [DirectoryObject](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObject-class.html).

     ```dart
     final searchHistoryItem = sdk.SearchHistoryItem.directoryObject(directoryObject);
     ```

2. Создайте объект [SearchHistory](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory-class.html):

   ```dart
   final searchHistory = sdk.SearchHistory(sdkContext);
   ```

3. Добавьте элементы в историю поиска по одному или списком с помощью методов [addItem()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory/addItem.html) или [addItems()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory/addItems.html) объекта [SearchHistory](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory-class.html) соответственно:

   * Добавить один элемент:

     ```dart
     searchHistory.addItem(searchHistoryItem);
     ```

   * Добавить список элементов:

     ```dart
     searchHistory.addItems(searchHistoryItemsList);
     ```

Если в истории поиска уже существует добавленный элемент, более старый дубликат будет удалён.

### Отображение истории поиска {#show-history-page}

Чтобы показать страницу поиска со списком элементов, создайте объект [SearchHistoryPage](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistoryPage-class.html) и передайте его в метод [items()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory/items.html). Дополнительно вы можете настроить следующие параметры:

* Ограничить количество элементов на странице (параметр `limit`). Значение по умолчанию — 100.
* Задать смещение относительно начала списка (параметр `offset`): сколько элементов с начала списка пропустить. Значение по умолчанию — 0 (смещение отсутствует, список отображается с самого начала).
* Отфильтровать список по типу элементов (параметр `filter`). Доступные фильтры перечислены в [SearchHistoryFilter](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistoryFilter.html). Например, чтобы показывать в истории только поисковые запросы, используйте значение `searchQuery`. По умолчанию фильтрация отсутствует.

```dart
// Создаём объект SearchHistory
final searchHistory = SearchHistory(sdkContext);

// Создаём страницу истории с лимитом 10 элементов и смещением 0
final page = sdk.SearchHistoryPage(
    limit: 10,        // Максимум 10 элементов на странице
    filter: sdk.SearchHistoryFilterEnumSet(),    // Без фильтров
);

// Получаем элементы страницы истории
searchHistory.items(page).then((result) {
    for (final item in result.items) {
    // Логируем каждый элемент истории
    print('Элемент истории: $item');
    }
});
```

Элементы на странице упорядочены по времени добавления в историю (от новых к старым).

### Очистка истории поиска {#clean-history}

Чтобы убрать отдельные элементы из истории поиска:

* Чтобы убрать один элемент, используйте метод [removeItem()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory/removeItem.html) объекта [SearchHistory](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory-class.html) и передайте ему нужный объект [SearchHistoryItem](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistoryItem-class.html). Подробнее о наполнении `SearchHistoryItem` см. в инструкции [Добавить элементы в историю поиска](#add-elements-to-history).

  ```dart
  searchHistory.removeItem(searchHistoryItem);
  ```

* Чтобы убрать несколько элементов, используйте метод [removeItems()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory/removeItems.html) объекта [SearchHistory](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory-class.html) и передайте ему список нужных объектов [SearchHistoryItem](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistoryItem-class.html). Подробнее о наполнении `SearchHistoryItem` см. в инструкции [Добавить элементы в историю](#add-elements-to-history).

  ```dart
  searchHistory.removeItems(searchHistoryItemsList);
  ```

Чтобы очистить историю поиска полностью, используйте метод [clear()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory/clear.html) объекта [SearchHistory](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory-class.html):

```dart
searchHistory.clear();
```

### Подписка на изменения истории {#subscribe-to-history-changes}

Чтобы отслеживать изменения истории поиска (добавление и удаление элементов), вы можете подписаться на канал [onHistoryChanged](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SearchHistory/onHistoryChanged.html):

```dart
final searchHistory = SearchHistory(sdkContext);

searchHistory.onHistoryChanged.listen((changeType) {
    switch (changeType) {
        case sdk.ChangeType.add:
            print('Элемент добавлен в историю');
            break;
        case sdk.ChangeType.remove:
            print('Элемент удалён из истории');
            break;
      }
});
```

## Информация о подъездах в справочнике

Вы можете искать адреса в справочнике с точностью до квартиры или подъезда.

Например, по запросу `"Томск Кирова 17 кв 5"` вы получите объект, который содержит информацию с точностью до подъезда.

[DgisObjectId](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DgisObjectId-class.html) содержит два идентификатора:

* [objectId](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DgisObjectId/objectId.html) — стабильный числовой идентификатор объекта;
* [entranceId](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DgisObjectId/entranceId.html) — стабильный числовой идентификатор входа/подъезда для объекта.

Если [entranceId](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DgisObjectId/entranceId.html) отличен от нуля, то результатом поиска является не просто дом, а конкретный подъезд в доме.

Если необходимо нарисовать маркер на конкретном подъезде или взять его координаты для построения маршрута, не используйте [markerPosition](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObject/markerPosition.html). В этом свойстве будет располагаться позиция, относящаяся к маркеру дома. Для получения позиции подъезда используйте информацию из [entrances](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObject/entrances.html):

```dart
import 'package:dgis_mobile_sdk_map/dgis.dart' as sdk;

sdk.GeoPoint? getMarkerPosition(sdk.DirectoryObject directoryObject) {
  final int entranceId = directoryObject.id?.entranceId ?? 0;
  if (entranceId != 0) {
    return directoryObject.entrances
            .firstWhere((entrance) => entrance.id.entranceId == entranceId)
            .geometry
            ?.entrancePoints
            .firstOrNull ??
        directoryObject.markerPosition?.point;
  }
  return null;
}
```
