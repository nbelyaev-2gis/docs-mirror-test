# Релизы

## 13.6.0 (2026-06-22)

### 🌟 Новые возможности

* Новый класс [MapController](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapController-class.html):

  * [MapController](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapController-class.html) – класс для управления картой, объединяющий [Map](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Map-class.html), [MapRenderer](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapRenderer-class.html) и [MapGestureRecognizer](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapGestureRecognizer-class.html).
  * [MapControllerOptions](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapControllerOptions-class.html) – класс с параметрами создания [MapController](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapController-class.html).
  * [MapController.create](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapController/create.html) – статический метод для асинхронного создания [MapController](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapController-class.html) на основе `Context` и [MapControllerOptions](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapControllerOptions-class.html).
  * [MapController.map](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapController/map.html) – свойство для получения объекта [Map](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Map-class.html) из [MapController](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapController-class.html).
  * [MapController.renderer](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapController/renderer.html) – свойство для получения объекта [MapRenderer](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapRenderer-class.html) из [MapController](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapController-class.html).
  * [MapController.gestureRecognizer](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapController/gestureRecognizer.html) – свойство для получения объекта [MapGestureRecognizer](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapGestureRecognizer-class.html) из [MapController](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/MapController-class.html).

* [CameraZoomRestrictions.minZoom](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/CameraZoomRestrictions/minZoom.html) – значение по умолчанию изменено с `0` на `2`.

### 🛠 Исправления

* Механизм восстановления автонаклона доработан так, чтобы он работал независимо от других перелётов.
* Исправлена ошибка, вызывавшая сбой приложения из-за некорректного срабатывания `assert` в `libflutter.so` (`assert.cc:54`).
* Улучшена потокобезопасность удаления iOS-текстуры Flutter SDK; также добавлена приостановка слоя рендеринга Flutter iOS при переходе приложения в неактивное состояние.
* Исправлен [WorkStatus](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/WorkStatus-class.html) для расписаний, работающих почти круглосуточно.
* Исправлена ошибка, вызывавшая сбой приложения при увеличении масштаба карты на Samsung Tab A8 с Android 14.
* Исправлена ошибка, из-за которой не работали повторные добавления и удаления дорожных событий через [RoadEventManager](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/RoadEventManager-class.html).
* Добавлена корректная логика расчёта и заполнения полей `ageMs` и `lastSeenMs` для сотовых вышек в модуле `Radar` с учётом текущего времени и версии Flutter Android API; также исправлена очистка устаревших значений.

* [SuggestQueryBuilder.setTerritoryOfInterest()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/SuggestQueryBuilder/setTerritoryOfInterest.html) – метод для задания нестрогого ограничения области для поисковых подсказок: если объект не попадает в область, он не отбрасывается, но его ранг в результатах поиска понижается.

* [PackedSearchQuery.territoryOfInterest](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/PackedSearchQuery/territoryOfInterest.html) – свойство для нестрогого ограничения области поиска объектов: если объект не попадает в область, он не отбрасывается, но его ранг в результатах поиска понижается.

## 13.3.0 (2026-03-21)

### 🌟 Новые возможности

* Теперь вы можете создать и скачать файл ключа в Менеджере Платформы: см. [Получение ключей доступа](/flutter/sdk/start#get-api-key).
* В манифесте ключа, передаваемого в метод [DGis.initialize()](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DGis/initialize.html), теперь необязательно наличие заполненного `restrictions/app_id`, если не планируется работа с офлайн-данными (`offlineRouting`, `offlineMap`, `offlineDirectory`). В противном случае наличие `restrictions/app_id` обязательно.
* Для каждого класса добавлена обёртка `Message`, чтобы передавать объекты этих классов в изоляты. В каждом классе также добавлены методы для получения обёрток и фабрики для формирования объектов на их основе.
* [DirectoryObjectWidget](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObjectWidget-class.html) — UI-элемент полной карточки объекта справочника. Чтобы настроить элемент, используйте [DirectoryObjectWidgetTheme](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObjectWidgetTheme-class.htm). Чтобы кастомизировать внутренние UI-элементы, используйте [DirectoryObjectWidgetBuilder](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/DirectoryObjectWidgetBuilder-class.html).
* При передаче пустого идентификатора этажа в параметр [IndoorControlModel.activeLevelIndex](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/IndoorControlModel/activeLevelIndex.html) этажный план здания будет скрываться.
* [Closed](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Closed-class.html) — класс, представляющий статус `Закрыто` у объекта справочника с информацией о причине закрытия и времени до открытия.
* [ClosedReason](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/ClosedReason.html) — причины установки статуса `Закрыто` у объекта справочника.
* [ClosingHours](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/ClosingHours-class.html) — класс с информацией о нерабочих интервалах и причине закрытия объекта справочника.
* [ClosingSoon](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/ClosingSoon-class.html) — класс, представляющий статус `Скоро закроется` у объекта справочника с информацией о причине и времени закрытия.
* [OpenStatus](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/OpenStatus-class.html) — класс, описывающий детализированный статус работы объекта справочника в данный момент (`Открыто`, `Скоро откроется`, `Скоро закроется`, `Закрыто`).
* [Opened](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Opened-class.html) — класс, представляющий статус `Открыто` у объекта справочника с информацией о времени до закрытия.
* [OpeningSoon](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/OpeningSoon-class.html) — класс, представляющий статус `Скоро откроется` у объекта справочника с информацией о времени до открытия.
* [OpeningHours.weekClosingHours](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/OpeningHours/weekClosingHours.html) — свойство для хранения нерабочих интервалов объекта справочника по каждому дню недели.
* [OpeningHours.isOpen24x7](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/OpeningHours/isOpen24x7.html) — свойство для признака круглосуточной работы объекта справочника.
* [WorkStatus.openStatus](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/WorkStatus/openStatus.html) — свойство с детальным статусом `Открыто` у объекта справочника.
* [WorkStatus.openStatusHint](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/WorkStatus/openStatusHint.html) — свойство текстовой подсказки о статусе `Открыто` у объекта справочника.
* [WorkStatus.scheduleHint](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/WorkStatus/scheduleHint.html) — свойство текстовой подсказки о расписании работы объекта справочника.
* [WorkStatus.breakHint](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/WorkStatus/breakHint.html) — свойство текстовой подсказки о текущем или ближайшем перерыве в работе объекта справочника.

* Функция `expandPoint()` удалена. Используйте расширение [expandPoint](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/GeoRectExpandPoint/expandPoint.html).

### 🛠 Исправления

* Оптимизирована отрисовка карты на Flutter SDK для iOS: уведомление о готовности текстуры теперь отправляется в main-потоке, только если приложение активно.
* Запросы тайлов пробок через источник [TrafficSource](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/TrafficSource-class.html) теперь компонуются для оптимизации количества запросов.
* Оптимизирована отрисовка тайлов карты, чтобы не нагружать все доступные ядра процессора.
* Увеличен уровень масштабирования для растровых тайлов, добавляемых через источник [RasterTileSource](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/RasterTileSource-class.html), до 16.
* Исправлена ошибка при добавлении анимированной модели [ModelMapObject](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/ModelMapObject-class.html), из-за которой не перезапускались анимации и не поворачивались модели к камере.
* Исправлена работа канала готовности карты [Map.dataLoadingStateChannel](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Map/dataLoadingStateChannel.html) в иммерсивном режиме (когда для [Map.graphicsPreset](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Map/graphicsPreset.html) установлено значение [GraphicsPreset.immersive](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/GraphicsPreset.html)).
* Исправлено поведение значения [TextStyle.textPlacement](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/TextStyle/textPlacement.html) в свойстве [Marker.textStyle](https://pub.dev/documentation/dgis_mobile_sdk_full/latest/dgis_mobile_sdk/Marker/textStyle.html): теперь при изменении значения корректно меняется положение текста относительно маркера.
