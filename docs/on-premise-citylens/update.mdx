# Обновление <Brand /> Ситискан

## Обновление CityLens Routes UI {#update-citylens-ui}

1. Получите новые артефакты для установки.

   1. Укажите в конфигурационном файле 2GIS CLI:

      * Новую версию компонента `citylens` (см. [Релизы <Brand /> Ситискан](/on-premise-citylens/releases)).
      * Версии компонентов `core`, `api-platform` и `pro`, которые требуются для установки новой версии <Brand /> Ситискан (см. подраздел **Зависимости** в описании релизов <Brand /> Ситискан).

   2. [Выполните команду загрузки](/on-premise-api-platform/preparation#fetch-artifacts).

      Дополнительные настройки см. в [справке по командам и аргументам 2GIS CLI](/on-premise-api-platform/overview/2gis-cli/help).

2. Скопируйте путь к новому файлу манифеста из консольного вывода команды и подставьте его в параметр `dgctlStorage.manifest` в конфигурационном файле `values-citylens-ui.yaml`. Подробнее см. в [руководстве по установке](/on-premise-citylens/install/installation#install-citylens-ui).

   :::note[Выбор манифеста]

   Если при очередном обновлении сервиса <Brand /> Ситискан вы не хотите импортировать новые данные, **используйте тот же манифест, что и в предыдущем импорте**. В остальных случаях рекомендуется использовать новый манифест.

   :::

3. Ознакомьтесь сo [списком критических изменений](/on-premise-citylens/releases). Если необходимо, обновите другие настройки в конфигурационном файле `values-citylens-ui.yaml`.

4. Выполните команду для обновления сервиса:

   ```bash
   helm upgrade --version=VERSION --atomic --values ./values-citylens-ui.yaml citylens 2gis-on-premise/citylens
   ```

   В параметр `--version` подставьте новую версию компонента `citylens`, которую вы указывали в конфигурационном файле 2GIS CLI.

   :::warning

   Версии сервисов внутри компонента должны быть одинаковыми и совпадать с версией компонента.

   :::

5. [Проверьте работоспособность сервиса](/on-premise-citylens/install/installation#test).

## Обновление CityLens API и CityLens Routes API {#update-citylens-api}

1. Получите новые артефакты для установки.

   1. Укажите в конфигурационном файле 2GIS CLI:

      * Новую версию компонента `citylens` (см. [Релизы <Brand /> Ситискан](/on-premise-citylens/releases)).
      * Версии компонентов `core`, `api-platform` и `pro`, которые требуются для установки новой версии <Brand /> Ситискан (см. подраздел **Зависимости** в описании релизов <Brand /> Ситискан).

   2. [Выполните команду загрузки](/on-premise-api-platform/preparation#fetch-artifacts). Дополнительно вы можете указать следующие флаги:

      * Чтобы обновить только образ сервиса, добавьте флаг `--only-apps` после команды `pull`.
      * Чтобы обновить только данные, добавьте флаг `--only-data` после команды `pull`.
      * Чтобы обновить и сервис, и данные, выполните команду без добавления флагов выше.

      Дополнительно см. [справку по командам и аргументам 2GIS CLI](/on-premise-api-platform/overview/2gis-cli/help).

2. Скопируйте путь к новому файлу манифеста из консольного вывода команды и подставьте его в параметр `dgctlStorage.manifest` в конфигурационном файле `values-citylens-api.yaml`. Подробнее см. в [руководстве по установке](/on-premise-citylens/install/installation#install-services).

   :::note[Выбор манифеста]

   Если при очередном обновлении сервиса <Brand /> Ситискан вы не хотите импортировать новые данные, **используйте тот же манифест, что и в предыдущем импорте**. В остальных случаях рекомендуется использовать новый манифест.

   :::

3. Ознакомьтесь сo [списком критических изменений](/on-premise-citylens/releases). Если необходимо, обновите другие настройки в конфигурационном файле `values-citylens-api.yaml`.

4. Выполните команду для обновления сервиса:

   ```bash
   helm upgrade --version=VERSION --atomic --values ./values-citylens-api.yaml citylens 2gis-on-premise/citylens
   ```

   В параметр `--version` подставьте версию компонента `citylens`:

   * Чтобы обновить только сервис или сервис и данные, используйте новую версию, которую вы указывали в конфигурационном файле 2GIS CLI.
   * Чтобы обновить только данные, подставьте текущую установленную версию.

   :::warning

   Версии сервисов внутри компонента должны быть одинаковыми и совпадать с версией компонента.

   :::

5. [Проверьте работоспособность сервиса](/on-premise-citylens/install/installation#test).
