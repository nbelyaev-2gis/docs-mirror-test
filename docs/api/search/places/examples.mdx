# Примеры

## Поиск компаний

### Ограничение количества результатов поиска {#page-limit}

Чтобы ограничить количество объектов в результатах поиска, используйте параметры:

* `page_size` — количество объектов на странице. Если вы используете [демо-ключ](/platform-manager/subscription/keys#demo) для доступа к API, максимальное значение — 10.
* `page` — номер страницы. Для демо-ключа максимальное значение — 5.

```http
https://catalog.api.2gis.com/3.0/items?q=Москва кафе&type=branch&page_size=10&page=1&key=YOUR_KEY
```

### Поиск по текстовому параметру {#text-query}

```http
https://catalog.api.2gis.com/3.0/items?q=Москва кафе&type=branch&key=YOUR_KEY
```

Для более точного поиска без геоограничения в тексте запроса следует указывать город (посёлок, район, область), где выполняется поиск.

***

### Поиск в радиусе {#within-radius}

```http
https://catalog.api.2gis.com/3.0/items?q=кафе&type=branch&point=37.416469%2C55.619325&radius=1000&key=YOUR_KEY
```

***

### Поиск в радиусе с сортировкой по удалённости {#within-radius-distance}

```http
https://catalog.api.2gis.com/3.0/items?q=кафе&point=37.545423%2C55.740693&radius=1000&location=37.545423%2C55.740693&sort=distance&key=YOUR_KEY
```

***

### Поиск с указанием точки поиска {#with-location}

```http
https://catalog.api.2gis.com/3.0/items?q=кафе&location=37.545423%2C55.740693&key=YOUR_KEY
```

Первыми будут выведены наиболее близкие к точке объекты. Город в тексте запроса можно не указывать.

***

### Поиск в прямоугольной области {#rectangular-area}

```http
https://catalog.api.2gis.com/3.0/items?q=кафе&fields=items.point&point1=37.602631%2C55.764592&point2=37.648702%2C55.743089&key=YOUR_KEY
```

***

### Поиск в произвольной области {#arbitrary-area}

```http
https://catalog.api.2gis.com/3.0/items?q=кафе&fields=items.point&polygon=POLYGON((37.5930 55.7667,37.6494 55.7667,37.6494 55.7405,37.5930 55.7405,37.5930 55.7667))&key=YOUR_KEY
```

Область поиска задается в формате [WKT](https://en.wikipedia.org/wiki/Well-known_text_representation_of_geometry).

***

### Поиск в определённом городе {#within-city}

1. Определите ID города, в котором выполняется поиск, одним из способов:

   * Поиск города по точке:

     ```http
     https://catalog.api.2gis.com/3.0/items?lon=37.62143&lat=55.752966&type=adm_div.city&key=YOUR_KEY
     ```

   * Поиск города по названию:

     ```http
     https://catalog.api.2gis.com/3.0/items?q=Москва&key=YOUR_KEY
     ```

2. В ответе найдите объект с подтипом `city`, скопируйте его идентификатор (значение из поля `id`) и передайте в поле `city_id`:

   ```http
   https://catalog.api.2gis.com/3.0/items?q=кафе&fields=items.point&city_id=4504222397630173&key=YOUR_KEY
   ```

***

### Поиск в определённом городе с сортировкой по рейтингу {#within-city-by-rating}

```http
https://catalog.api.2gis.com/3.0/items?q=красота&city_id=4504222397630173&sort=rating&key=YOUR_KEY
```

***

### Поиск рядом со станцией метро {#near-metro}

Вы можете выполнить поиск двумя способами:

* Поиск с названием станции в запросе:

   ```http
   https://catalog.api.2gis.com/3.0/items?q=метро косино красота&city_id=4504222397630173&key=YOUR_KEY
   ```

* Поиск с фильтром по станции метро:

  1. Определите ID метро:

     ```http
     https://catalog.api.2gis.com/3.0/items?type=station.metro&q=косино&fields=items.point&location=37.617135%2C55.757705&key=YOUR_KEY
     ```

  2. В ответе найдите объект с подтипом `metro`, скопируйте его идентификатор (значение из поля `id`) и передайте в поле `subway`:

     ```http
     https://catalog.api.2gis.com/3.0/items?q=красота&fields=items.point&subway=70030076180130124&location=37.617135%2C55.757705&key=YOUR_KEY
     ```

***

### Поиск в здании {#within-building}

1. Определите ID здания:

   ```http
   https://catalog.api.2gis.com/3.0/items?q=Москва Доватора, 6/6 к8&type=building&key=YOUR_KEY
   ```

2. В ответе найдите объект с типом `building`, скопируйте его идентификатор (значение из поля `id`) и передайте в поле `building_id`, а также укажите тип поиска `search_type=indoor`:

   ```http
   https://catalog.api.2gis.com/3.0/items?q=пункт выдачи&search_type=indoor&building_id=4504235282582289&key=YOUR_KEY
   ```

***

## Список компаний

### Список компаний в категории {#companies-in-category}

1. Определите ID региона, в котором будет выполняться поиск, одним из способов:

   * Поиск региона по названию:

     ```http
     https://catalog.api.2gis.com/2.0/region/search?q=Москва&key=YOUR_KEY
     ```

   * Поиск региона по точке:

     ```http
     https://catalog.api.2gis.com/2.0/region/search?q=37.624186%2C55.754285&key=YOUR_KEY
     ```

   Подробнее о территориальном делении карты на регионы см. в [описании Regions API](/api/search/regions/overview).

2. Определите ID рубрики, по которой будет раскрываться список филиалов. Передайте значения полей `id` нужных рубрик в поле `region_id` при запросе к методу `/2.0/catalog/rubric/search`:

   ```http
   https://catalog.api.2gis.com/2.0/catalog/rubric/search?region_id=32&q=супермаркеты&key=YOUR_KEY
   ```

3. Скопируйте значение из поля `id` и передайте в поле `rubric_id` (или укажите несколько `id` через запятую) при запросе к методу `/3.0/items`. Добавьте геоограничение (`point` + `radius` / `location` / `point1` + `point2` / `polygon` / `city_id`):

   ```http
   https://catalog.api.2gis.com/3.0/items?rubric_id=350,12127&point=37.624186%2C55.754285&radius=1000&key=YOUR_KEY
   ```

***

### Список филиалов одной организации {#branches}

1. Определите ID филиалов организации:

   ```http
   https://catalog.api.2gis.com/3.0/items?q=москва бургеры&fields=items.org&key=YOUR_KEY
   ```

2. Скопируйте значение из поля `items->org->id` и передайте в поле `org_id` (`org_id` организации уникален для каждого региона). Добавьте геоограничение (`point` + `radius` / `location` / `point1` + `point2` / `polygon` / `city_id`):

   ```http
   https://catalog.api.2gis.com/3.0/items?org_id=4504136499435533&city_id=4504222397630173&key=YOUR_KEY
   ```

***

### Список компаний, обслуживающих здания {#servicing-companies}

Обслуживающие компании — это компании, выполняющие работы по эксплуатации здания и оказывающие различные услуги населению. Например, почта, поликлиника, ЖКХ, интернет-провайдеры, школа, пункт полиции.

1. Определите ID здания:

   ```http
   https://catalog.api.2gis.com/3.0/items?q=Москва, Доватора, 9&key=YOUR_KEY
   ```

2. Скопируйте значение из поля `id` (часть до знака «\_») и передайте в поле `building_id` при запросе к методу `/3.0/items/byservicing`:

   * Получение городских обслуживающих организаций, относящихся к зданию (почта, поликлиника, ЖКХ):

     ```http
     https://catalog.api.2gis.com/3.0/items/byservicing?building_id=4504235282737147&servicing_group=default&key=YOUR_KEY
     ```

   * Получение интернет-провайдеров в здании:

     ```http
     https://catalog.api.2gis.com/3.0/items/byservicing?building_id=4504235282737147&servicing_group=internet&key=YOUR_KEY
     ```

   * Получение городских обслуживающих организаций и интернет-провайдеров в одном запросе:

     ```http
     https://catalog.api.2gis.com/3.0/items/byservicing?building_id=4504235282737147&key=YOUR_KEY
     ```

***

### Получение ближайших парковок и остановок {#nearest-parking}

```http
https://catalog.api.2gis.com/3.0/items/byid?id=4504235282757081&fields=items.links&key=YOUR_KEY
```

В теле ответа вы можете найти:

* `links.nearest_stations` — информацию о ближайших остановках общественного транспорта. Обратите внимание на приоритет типов транспорта:

  * Если объект по указанному ID расположен недалеко от метро (в радиусе 1,5-2 км), то в списке ближайших остановок будут только станции метро (включая лёгкое метро и МЦК).
  * Если объект расположен далеко от метро или если в городе поиска нет метро, указывается одна ближайшая (в радиусе 1 км) остановка транспорта. Самый низкий по приоритету тип транспорта — электричка.
  * Если в радиусе поиска нет метро и остановок общественного транспорта, то указывается ближайшая (в радиусе 3 км) железнодорожная станция.

* `links.nearest_parking` — ID ближайших парковок.
