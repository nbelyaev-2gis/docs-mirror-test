# Обзор

Regions API используется для выбора региона, в границах которого выполняется поиск организаций, зданий и мест.

Регион (`region_id`) в поисковых API — это агломерация, включающая крупный город и ближайшие населённые пункты, а также города-спутники. Это территории с повышенной детализацией карты и подробным набором данных и атрибутов справочника. Территории за границами региона (населённые пункты, область в целом, край, республика и т. д.) называются сегментами (`segment_id`).

Обратите внимание, что регионы и сегменты отвечают только внутреннему формату хранения данных в <Brand /> и в общем случае никак не связаны с субъектами Российской Федерации.

Для работы с местами и компаниями за границами регионов используется поиск с геоограничениями.

## Формат запроса {#request}

Пример запроса:

```http
https://catalog.api.2gis.com/2.0/region/search?q=Москва&key=YOUR_KEY
```

В запросе используются следующие параметры:

* `q=Москва` — поиск выполняется по запросу «Москва».
* `key=YOUR_KEY` — ваш ключ API.

## Формат ответа {#response}

Ответ на запрос возвращается в формате JSON:

```json
{
    "meta": {
        "api_version": "2.0.448950",
        "code": 200,
        "issue_date": "20200629"
    },
    "result": {
        "items": [
            {
                "id": "32",
                "name": "Москва",
                "type": "region"
            }
        ],
        "total": 1
    }
}
```

## Примеры использования {#usage-examples}

### Определение области поиска

```http
https://catalog.api.2gis.com/2.0/region/search?q=Москва&key=YOUR_KEY
```

### Поиск категории с указанием области поиска

```http
https://catalog.api.2gis.com/2.0/catalog/rubric/search?q=кафе&region_id=32&key=YOUR_KEY
```

### Поиск филиалов в найденной рубрике

```http
https://catalog.api.2gis.com/3.0/items?rubric_id=162&sort_point=37.622161%2C55.751597&key=YOUR_KEY
```

## Тарифы {#tariffs}

* Стоимость сервиса рассчитывается исходя из количества успешных запросов в месяц. Успешным запросом к API считается запрос, который возвращает HTTP-код 200 в поле `meta.code` в теле ответа, например:

    ```json
    {
        "meta": {
            "api_version": "3.0.17799",
            "code": 200,
            "issue_date": "20240524"
        },
        ...
    }
    ```

* Актуальную стоимость и лимиты см. в разделе [Тарифы](/platform-manager/subscription/pricing).

## Варианты размещения {#distribution}

* **Облако**: все методы Regions API доступны через публичные endpoint-ы <Brand />.
* **On-Premise**: все методы Regions API доступны при установке API-платформы <Brand /> в закрытом контуре. Подробнее см. в разделе [API-платформа для сервера](/on-premise-api-platform/overview/summary).
