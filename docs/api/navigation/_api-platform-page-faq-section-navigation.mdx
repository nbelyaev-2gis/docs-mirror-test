<AccordionItem title="Как начать работать с API навигации?">

Чтобы работать с API, получите ключ доступа:

1. Зарегистрируйтесь в личном кабинете Менеджер Платформы.

   <CtaButton href="https://platform.2gis.ru/ru/?utm_source=docs&utm_medium=referral&utm_campaign=api_navi_faq" target="_blank">
      В кабинет
   </CtaButton>

2. Создайте [демо-ключ](/platform-manager/subscription/keys#demo) или купите [подписку](/platform-manager/subscription/purchase) для доступа к API.
3. Изучите примеры запросов и ответов в документации к нужному API.

</AccordionItem>

<AccordionItem title="Как рассчитывается стоимость использования API?">

Стоимость использования API навигации может рассчитываться по количеству запросов к API, количеству построенных маршрутов или количеству расчётов времени и расстояния между парами точек. Подробнее о способе расчёта см. в разделе **Тарифы** на странице каждого API.

</AccordionItem>

<AccordionItem title="Можно ли использовать API навигации бесплатно?">

Вы можете [оформить демо-ключ доступа](/platform-manager/subscription/keys#demo) для тестирования API навигации на один месяц.

</AccordionItem>

<AccordionItem title="Как отслеживать фактическое использование API-ресурсов?">

Вы можете отслеживать прогресс расходования лимитов по сервисам и статистику распределения запросов в личном кабинете [Менеджер Платформы](https://platform.2gis.ru/ru/?utm_source=docs&utm_medium=referral&utm_campaign=api_navi_faq). Подробнее см. в разделе [Статистика](/platform-manager/subscription/statistics).

</AccordionItem>

<AccordionItem title="Как контролировать потребление API-ресурсов?">

Чтобы избежать преждевременного исчерпания лимитов, вы можете ограничить использование API-ресурсов вашими приложениями. В личном кабинете [Менеджер Платформы](https://platform.2gis.ru/ru/?utm_source=docs&utm_medium=referral&utm_campaign=api_navi_faq) доступно несколько способов настройки ограничений:

- Установить лимиты на запросы к конкретным сервисам по всем API-ключам в подписке или по отдельным ключам. Превышение лимитов заблокирует использование продукта: запросы к API будут возвращать ошибку.
- Настроить отправку уведомлений при приближении к лимитам.

Подробнее см. в разделе [Управление ключами доступа](/platform-manager/subscription/managing-keys).

</AccordionItem>

<AccordionItem title="Как получить информацию о пробках?">

Routing API, Distance Matrix API, TSP API, Isochrone API и Route Planner API поддерживают учёт текущих или статистических данных о дорожной обстановке, чтобы наиболее точно рассчитать оптимальный маршрут и время в пути. Отдельно информация о пробках не предоставляется.

</AccordionItem>

<AccordionItem title="Как быстро обновляются данные о дорожной обстановке?">

Информация о дорожной обстановке (пробки и перекрытия) обновляется в <Brand /> раз в несколько минут и становится доступна в ваших проектах одновременно с продуктами <Brand />.

</AccordionItem>

<AccordionItem title="Как отобразить маршрут на карте?">

- Для веб-приложений: установите векторную карту с помощью MapGL JS API и отобразите маршрут с помощью плагина Directions. Подробнее см. в разделе [Построение маршрутов](/mapgl/map/tools/directions).
- Для мобильных приложений: инициализируйте мобильный SDK <Brand /> и используйте инструменты для создания карты и построения маршрутов. Подробнее см. в разделе **Навигация** в документации SDK для [iOS](/ios/sdk/examples/routing), [Android](/android/sdk/examples/routing) или [Flutter](/flutter/sdk/examples/routing).

</AccordionItem>

<AccordionItem title="Как создать навигатор по построенному маршруту?">

Используйте мобильный SDK <Brand /> для создания навигатора по построенному маршруту. SDK предоставляет инструменты для отслеживания местоположения пользователя, поиску альтернативных маршрутов в пути, голосовых подсказок и других функций навигатора. Подробнее см. в разделе **Навигация** в документации SDK для [iOS](/ios/sdk/examples/navigation), [Android](/android/sdk/examples/navigation) или [Flutter](/flutter/sdk/examples/navigation).

</AccordionItem>

</AccordionItem>
