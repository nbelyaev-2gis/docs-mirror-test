# Релизы

## 1 декабря 2025 г.

* Обновлена сетка тарифов на API-сервисы. Подробнее об изменениях см. в [личном кабинете](https://platform.2gis.ru/ru/tariffs?utm_source=docs&utm_medium=referral&utm_campaign=api_dm).

## 18 ноября 2025 г.

* Добавлена возможность строить [маршруты для общественного транспорта](/api/navigation/distance-matrix/examples#public-transport) в синхронном и асинхронном режимах.

## 16 сентября 2025 г.

* Добавлена возможность строить [маршруты для мотоциклов](/api/navigation/distance-matrix/examples#motorcycle) в синхронном режиме.

## 03 февраля 2025 г.

* Добавлена возможность строить [самокатные маршруты](/api/navigation/distance-matrix/examples#scooter) в синхронном режиме.
* Добавлена возможность строить пешеходные, велосипедные и самокатные маршруты в асинхронном режиме.
* Добавлен параметр запроса `need_altitudes`, с помощью которого можно [получить информацию о высоте](/api/navigation/distance-matrix/examples#route-altitude) пешеходного и велосипедного маршрута при построении в синхронном и асинхронном режиме.

## 25 ноября 2024 г.

* Параметр запроса `mode`, который указывает тип движения, заменён на параметр `transport`.

## 30 сентября 2024 г.

* Добавлен параметр запроса [vehicle\_speed\_limit](/api/navigation/distance-matrix/reference/get_dist_matrix#:~:text=vehicle_speed_limit), с помощью которого можно задать [дополнительное ограничение скорости движения](/api/navigation/distance-matrix/examples#additional-speed-limit) при построении маршрута.

## 28 августа 2023 г.

* Добавлена возможность строить маршруты для [грузового транспорта](/api/navigation/distance-matrix/examples#truck). При построении можно указать дополнительные параметры грузового автомобиля в [truck\_params](/api/navigation/distance-matrix/reference/get_dist_matrix#:~:text=truck_params).

## 18 мая 2023 г.

* Добавлен новый код ошибки `422` для запроса [/get\_dist\_matrix](/api/navigation/distance-matrix/reference/get_dist_matrix).

## 30 июня 2021 г.

* Добавлена возможность строить велосипедные маршруты (параметр запроса `mode: "bicycle"`). При построении велосипедного маршрута можно исключить новые типы дорог (с помощью параметра [filters](/api/navigation/distance-matrix/reference/get_dist_matrix#:~:text=filters)): подземные/надземные переходы, лестницы и автомобильные дороги.

## 31 марта 2021 г.

* Добавлена возможность задать области, которые следует избегать при построении маршрута. См. раздел [Исключение областей](/api/navigation/distance-matrix/examples#exclude) для подробной информации.
* Добавлена возможность указать направление движения в начальной и конечной точках маршрута (см. параметр [azimuth](/api/navigation/distance-matrix/reference/get_dist_matrix#:~:text=azimuth)).
* Добавлена возможность строить автомобильные маршруты с учётом полос общественного транспорта (параметр запроса `mode: "taxi"`).
* Для начальной точки маршрута теперь можно указать необязательный параметр [distance\_direction](/api/navigation/distance-matrix/reference/get_dist_matrix#:~:text=distance_direction), который определяет расстояние, на котором должно быть сохранено направление движения. Если параметр `distance_direction` указан, ответ будет содержать дополнительное поле `reliability`, показывающее точность построенного маршрута.
* Добавлен параметр запроса [type](/api/navigation/distance-matrix/reference/get_dist_matrix#:~:text=type), с помощью которого можно изменить тип маршрута (кратчайший по времени или расстоянию).
* Добавлена поддержка параметра запроса [filters](/api/navigation/distance-matrix/reference/get_dist_matrix#:~:text=filters), который используется для исключения типов дорог из маршрута. Подробнее см. раздел [Исключение типов дорог](/api/navigation/distance-matrix/examples#exclude).
* Параметр запроса `start_time` стал необязательным. Если время начала движения не задано, используется текущее время.
* Удалено поле `mode` в ответе.
