# Обзор

TSP API позволяет решить [задачу коммивояжёра](https://ru.wikipedia.org/wiki/Задача_коммивояжёра) (TSP/VRP): построить кратчайший по времени или расстоянию маршрут обхода указанных точек. Для обхода точек можно указать как одного курьера, так и нескольких. Если указано несколько курьеров, маршрут будет разделён между ними наиболее оптимальным способом (некоторые курьеры могут быть исключены, если использование меньшего числа курьеров будет оптимальнее).

Для маршрута можно указать до 4000 точек и до 200 курьеров.

## Получение ключа доступа {#getting-access-key}

1. Зарегистрируйтесь в личном кабинете Менеджер Платформы.

   </CtaButton>

2. Создайте [демо-ключ](/platform-manager/subscription/keys#demo) или купите [подписку](/platform-manager/subscription/purchase) для доступа к API. Подробнее о стоимости сервиса см. в разделе [Тарифы](#tariffs).

Подробнее о работе с ключами и подписками см. в [документации личного кабинета](/platform-manager/overview).

## Пример задачи {#task-example}

* используется один курьер с грузоподъёмностью до 2000 кг;

* курьер начинает и заканчивает путь на складе;

TSP API решает задачу коммивояжёра в асинхронном режиме. Чтобы построить маршрут (порядок прохождения точек):

1. Отправьте запрос, чтобы создать задачу на построение маршрута.
2. Периодически проверяйте статус задачи, пока не завершится расчёт маршрута.
3. Получите готовый маршрут после завершения задачи.

Подробнее о параметрах из запросов ниже см. в [Справочнике API](/api/navigation/tsp/reference/create_200).

:::tip

TSP API предлагает наиболее оптимальный *порядок* прохождения указанных точек с учётом всех особенностей дорожной ситуации. Чтобы получить детальный маршрут проезда или прохода, используйте [Routing API](/api/navigation/routing/overview) после получения решения от TSP API.

:::

### 1. Создание задачи {#create-task}

Чтобы создать задачу на построение маршрута, отправьте POST-запрос на endpoint `/logistics/vrp/2.0.0/create`. В строке запроса укажите ваш ключ API в качестве значения параметра `key`:

```http
https://routing.api.2gis.com/logistics/vrp/2.0.0/create?key=API_KEY
```

Координаты точек маршрута, количество курьеров и другие параметры передайте в виде JSON в теле запроса. Например, чтобы задать параметры обхода из [примера задачи](#task-example):

1. Перечислите все точки для посещения в массиве `waypoints` и укажите параметры:

    * `id` (обязательный): идентификатор точки.
    * `point` (обязательный): координаты точки (`lat`, `lon`) и её расположение относительно дороги (`type`).
    * `type`: [тип точки](/api/navigation/tsp/examples#types) (`delivery`, `pickup` или `agent`).
    * `service_duration_s`: [продолжительность работы курьера в точке](/api/navigation/tsp/examples#service-time) (в секундах).
    * `depot_ids`: массив идентификаторов [складов](/api/navigation/tsp/examples#depots), с которых курьеры могут забрать заказ.
    * `time_windows`: [временные окна](/api/navigation/tsp/examples#time-windows).
    * `shipment_size`: размер груза.

   ```json
   {
       "waypoints": [
           {
            "id": "1",
            "point": { "lat": 55.708272, "lon": 37.46326, "type": "stop" },
            "type": "delivery",
            "service_duration_s": 0,
            "depot_ids": ["0"],
            "time_windows": [
                {
                    "time_window": "07:00-15:00",
                    "hard_time_window": "07:00-15:00"
                }
            ],
            "shipment_size": { "weight_kg": 800 }
            }
       ]
   }
   ```

2. Перечислите курьеров в массиве `agents` и укажите параметры:

    * `id` (обязательный): идентификатор курьера.
    * `capacity`: грузоподъёмность курьера.
    * `travel_time_multiplier`: коэффициент скорости движения.
    * `cost`: стоимость работы курьера.
    * `start_at`: идентификатор склада, из которого курьер начинает маршрут.
    * `finish_at`: идентификатор склада, в который курьер возвращается.

   ```json
   {
    "agents": [
        {
            "id": "1",
            "capacity": { "weight_kg": 2000 },
            "travel_time_multiplier": 1.0,
            "cost": {
                "fixed": 3000,
                "hour": 100,
                "km": 8
            },
            "start_at": "0",
            "finish_at": "0"
        }
      ]
    }
   ```

3. (Опционально) Перечислите склады в массиве `depots`:

    * `id`: идентификатор склада.
    * `point`: координаты склада с указанием типа.
    * `service_duration_s`: время обслуживания на складе.
    * `time_window`: временное окно работы склада (`time_window`, `hard_time_window`).

    ```json
    {
      "depots": [
        {
            "id": "0",
            "point": {
                "lat": 55.734157,
                "lon": 37.589346,
                "type": "stop"
            },
            "service_duration_s": 0,
            "time_window": {
                "time_window": "07:00-15:00",
                "hard_time_window": "07:00-15:00"
             }
        }
      ]
    }
    ```

4. (Опционально) Укажите параметры маршрута в объекте `options`:

    * `route_type`: тип маршрута (например, `"jam"` — с учётом пробок).
    * `routing_type`: способ передвижения (например, `"driving"`).
    * `date`: дата маршрута в формате `YYYY-MM-DD`.
    * `time_zone`: часовой пояс (например, `"Europe/Moscow"`).
    * `solver_time_limit_s`: ограничение времени на решение задачи (в секундах).
    * `need_altitudes`: нужно ли учитывать высоты (`true` или `false`).

   ```json
   {
       "options": {
            "route_type": "jam",
            "routing_type": "driving",
            "date": "2021-02-19",
            "time_zone": "Europe/Moscow",
            "solver_time_limit_s": 60,
            "need_altitudes": false
        }
   }
   ```

#### Ограничения {#limitations}

Запрос не будет выполнен, если: 

* начальная или конечная точка курьера была исключена из маршрута;
* время выполнения запроса не входит в рабочее время ни одного из курьеров.

### 2. Проверка статуса задачи {#check-task-status}

Чтобы проверить статус задачи, отправьте GET-запрос на endpoint `/logistics/vrp/2.0.0/status`. В строке запроса укажите два параметра:

* `task_id` — идентификатор задачи;
* `key` — ваш ключ API.

```bash
curl --location --request GET 'https://routing.api.2gis.com/logistics/vrp/2.0.0/status?task_id=dcabb3f6188935187d54088556238b83&key=API_KEY'
```

Если задача в процессе обработки, запрос вернёт тот же ответ, что и при создании задачи (поле `status` будет содержать значение `"Run"`):

```json
{
    "task_id": "dcabb3f6188935187d54088556238b83",
    "status": {
        "status": "Run",
        "queued": "2026-04-20T11:44:14.906+00:00",
        "started": null,
        "matrix_downloaded": null,
        "calculated": null,
        "completed": null
    },
    "result": null
}
```

Если задача завершена, тело ответа будет содержать следующие данные:

* `status`: одно из следующих значений:

  * `Done` — маршрут успешно построен.
  * `Partial` — маршрут построен, но из него были исключены точки или курьеры.
  * `Fail` — при построении маршрута возникла ошибка.

* `queued`: время постановки задачи в очередь в формате ISO 8601.

* `started`: время начала обработки.

* `matrix_downloaded`: время завершения загрузки матрицы достижимости (расстояний и времени в пути между всеми возможными парами точек из задачи).

* `calculated`: время завершения расчёта.

* `completed`: время завершения задачи. 

```json
{
    "task_id": "dcabb3f6188935187d54088556238b83",
     "status": {
        "status": "Done",
        "queued": "2026-04-20T11:44:14.906+00:00",
        "started": "2026-04-20T11:44:14.906+00:00",
        "matrix_downloaded": "2026-04-20T11:44:15.094+00:00",
        "calculated": "2026-04-20T11:44:15.144+00:00",
        "completed": "2026-04-20T11:44:15.144+00:00"
    }
}
```

## Тарифы {#tariffs}

* Стоимость сервиса рассчитывается исходя из объёма решаемой задачи: количества точек для посещения, умноженного на количество курьеров. Например, если в задаче 5 точек и 2 курьера, оплачиваются 10 расчётов.
