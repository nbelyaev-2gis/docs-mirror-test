# Обзор примеров

В разделе приведены основные сценарии работы и примеры запросов к Routing API. Подробную информацию о параметрах запросов см. в [Справочнике API](/api/navigation/routing/reference/routing).

Обязательные параметры для построения маршрута:

* [Ваш API-ключ](/api/navigation/routing/start) (`key`).
* Минимум две [точки маршрута](/api/navigation/routing/examples/points):

   * Для общественного транспорта: координаты начальной (`source`) и конечной (`target`) точки маршрута.
   * Для остальных [способов передвижения](/api/navigation/routing/examples/types): координаты и тип точек в массиве `points`.

* Только для общественного транспорта: [вид транспорта](/api/navigation/routing/examples/types#public-transport) (`transport`).

<br />

Если указаны только обязательные параметры, строятся маршруты со следующими свойствами (при возможности строится несколько [вариантов маршрута](/api/navigation/routing/examples/options)):

* Для общественного транспорта:

   * Кратчайший по времени.
   * C текущим временем отправления по местному времени.
   * С учётом расписания движения общественного транспорта.

* Для остальных способов передвижения:

   * Маршрут на автомобиле.
   * Кратчайший по времени.
   * С учётом пробок и дорожных перекрытий, актуальных на момент отправки запроса.

Вы можете изменить эти параметры или добавить дополнительные. Примеры запросов приведены с использованием [cURL](https://curl.se/).
