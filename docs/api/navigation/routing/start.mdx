# Начало работы

Выполните шаги ниже, чтобы познакомиться с возможностями Routing API и отправить запрос на построение маршрута на автомобиле или на общественном транспорте. Чтобы построить маршруты для других способов передвижения (например, на велосипеде или пешком), см. инструкцию [Способы передвижения](/api/navigation/routing/examples/types).

Примеры запросов приведены с использованием [cURL](https://curl.se/).

## Получение ключа доступа {#get-access-key}

1. Зарегистрируйтесь в личном кабинете Менеджер Платформы.

   <CtaButton href="https://platform.2gis.ru/ru/?utm_source=docs&utm_medium=referral&utm_campaign=api_routing" target="_blank">
      В кабинет
   </CtaButton>

2. Создайте [демо-ключ](/platform-manager/subscription/keys#demo) или купите [подписку](/platform-manager/subscription/purchase) для доступа к API. Подробнее о стоимости сервиса см. в разделе [Тарифы](/api/navigation/routing/overview#tariffs).

Подробнее о работе с ключами и подписками см. в [документации личного кабинета](/platform-manager/overview).

## Построение маршрута на автомобиле {#car-route}

Чтобы построить маршрут на автомобиле для одной пары точек, отправьте POST-запрос к `/routing/7.0.0/global`:

1. В строке запроса укажите значение API-ключа в параметре `key`:

   ```http
   https://routing.api.2gis.com/routing/7.0.0/global?key=API_KEY
   ```

2. В теле запроса передайте JSON с необходимыми параметрами:

   ```json
   {
       "points": [
           {
               "type": "stop",
               "lon": 37.582591,
               "lat": 55.775364
           },
           {
               "type": "stop",
               "lon": 37.656625,
               "lat": 55.765036
           }
       ],
       "transport": "driving",
       "route_mode": "fastest",
       "traffic_mode": "jam",
       "locale": "ru"
   }
   ```

### Пример запроса {#car-request-example}

```curl
curl --request POST \
   --url 'https://routing.api.2gis.com/routing/7.0.0/global?key=API_KEY' \
   --header 'Content-Type: application/json' \
   --data '{
      "points": [
         {
            "type": "stop",
            "lon": 37.582591,
            "lat": 55.775364
         },
         {
            "type": "stop",
            "lon": 37.656625,
            "lat": 55.765036
         }
      ],
      "transport": "driving",
      "route_mode": "fastest",
      "traffic_mode": "jam",
      "locale": "ru"
}'
```

Где:

* `points` — массив координат точек маршрута: начальной и конечной.
* `transport: driving` — способ передвижения: на автомобиле.
* `route_mode: fastest` — тип построения маршрута: кратчайший по времени.
* `traffic_mode: jam` — построение маршрута с учётом текущих пробок.
* `locale: ru` — язык текстовых описаний элементов маршрута: русский.

Подробную информацию о параметрах см. в [Справочнике API](/api/navigation/routing/reference/routing).

* `end_pedestrian_path` — геометрия пешеходного участка маршрута от последнего манёвра до конечной точки маршрута в формате WKT (Well-Known Text).
* `filter_road_types` — типы дорог, которые были исключены из варианта маршрута.
* `id` — ID варианта маршрута.
* `maneuvers` — массив с данными об участках маршрута (манёврах). Каждый манёвр содержит:

   * `comment` — описание манёвра.
   * `icon` — иконка манёвра.
   * `id` — ID манёвра.
   * `outcoming_path` — путь от манёвра до следующего манёвра:

      * `distance` — длина манёвра в метрах.
      * `duration` — время в пути на манёвре в секундах.
      * `geometry` — геометрия пути в формате WKT (Well-Known Text).
      * `names` — названия улиц, по которым проходит путь.

      Если манёвр последний на варианте маршрута, то `outcoming_path` отсутствует.

   * `outcoming_path_comment` — описание манёвра в текстовом виде.
   * `type` — тип манёвра (например, перекрёсток, начало маршрута, конец маршрута и т. д.).

* `reliability` — надёжность варианта маршрута от 0 до 1.
* `route_id` — уникальный идентификатор варианта маршрута.
* `total_distance` — общая длина варианта маршрута в метрах.
* `total_duration` — общее время в пути на варианте маршрута в секундах.
* `type` — способ передвижения на варианте маршрута.
* `ui_total_distance` — общая длина варианта маршрута в текстовом виде.
* `ui_total_duration` — общее время в пути на варианте маршрута в текстовом виде.
* `waypoints` — массив исходных точек и точек притяжки для варианта маршрута.

Подробнее о каждом параметре см. в [Справочнике API](/api/navigation/routing/reference/routing).

## Построение маршрута на общественном транспорте {#public-transport-route}

Чтобы построить маршрут на общественном транспорте, отправьте POST-запрос к `/public_transport/2.0`:

1. В строке запроса укажите значение API-ключа в параметре `key`:

   ```http
   https://routing.api.2gis.com/public_transport/2.0?key=API_KEY
   ```

2. В теле запроса передайте JSON с необходимыми параметрами:

   ```json
   {
      "source":
      {
         "point":
         {
            "lat": 55.798227,
            "lon": 37.697461
         }
      },
      "target":
      {
         "point":
         {
            "lat": 55.821029,
            "lon": 37.641507
         }
      },
      "transport": ["metro", "tram"],
      "locale": "ru"
   }
   ```

### Пример запроса {#public-request-example}

```curl
curl --request POST \
  --url 'https://routing.api.2gis.com/public_transport/2.0?key=API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "source":
    {
      "point":
      {
        "lat": 55.798227,
        "lon": 37.697461
      }
    },
    "target":
    {
      "point":
      {
        "lat": 55.821029,
        "lon": 37.641507
      }
    },
    "transport": ["metro", "tram"],
    "locale": "ru"
}'
```

Где:

* `source` — координаты начальной точки (широта и долгота).
* `target` — координаты конечной точки (широта и долгота).
* `transport: [metro, tram]` — массив видов общественного транспорта: `metro` — метро и `tram` — трамвай.
* `locale: ru` — язык текстовых описаний элементов маршрута: русский.

Подробную информацию о параметрах см. в [Справочнике API](/api/navigation/routing/reference/public_transport).

* `transfer_count` — количество пересадок в рамках одной остановочной платформы для варианта маршрута. Например, пересадка с одного маршрута автобуса на другой. Подробнее см. в разделе [Транспорт и пересадки](/api/navigation/routing/examples/options#route-transport).
* `waypoints` — краткая информация о варианте маршрута: можно ли проехать маршрут на одном виде транспорта, виды транспорта и т. д. Подробнее см. в разделе [Транспорт и пересадки](/api/navigation/routing/examples/options#route-transport).

Подробнее о каждом параметре см. в [Справочнике API](/api/navigation/routing/reference/public_transport) и в разделе [Общественный транспорт](/api/navigation/routing/examples/options#public).
