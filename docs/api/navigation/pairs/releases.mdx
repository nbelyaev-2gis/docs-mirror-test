# Релизы

## 25 ноября 2024 г.

* Для параметра запроса `routing_type`, который указывает тип маршрутизации, значения `car` и `pedestrian` заменены на `driving` и `walking` соответственно.

## 28 июля 2023 г.

* Опубликована первая версия [Routing API](/api/navigation/routing/overview) с полной поддержкой всех функций [Directions API](/api/navigation/directions/overview), [Pairs Directions API](/api/navigation/pairs/overview) и [Truck Directions API](/api/navigation/truck-directions/overview).

## 30 июня 2021 г.

* Добавлена возможность построения велосипедных маршрутов (параметр запроса `routing_type: "bicycle"`). При построении велосипедного маршрута можно исключить новые типы дорог (с помощью параметра [filters](/api/navigation/pairs/reference#:~:text=filters)): автомобильные дороги, лестницы, подземные и надземные переходы.

## 31 марта 2021 г.

* Добавлена возможность задать области, которые следует избегать при построении маршрута, при помощи параметра запроса `exclude` (см. раздел [Исключение областей](/api/navigation/directions/overview#исключение-областей)).
* Добавлена возможность указать направление движения в начальной и конечной точках (параметры запроса `azimuth1` и `azimuth2` для каждой пары точек).
* Для начальной точки маршрута теперь можно указать необязательный параметр [distance\_direction](/api/navigation/pairs/reference#:~:text=distance_direction), который определяет расстояние, на котором должно быть сохранено направление движения. Если параметр `distance_direction` указан, ответ будет содержать дополнительное поле `reliability`, показывающее точность построенного маршрута.
* Добавлена возможность строить автомобильные маршруты с учётом полос общественного транспорта (параметр запроса `routing_type: "taxi"`).
